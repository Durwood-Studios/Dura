## Step 1: What Hiring Managers Actually Look For

Your portfolio isn't a gallery of projects — it's **evidence of competence**. Hiring managers want to see:

1. **You can ship** — Finished, deployed projects that actually work
2. **You can think** — Technical decisions explained clearly
3. **You can learn** — Projects that show growth over time
4. **You care about quality** — Clean code, good UX, accessible design

**What doesn't impress:**

- 20 half-finished projects
- Tutorial follow-alongs without customization
- Beautiful landing pages with no functionality
- Projects with no README explaining what they do

**What does impress:**

- 3-4 solid projects that solve real problems
- Clean, well-documented code on GitHub
- A live demo link for each project
- A brief explanation of your technical choices

## Step 2: Choose Your Projects Strategically

Your projects should demonstrate **breadth and depth**:

**Project 1 — Full-stack app** (shows end-to-end skills):

- User authentication
- Database CRUD operations
- Responsive UI
- Example: Task manager, expense tracker, recipe app

**Project 2 — Technical challenge** (shows problem-solving):

- Algorithm visualization
- Data processing pipeline
- Real-time feature (WebSockets, live updates)
- Example: Markdown editor, code playground, chat app

**Project 3 — Open source contribution** (shows collaboration):

- Link to your merged PRs
- Explain what you contributed and why it mattered

**Project 4 — Something you're passionate about** (shows personality):

- A tool that solves your own problem
- Something related to your hobbies
- Example: Workout tracker, music playlist analyzer, book recommendation engine

<Checkpoint id="project-selection" label="Project Selection" status="active">
  Pick projects that tell a story about your skills and interests.
</Checkpoint>

## Step 3: Build Projects That Stand Out

The difference between a "tutorial project" and a "portfolio project":

**Tutorial project (don't use as-is):**

```text
- Todo app from a YouTube tutorial
- Generic e-commerce template
- Weather app that just calls an API
```

**Portfolio project (customized, extended):**

```text
- Todo app with drag-and-drop, offline sync, and keyboard shortcuts
- E-commerce with real Stripe integration and inventory management
- Weather app with location-based alerts, historical data charts, and PWA support
```

**The customization formula:**

1. Start with a common project idea
2. Add one unique feature that shows technical depth
3. Add one feature that shows product thinking (UX improvement)
4. Polish the details (loading states, error handling, empty states)

## Step 5: Build Your Portfolio Site

Your portfolio site itself is a project. Keep it simple and fast:

**Must-have sections:**

- **Hero** — Your name, title, one-line pitch
- **Projects** — 3-4 projects with screenshots, descriptions, and links
- **About** — Brief background and what you're looking for
- **Contact** — Email, GitHub, LinkedIn

**Technical choices for the portfolio itself:**

```typescript
// Next.js static site — fast, free hosting, shows you know the framework
// Use the same stack as your strongest project to demonstrate consistency

// pages/index.tsx
export default function Home() {
  return (
    <main>
      <Hero />
      <Projects />
      <About />
      <Contact />
    </main>
  );
}
```

**Design tips:**

- Dark mode by default (developers love it)
- Clean typography — let the content breathe
- Fast loading — aim for 100 on Lighthouse performance
- Mobile responsive — recruiters often browse on phones
- Accessible — proper heading hierarchy, alt text, keyboard navigation

<Checkpoint id="portfolio-site" label="Portfolio Site" status="active">
  Your portfolio site is itself a project. Make it reflect your standards.
</Checkpoint>

## Step 6: Optimize Your GitHub Profile

Your GitHub profile is your developer resume:

**Profile README** (create a repo named after your username):

```markdown
# Hi, I'm [Name] 👋

I'm a [role] focused on [specialization].

## Currently working on

- [Project] — brief description

## Technologies

![TypeScript](https://img.shields.io/badge/-TypeScript-3178C6?logo=typescript&logoColor=white)
![React](https://img.shields.io/badge/-React-61DAFB?logo=react&logoColor=black)

## Recent activity

<!-- GitHub activity widget or pinned repos -->
```

**Pin your best repositories:**

1. Go to your GitHub profile
2. Click "Customize your pins"
3. Select your 4-6 best projects

**Green squares matter (but quality > quantity):**

- Consistent commits show work ethic
- But don't pad with meaningless commits
- Contributions to real projects matter more than personal repo noise

<Checkpoint id="github-profile" label="GitHub Profile" status="active">
  Optimize your GitHub profile. It's often the first thing people check.
</Checkpoint>

## Step 7: The Portfolio Launch Checklist

Before sharing your portfolio:

- [ ] All project links work (live demos + GitHub repos)
- [ ] Screenshots are current and high-quality
- [ ] READMEs are complete for each project
- [ ] Code is clean — no console.logs, no commented-out code, no secrets
- [ ] Portfolio site loads in under 3 seconds
- [ ] Works on mobile
- [ ] Contact information is current
- [ ] GitHub profile is optimized with pinned repos
- [ ] LinkedIn matches your portfolio narrative

**Where to share:**

- LinkedIn posts about your projects
- Twitter/X developer community
- Discord communities (Reactiflux, Next.js, etc.)
- Dev.to or Hashnode blog posts about what you built
- Reddit (r/webdev, r/reactjs, r/cscareerquestions)

<Checkpoint id="launch" label="Portfolio Launch" status="active">
  Ship it. A published portfolio beats a perfect draft that nobody sees.
</Checkpoint>
