## Why this matters

Individual CSS properties are tools. Building a real page means combining those tools into a coherent system. Professional projects don't write CSS one-off — they establish a reset, define design tokens as custom properties, and compose layouts from flexbox and grid. This lesson ties everything together into the workflow you'll use on real projects.

## Step 1: Modern CSS reset

Browsers have default styles (margins on `<body>`, padding on `<ul>`, different font sizes on headings). A reset normalizes these so you start from a consistent baseline.

```css
/* Modern CSS reset — minimal and complete */
*,
*::before,
*::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

html {
  -webkit-text-size-adjust: 100%;
  text-size-adjust: 100%;
}

body {
  min-height: 100dvh;
  line-height: 1.8;
  -webkit-font-smoothing: antialiased;
}

img,
picture,
video,
canvas,
svg {
  display: block;
  max-width: 100%;
}

input,
button,
textarea,
select {
  font: inherit;
}

p,
h1,
h2,
h3,
h4,
h5,
h6 {
  overflow-wrap: break-word;
}
```

Key decisions in this reset: `border-box` everywhere (lesson 2), `min-height: 100dvh` on body (lesson 7), `font: inherit` on form elements (browsers don't do this by default), and `max-width: 100%` on images (prevents overflow).

## Step 2: Design system variables

CSS custom properties (variables) centralize your design decisions. Change one value, and every element using it updates:

```css
:root {
  /* Colors */
  --bg-primary: #08080d;
  --bg-surface: rgba(255, 255, 255, 0.04);
  --bg-surface-hover: rgba(255, 255, 255, 0.06);
  --border: rgba(255, 255, 255, 0.08);
  --text-primary: #f0f0f0;
  --text-secondary: #a0a0a8;
  --text-muted: #6b6b75;
  --accent: #10b981;

  /* Typography */
  --font-body: "DM Sans", system-ui, sans-serif;
  --font-code: "JetBrains Mono", ui-monospace, monospace;

  /* Spacing scale */
  --space-xs: 0.25rem; /* 4px */
  --space-sm: 0.5rem; /* 8px */
  --space-md: 1rem; /* 16px */
  --space-lg: 1.5rem; /* 24px */
  --space-xl: 2rem; /* 32px */
  --space-2xl: 3rem; /* 48px */

  /* Radius */
  --radius-sm: 8px;
  --radius-md: 12px;
  --radius-lg: 16px;
}
```

Use these variables everywhere instead of raw values:

```css
.card {
  background: var(--bg-surface);
  border: 1px solid var(--border);
  border-radius: var(--radius-md);
  padding: var(--space-lg);
}
```

Custom properties cascade and can be scoped. Override them on a container to theme an entire section:

```css
.light-section {
  --bg-primary: #ffffff;
  --text-primary: #1a1a2e;
  --border: rgba(0, 0, 0, 0.1);
}
```

Every child inside `.light-section` that uses these variables automatically gets the light values.

## Step 3: Page layout with Grid

The page shell uses CSS Grid for the two-dimensional structure:

```css
.page {
  display: grid;
  grid-template-columns: 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header"
    "main"
    "footer";
  min-height: 100dvh;
}

@media (min-width: 1024px) {
  .page {
    grid-template-columns: 250px 1fr;
    grid-template-areas:
      "sidebar header"
      "sidebar main"
      "sidebar footer";
  }
}

.header {
  grid-area: header;
}
.sidebar {
  grid-area: sidebar;
}
.main {
  grid-area: main;
}
.footer {
  grid-area: footer;
}
```

On mobile: single column, no sidebar. On desktop: sidebar + content. The sidebar only appears at 1024px+.

## Step 4: Component layout with Flexbox

Inside the grid areas, flexbox handles component-level layout:

```css
/* Navigation bar */
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: var(--space-md) var(--space-lg);
  border-bottom: 1px solid var(--border);
}

/* Card grid inside main content */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: var(--space-lg);
  padding: var(--space-lg);
}

/* Individual card using flexbox for internal layout */
.card {
  display: flex;
  flex-direction: column;
  gap: var(--space-md);
  padding: var(--space-lg);
  background: var(--bg-surface);
  border: 1px solid var(--border);
  border-radius: var(--radius-md);
  transition:
    background 200ms ease,
    border-color 200ms ease;
}

.card:hover {
  background: var(--bg-surface-hover);
  border-color: var(--accent);
}

/* Push the card footer to the bottom regardless of content height */
.card-footer {
  margin-top: auto;
}
```

The `margin-top: auto` trick in flexbox pushes the footer to the bottom of the card. This ensures all cards in a row have their footers aligned, even when content heights vary.

## Step 5: Responsive typography

```css
body {
  font-family: var(--font-body);
  font-size: 1rem;
  color: var(--text-primary);
}

h1 {
  font-size: clamp(1.75rem, 4vw, 2.5rem);
  font-weight: 700;
}
h2 {
  font-size: clamp(1.375rem, 3vw, 1.875rem);
  font-weight: 600;
}
h3 {
  font-size: clamp(1.125rem, 2.5vw, 1.5rem);
  font-weight: 600;
}

.prose {
  max-width: 700px;
  line-height: 1.8;
  color: var(--text-secondary);
}
```

`clamp()` creates fluid typography that scales smoothly between breakpoints without media queries. The syntax is `clamp(minimum, preferred, maximum)`.

## The complete picture

Grid handles the page shell. Flexbox handles components. Custom properties centralize design decisions. Media queries add breakpoints where the layout needs them. `clamp()` handles fluid typography. Transitions add polish. The CSS reset ensures consistency.

These aren't separate techniques — they're layers of a system. Every professional CSS codebase uses all of them together.

{`// Build a spacing scale generator.
// Given a base size in px and a scale ratio,
// generate a spacing scale object.

function spacingScale(basePx, ratio, steps) {
// steps is an array of names like ["xs", "sm", "md", "lg", "xl"]
// Each step multiplies the previous by the ratio.
// "md" (middle) uses the basePx.
// Steps before "md" divide, steps after multiply.
// Return an object mapping names to rem values (assuming 16px root).

const midIndex = Math.floor(steps.length / 2);
const scale = {};

for (let i = 0; i < steps.length; i++) {
const exponent = i - midIndex;
const px = basePx _ Math.pow(ratio, exponent);
const rem = Math.round((px / 16) _ 1000) / 1000;
scale[steps[i]] = rem + "rem";
}

return scale;
}

// Base: 16px, ratio: 2, steps: xs sm md lg xl
const result = spacingScale(16, 2, ["xs", "sm", "md", "lg", "xl"]);
console.log(result.xs === "0.25rem"); // 4px
console.log(result.sm === "0.5rem"); // 8px
console.log(result.md === "1rem"); // 16px
console.log(result.lg === "2rem"); // 32px
console.log(result.xl === "4rem"); // 64px`}

## What you learned

- A modern CSS reset (border-box, dvh, font inheritance, image max-width) creates a consistent baseline that prevents cross-browser inconsistencies
- CSS custom properties centralize design tokens (colors, spacing, radii) into a cascading system where overriding a variable themes an entire section
- Professional layouts combine Grid for page-level structure, Flexbox for component-level alignment, clamp() for fluid typography, and media queries for structural breakpoints
