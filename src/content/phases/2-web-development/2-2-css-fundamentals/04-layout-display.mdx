## Why this matters

Before you can use flexbox or grid, you need to understand how elements naturally arrange themselves. The `display` property is the foundation of every CSS layout. If you don't understand the difference between block and inline, you'll fight the browser instead of working with it.

## Normal document flow

Without any CSS layout rules, HTML elements stack according to **normal flow**. There are two fundamental behaviors:

**Block elements** start on a new line and stretch to fill their container's width. Think of them as paragraphs — each one gets its own line.

**Inline elements** flow within text, side by side. They only take up as much width as their content needs. Think of them as words within a sentence.

```html
<!-- Block elements: div, p, h1-h6, section, article, form -->
<div>I take the full width.</div>
<div>I start on a new line.</div>

<!-- Inline elements: span, a, strong, em, img -->
<p>This is <strong>bold</strong> and <a href="#">linked</a> text.</p>
```

## display: block

Block elements accept **all box model properties**: width, height, margin, and padding in every direction.

```css
.block-element {
  display: block;
  width: 400px; /* Respected */
  height: 200px; /* Respected */
  margin: 20px; /* All four sides work */
  padding: 16px; /* All four sides work */
}
```

By default, a block element's width is `auto` — it fills the available horizontal space. Setting an explicit width constrains it, and you can center it with `margin: 0 auto`.

## display: inline

Inline elements have significant restrictions. The browser ignores `width` and `height` on inline elements. Vertical margin and padding have no effect on surrounding layout (they visually render but don't push other elements away).

```css
.inline-element {
  display: inline;
  width: 200px; /* IGNORED */
  height: 100px; /* IGNORED */
  margin-top: 20px; /* IGNORED for layout */
  margin-left: 10px; /* Works — horizontal margin is fine */
  padding: 10px; /* Renders visually but doesn't affect flow vertically */
}
```

This is why you can't set a width on a `<span>` or `<a>` tag by default. They're inline.

## display: inline-block

`inline-block` gives you the best of both worlds: the element flows inline (side by side with siblings), but it **respects** width, height, and vertical margin/padding.

```css
.tag {
  display: inline-block;
  width: 80px; /* Respected */
  height: 32px; /* Respected */
  margin: 4px; /* All four sides work */
  padding: 4px 12px; /* All four sides work */
  text-align: center;
  background: #333;
  border-radius: 4px;
}
```

`inline-block` was heavily used before flexbox existed. Today, flexbox and grid handle most layout needs more cleanly, but `inline-block` is still useful for small, self-contained elements like tags and badges.

## display: none

`display: none` removes the element from the layout entirely. It takes up no space — as if the HTML element doesn't exist.

```css
.hidden {
  display: none;
}
```

This is different from `visibility: hidden`, which hides the element visually but **keeps its space** in the layout. And both differ from `opacity: 0`, which makes the element invisible but still interactive (it can receive clicks).

```css
/* Takes no space, removed from flow */
.removed {
  display: none;
}

/* Invisible but occupies space */
.invisible {
  visibility: hidden;
}

/* Invisible but occupies space AND is still clickable */
.transparent {
  opacity: 0;
}
```

Choose the right one based on your intent. For toggling UI elements, `display: none` is the most common choice.

## How display affects width and height

Here's a summary that clarifies the behavior:

| Property           | `block`                  | `inline`     | `inline-block`        |
| ------------------ | ------------------------ | ------------ | --------------------- |
| Width              | Full container (default) | Content only | Content (or explicit) |
| Height             | Content (or explicit)    | Content only | Content (or explicit) |
| `width` / `height` | Respected                | Ignored      | Respected             |
| Horizontal margin  | Respected                | Respected    | Respected             |
| Vertical margin    | Respected                | Ignored      | Respected             |
| Starts new line    | Yes                      | No           | No                    |

{`// Given a display type and CSS properties, determine
// which properties the browser will actually apply.

function effectiveProperties(displayType, properties) {
// Returns an object with only the properties that
// the browser respects for the given display type.
// Properties: width, height, marginTop, marginLeft, paddingTop, paddingLeft
// block: all respected
// inline: only marginLeft and paddingLeft (horizontal ones)
// inline-block: all respected
// none: returns empty object (element is removed)

if (displayType === "none") return {};
if (displayType === "block" || displayType === "inline-block") {
return { ...properties };
}
if (displayType === "inline") {
const result = {};
if ("marginLeft" in properties) result.marginLeft = properties.marginLeft;
if ("paddingLeft" in properties) result.paddingLeft = properties.paddingLeft;
return result;
}
return properties;
}

// Tests
const props = { width: 200, height: 100, marginTop: 10, marginLeft: 10, paddingTop: 8, paddingLeft: 8 };

const blockResult = effectiveProperties("block", props);
console.log(Object.keys(blockResult).length === 6);

const inlineResult = effectiveProperties("inline", props);
console.log(Object.keys(inlineResult).length === 2);
console.log(inlineResult.marginLeft === 10);

const noneResult = effectiveProperties("none", props);
console.log(Object.keys(noneResult).length === 0);`}

## What you learned

- Block elements take the full width and start on new lines; inline elements flow within text and ignore width/height
- `inline-block` combines inline flow with block-level box model control, making it useful for elements like tags and badges
- `display: none` removes an element from layout entirely, while `visibility: hidden` and `opacity: 0` hide it visually but preserve its space
