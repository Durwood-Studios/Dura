## Why this matters

Color and typography account for the majority of a user's visual experience. Choosing the wrong units or color format leads to inconsistent designs, poor accessibility (failing contrast ratios), and text that's unreadable on certain devices. These aren't aesthetic choices — they're engineering decisions.

## Color formats

CSS supports several color formats. Each has a use case.

**Hex** — the most common format. 6-digit (or 3-digit shorthand) hexadecimal:

```css
.element {
  color: #1a1a2e; /* Full hex */
  color: #333; /* Shorthand: same as #333333 */
  color: #1a1a2e80; /* 8-digit hex: last 2 digits = alpha */
}
```

**RGB** — red, green, blue channels (0-255). Useful when you need alpha transparency:

```css
.element {
  color: rgb(26, 26, 46);
  background: rgb(26 26 46 / 0.5); /* Modern syntax with alpha */
}
```

**HSL** — hue (0-360), saturation (%), lightness (%). Easier to reason about when creating color palettes because you can adjust lightness without changing the hue:

```css
.element {
  color: hsl(240, 28%, 14%);
  color: hsl(240 28% 14% / 0.8); /* With alpha */
}
```

**OKLCH** — the modern choice. Perceptually uniform, meaning equal lightness steps look equally bright to human eyes. Wider gamut support for modern displays:

```css
.element {
  color: oklch(0.5 0.15 240); /* Lightness, chroma, hue */
  color: oklch(0.5 0.15 240 / 0.8); /* With alpha */
}
```

For design systems, use CSS custom properties so you define colors once:

```css
:root {
  --color-primary: oklch(0.7 0.18 160);
  --color-text: #f0f0f0;
  --color-muted: #6b6b75;
}
```

## Typography fundamentals

### Font stacks

A **font stack** lists fonts in order of preference. The browser uses the first available one:

```css
body {
  font-family:
    "DM Sans",
    system-ui,
    -apple-system,
    sans-serif;
}

code {
  font-family: "JetBrains Mono", ui-monospace, "Cascadia Code", monospace;
}
```

**System font stacks** use the operating system's native font. They load instantly (no network request) and feel native to each platform:

```css
body {
  font-family:
    system-ui,
    -apple-system,
    "Segoe UI",
    Roboto,
    sans-serif;
}
```

To use **Google Fonts**, add a `<link>` in your HTML head or an `@import` in CSS. The `<link>` approach is faster because it starts downloading earlier:

```html
<link
  href="https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500;600;700&display=swap"
  rel="stylesheet"
/>
```

### Font properties

```css
.text {
  font-weight: 400; /* 100-900, or keywords: normal (400), bold (700) */
  line-height: 1.8; /* Unitless multiplier is best — scales with font-size */
  letter-spacing: -0.02em; /* Tighten for headings, leave default for body */
  font-style: italic;
}
```

Line height of 1.8 for body text ensures comfortable reading. Headings can use 1.2-1.3 since they're shorter.

## Sizing units: rem vs em vs px

This is where most beginners make mistakes. The unit you choose affects scalability and accessibility.

**px** — fixed pixels. Does not scale when the user changes their browser font size. Use sparingly for things that should never scale (borders, shadows):

```css
.border {
  border: 1px solid #ccc;
}
```

**rem** — relative to the root (`<html>`) font size. Default root is 16px, so `1rem = 16px`. This is your primary unit for font sizes, padding, and margins. It respects user accessibility settings:

```css
html {
  font-size: 16px;
} /* Usually the browser default */
h1 {
  font-size: 2rem;
} /* 32px */
p {
  font-size: 1rem;
} /* 16px */
.small {
  font-size: 0.875rem;
} /* 14px */
```

**em** — relative to the parent element's font size. Useful for component-internal spacing that should scale with the component's text size:

```css
.button {
  font-size: 1rem;
  padding: 0.5em 1em; /* Padding scales if font-size changes */
}
```

The rule: use `rem` for sizing and spacing. Use `em` only when you need something to scale relative to its own text size. Use `px` for borders and box shadows.

## What you learned

- CSS offers multiple color formats (hex, rgb, hsl, oklch), each with trade-offs — oklch is the modern choice for perceptual uniformity and wide gamut support
- Font stacks provide fallback fonts in priority order, and system font stacks load instantly with zero network cost
- Use `rem` for sizing and spacing (scales with user preferences), `em` for component-relative scaling, and `px` only for fixed details like borders
