## Why this matters

CSS Grid is purpose-built for two-dimensional layouts — rows AND columns at the same time. Flexbox handles rows or columns. Grid handles both. Dashboard layouts, magazine-style content, and application shells that were painful to build before Grid are now straightforward. It's the most powerful layout tool CSS has ever had.

## Defining a grid

Add `display: grid` to a container and define columns and rows:

```css
.grid {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: 60px 1fr;
  gap: 16px;
}
```

This creates a grid with 3 columns (one fixed at 200px, two sharing remaining space) and 2 rows (one fixed at 60px, one flexible).

### The fr unit

`fr` stands for **fraction of available space**. It distributes remaining space after fixed-size tracks are accounted for.

```css
.grid {
  display: grid;
  /* 1fr + 2fr = 3 parts. First column gets 1/3, second gets 2/3 */
  grid-template-columns: 1fr 2fr;
}
```

`fr` is to Grid what `flex-grow` is to Flexbox — it divides leftover space proportionally.

```css
.grid {
  display: grid;
  /* Sidebar: fixed 250px. Main content: remaining space */
  grid-template-columns: 250px 1fr;
  gap: 24px;
}
```

## Grid template areas

Named areas make complex layouts readable. Define a visual map of your layout using strings:

```css
.dashboard {
  display: grid;
  grid-template-columns: 250px 1fr 1fr;
  grid-template-rows: 64px 1fr 48px;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  gap: 16px;
  min-height: 100vh;
}

.header {
  grid-area: header;
}
.sidebar {
  grid-area: sidebar;
}
.main {
  grid-area: main;
}
.aside {
  grid-area: aside;
}
.footer {
  grid-area: footer;
}
```

```html
<div class="dashboard">
  <header class="header">Navigation</header>
  <nav class="sidebar">Links</nav>
  <main class="main">Content</main>
  <aside class="aside">Widgets</aside>
  <footer class="footer">Status</footer>
</div>
```

Each string in `grid-template-areas` represents one row. Repeating a name spans that area across multiple columns. Every row must have the same number of names.

## Placing items manually

You can place items by specifying which grid lines they span:

```css
.featured {
  grid-column: 1 / 3; /* Span from line 1 to line 3 (2 columns) */
  grid-row: 1 / 2; /* Occupy first row */
}

.wide {
  grid-column: span 2; /* Span 2 columns from wherever it's placed */
}
```

Grid lines are numbered starting at 1, not 0. A 3-column grid has 4 column lines (1, 2, 3, 4).

## Responsive grids with auto-fit and minmax

The most powerful Grid pattern creates responsive column layouts without media queries:

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 24px;
}
```

Breaking this down:

- `repeat()` creates multiple tracks
- `auto-fit` creates as many columns as will fit
- `minmax(280px, 1fr)` each column is at least 280px, but grows to fill remaining space

On a wide screen you get 4 columns. On a tablet, 2. On mobile, 1. No media queries needed.

**auto-fit vs auto-fill**: `auto-fit` collapses empty tracks (items stretch to fill). `auto-fill` keeps empty tracks (items stay at minimum size, leaving gaps). Use `auto-fit` in most cases.

## Building a dashboard layout

```css
.dashboard {
  display: grid;
  grid-template-columns: 250px 1fr;
  grid-template-rows: 64px 1fr;
  grid-template-areas:
    "nav header"
    "nav content";
  min-height: 100vh;
}

.nav {
  grid-area: nav;
}
.header {
  grid-area: header;
}
.content {
  grid-area: content;
}

/* Inside the content area, a responsive card grid */
.content-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 20px;
  padding: 24px;
}
```

Grid nests cleanly. The outer grid handles the page shell (sidebar + content area). The inner grid handles the card layout within the content area.

## Try it

<SandboxExercise language="javascript">
{`// Calculate column widths for a CSS grid with fr units.
// Given the container width, gap size, and an array of track definitions,
// compute the final width of each column.

function gridColumnWidths(containerWidth, gap, tracks) {
// tracks is an array like ["200px", "1fr", "2fr"]
// 1. Calculate total gap space: (tracks.length - 1) \* gap
// 2. Subtract fixed (px) widths and gap space from container
// 3. Distribute remaining space by fr ratios

const totalGap = (tracks.length - 1) \* gap;
let fixedSpace = 0;
let totalFr = 0;

const parsed = tracks.map(t => {
if (t.endsWith("px")) {
const px = parseInt(t);
fixedSpace += px;
return { type: "px", value: px };
}
const fr = parseFloat(t);
totalFr += fr;
return { type: "fr", value: fr };
});

const availableForFr = containerWidth - fixedSpace - totalGap;

return parsed.map(t => {
if (t.type === "px") return t.value;
return (t.value / totalFr) \* availableForFr;
});
}

// Container: 1000px, gap: 20px, tracks: 200px 1fr 2fr
// Total gap: 2 \* 20 = 40. Available for fr: 1000 - 200 - 40 = 760
// 1fr = 760/3 ≈ 253.33, 2fr ≈ 506.67
const result = gridColumnWidths(1000, 20, ["200px", "1fr", "2fr"]);
console.log(result[0] === 200);
console.log(Math.round(result[1]) === 253);
console.log(Math.round(result[2]) === 507);`}

## What you learned

- CSS Grid creates two-dimensional layouts with `grid-template-columns` and `grid-template-rows`, using `fr` units to distribute space proportionally
- `grid-template-areas` assigns named regions that make complex layouts readable and maintainable
- `repeat(auto-fit, minmax())` creates fully responsive column layouts that adapt to container width without media queries
