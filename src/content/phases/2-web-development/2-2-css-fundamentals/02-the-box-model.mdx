## Why this matters

Every element on a web page is a rectangular box. If you don't understand how those boxes calculate their size, you'll spend hours wondering why your layout is 20 pixels too wide or why elements overlap. The box model is the single most important mental model in CSS layout.

## The four layers

Every CSS box has four layers, from inside out:

1. **Content** — the text, image, or child elements
2. **Padding** — space between the content and the border
3. **Border** — a visible (or invisible) edge around the padding
4. **Margin** — space between this element and its neighbors

```css
.card {
  /* Content width and height */
  width: 300px;
  height: 200px;

  /* Padding: space inside the border */
  padding: 20px;

  /* Border: the visible edge */
  border: 2px solid #333;

  /* Margin: space outside the border */
  margin: 16px;
}
```

With the default `box-sizing: content-box`, the total width of this element is: 300 (content) + 20 + 20 (padding left/right) + 2 + 2 (border left/right) = **344px**. The margin adds space outside that but doesn't count toward the element's rendered width.

## box-sizing: border-box

The default `content-box` model is counterintuitive. When you set `width: 300px`, you probably want the element to be 300 pixels wide — not 300 plus padding and border. That's what `border-box` does.

```css
/* Apply to everything — this should be in every project */
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

With `border-box`, padding and border are included **inside** the declared width. Set `width: 300px` and the element is 300px wide, period. The content area shrinks to accommodate padding and border.

```css
.card {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid #333;
  /* Total width: 300px (content area is 256px) */
}
```

There is no good reason to use `content-box` in modern CSS. Every CSS reset and framework sets `border-box` globally. Do this at the top of every project.

## Shorthand properties

Padding and margin accept 1, 2, 3, or 4 values:

```css
.example {
  /* All four sides: 20px */
  padding: 20px;

  /* Vertical (top/bottom) 10px, Horizontal (left/right) 20px */
  padding: 10px 20px;

  /* Top 10px, Horizontal 20px, Bottom 30px */
  padding: 10px 20px 30px;

  /* Top 10px, Right 20px, Bottom 30px, Left 40px (clockwise) */
  padding: 10px 20px 30px 40px;
}
```

You can also target individual sides:

```css
.element {
  margin-top: 0;
  margin-bottom: 24px;
  padding-left: 16px;
  padding-right: 16px;
}
```

## Margin collapse

Vertical margins between adjacent elements **collapse** — only the larger margin is used. This catches every beginner off guard.

```css
.heading {
  margin-bottom: 20px;
}
.paragraph {
  margin-top: 30px;
}
/* The gap between them is 30px, not 50px */
```

Horizontal margins never collapse. Margins inside a flex or grid container don't collapse either. When you need precise spacing, prefer `padding` on a container or `gap` in flexbox/grid.

## Inspecting the box model

Every browser's DevTools has a box model diagram. In Chrome: right-click an element, choose "Inspect", and look at the "Computed" tab. You'll see the exact content, padding, border, and margin dimensions. This is the fastest way to debug layout issues.

Hover over any element in the Elements panel and the browser overlays the box model directly on the page — content in blue, padding in green, border in yellow, margin in orange.

## What you learned

- Every element is a box with four layers: content, padding, border, and margin — and their sizes combine to determine total rendered dimensions
- Setting `box-sizing: border-box` globally ensures declared widths include padding and border, eliminating the most common source of layout math errors
- Vertical margins collapse between adjacent elements, so the gap is the larger margin, not the sum
