## Why this matters

Forms are how users give your app data — login credentials, search queries, settings, messages. In vanilla HTML, the browser owns the form values. In React, you want React state to own them. This unlocks instant validation, dynamic UI based on what the user typed, and reliable access to the current value without querying the DOM. The "controlled input" pattern is the foundation of every production React form.

## Controlled vs uncontrolled inputs

An **uncontrolled input** works like a plain HTML input — the browser stores the value, and you read it when you need it (usually on submit) using a `ref`:

```jsx
function UncontrolledForm() {
  const inputRef = useRef(null);

  function handleSubmit(e) {
    e.preventDefault();
    console.log(inputRef.current.value); // read DOM directly
  }

  return (
    <form onSubmit={handleSubmit}>
      <input ref={inputRef} />
      <button type="submit">Submit</button>
    </form>
  );
}
```

A **controlled input** wires the input's value to React state and updates state on every keystroke:

```jsx
function ControlledForm() {
  const [name, setName] = useState("");

  return <input value={name} onChange={(e) => setName(e.target.value)} />;
}
```

The difference: with controlled inputs, `name` in state is always exactly what the input shows. React is the single source of truth. You can read the value at any time, validate it on every keystroke, or transform it (like forcing uppercase) before storing it.

## The onChange pattern

Every input type has a slightly different `onChange` pattern:

```jsx
// Text input — value is e.target.value
<input
  type="text"
  value={name}
  onChange={(e) => setName(e.target.value)}
/>

// Select — same as text
<select value={size} onChange={(e) => setSize(e.target.value)}>
  <option value="sm">Small</option>
  <option value="md">Medium</option>
  <option value="lg">Large</option>
</select>

// Checkbox — value is e.target.checked (boolean)
<input
  type="checkbox"
  checked={isAgreed}
  onChange={(e) => setIsAgreed(e.target.checked)}
/>
```

## Handling form submission

Always prevent the browser's default form submission behavior, then read your state values:

```jsx
function LoginForm() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");

  function handleSubmit(e) {
    e.preventDefault(); // Stop browser navigation

    // Values are already in state — no DOM querying needed
    console.log({ email, password });
    loginUser(email, password);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input type="email" value={email} onChange={(e) => setEmail(e.target.value)} />
      <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} />
      <button type="submit">Log in</button>
    </form>
  );
}
```

## Multi-field forms with a single state object

For forms with many fields, managing one state value per field becomes unwieldy. Use a single state object instead:

```jsx
function ProfileForm() {
  const [form, setForm] = useState({
    firstName: "",
    lastName: "",
    email: "",
    bio: "",
  });

  // Generic handler — uses the input's name attribute as the key
  function handleChange(e) {
    const { name, value } = e.target;
    setForm((prev) => ({ ...prev, [name]: value }));
  }

  return (
    <form>
      <input name="firstName" value={form.firstName} onChange={handleChange} />
      <input name="lastName" value={form.lastName} onChange={handleChange} />
      <input name="email" value={form.email} onChange={handleChange} />
      <textarea name="bio" value={form.bio} onChange={handleChange} />
    </form>
  );
}
```

The `[name]: value` syntax is computed property key — it uses the input's `name` attribute to know which field to update. One handler for every field.

## Form reset

Resetting a controlled form is trivial — just set state back to the initial values:

```jsx
function handleReset() {
  setForm({ firstName: "", lastName: "", email: "", bio: "" });
}
```

With uncontrolled forms, you would need to imperatively clear each input via refs.

## Basic validation

Controlled inputs make validation straightforward. You can validate on submit, on blur (when the user leaves a field), or on every keystroke:

```jsx
function EmailInput() {
  const [email, setEmail] = useState("");
  const [error, setError] = useState("");

  function handleBlur() {
    if (!email.includes("@")) {
      setError("Please enter a valid email address.");
    } else {
      setError("");
    }
  }

  return (
    <div>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        onBlur={handleBlur}
      />
      {error && <p className="error">{error}</p>}
    </div>
  );
}
```

For production forms with complex validation, a library like `react-hook-form` or `zod` is worth considering — but understanding the controlled pattern is the prerequisite.

{`// Build a pure JS form state manager — the logic behind every React form.
// createFormState(initialValues, validators) returns an object with:
// - setValue(field, value): update a field
// - getValues(): return all current values
// - validate(): return { valid: boolean, errors: { fieldName: errorMessage } }
// - reset(): restore initial values

function createFormState(initialValues, validators = {}) {
let values = { ...initialValues };

function setValue(field, value) {
values = { ...values, [field]: value };
}

function getValues() {
return { ...values };
}

function validate() {
const errors = {};

    for (const [field, validateFn] of Object.entries(validators)) {
      const error = validateFn(values[field]);
      if (error) errors[field] = error;
    }

    return {
      valid: Object.keys(errors).length === 0,
      errors,
    };

}

function reset() {
values = { ...initialValues };
}

return { setValue, getValues, validate, reset };
}

// Test it
const form = createFormState(
// Initial values
{ email: "", password: "", agree: false },

// Validators — return error string or null
{

!val.includes("@") ? "Must be a valid email" : null,
password: (val) =>

);

## What you learned

- **Controlled inputs** wire `value` to state and `onChange` to the setter — React owns the value, enabling validation and dynamic UI on every keystroke.
- **Uncontrolled inputs** let the browser own the value; you read it via a `ref` — useful for simple cases but harder to validate.
- The `onChange` handler uses `e.target.value` for text/select and `e.target.checked` for checkboxes.
- Multi-field forms stay clean with a single state object and a generic `handleChange` that uses `e.target.name` as the key.
- **Resetting** a controlled form is a single state update back to initial values.
