## Why this matters

JSX is what makes React code look like HTML inside JavaScript. It is the syntax you will write every single day as a React developer. Understanding what JSX actually compiles to — and how components receive and pass data — prevents an entire class of bugs and makes the "magic" feel logical.

## What JSX really is

JSX is not HTML. It is **syntax sugar for function calls**. When your build tool processes JSX, it transforms every tag into a call to `React.createElement`:

```jsx
// What you write:
const element = <h1 className="title">Hello, world!</h1>;

// What it compiles to:
const element = React.createElement("h1", { className: "title" }, "Hello, world!");
```

`React.createElement` returns a plain JavaScript object — a description of a DOM node. React's renderer reads these objects and builds the real DOM. This is why the virtual DOM is "just JavaScript."

## JSX rules

JSX looks like HTML but has important differences.

**Use `className` instead of `class`** — `class` is a reserved word in JavaScript:

```jsx
// Wrong
<div class="card">...</div>

// Correct
<div className="card">...</div>
```

**Self-closing tags must close** — in HTML, `<img>` is valid. In JSX, you must write `<img />`:

```jsx
<img src="photo.jpg" alt="A photo" />
<input type="text" />
<br />
```

**One root element** — a component's return value must be a single element. Wrap siblings in a `<div>` or a **Fragment** (`<>...</>`). Fragments add no DOM node:

```jsx
// Wrong — two root elements
return (
  <h1>Title</h1>
  <p>Content</p>
);

// Correct — fragment wrapper, no extra DOM node
return (
  <>
    <h1>Title</h1>
    <p>Content</p>
  </>
);
```

**Expressions go in curly braces** — any JavaScript expression can appear inside JSX:

```jsx
const name = "Dustin";
return <p>Hello, {name}!</p>;
return <p>2 + 2 = {2 + 2}</p>;
return <p>{isLoggedIn ? "Welcome back" : "Please sign in"}</p>;
```

## Function components

A React component is a **function that returns JSX**. The function name must start with a capital letter (this is how React distinguishes components from HTML elements):

```jsx
function Greeting() {
  return <h1>Hello, world!</h1>;
}
```

That is a complete, valid React component. Capital letter, returns JSX. Nothing more is required.

## Props — a component's inputs

Components are more useful when they can receive data. **Props** are the arguments you pass to a component, just like arguments to a function. They arrive as a single object parameter:

```jsx
function Greeting({ name, role }) {
  return (
    <div className="greeting">
      <h1>Hello, {name}!</h1>
      <p>Role: {role}</p>
    </div>
  );
}

// Usage:
<Greeting name="Dustin" role="admin" />;
```

Props are **read-only**. A component must never modify its own props. If a value needs to change, that is what state is for (next lesson).

## The children prop

When you put content between a component's opening and closing tags, it becomes the **`children` prop**:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

// Usage — anything between the tags becomes children:
<Card>
  <h2>My Title</h2>
  <p>Some content here.</p>
</Card>;
```

This is how React does composition — the slot model. `Card` does not need to know what goes inside it. Any JSX can be slotted in as `children`.

## Composing components

Components use other components. You build complex UIs by assembling small pieces:

```jsx
function Avatar({ src, name }) {
  return <img src={src} alt={name} className="avatar" />;
}

function UserCard({ user }) {
  return (
    <div className="user-card">
      <Avatar src={user.avatarUrl} name={user.name} />
      <h2>{user.name}</h2>
      <p>{user.bio}</p>
    </div>
  );
}

function UserList({ users }) {
  return (
    <ul>
      {users.map((user) => (
        <UserCard key={user.id} user={user} />
      ))}
    </ul>
  );
}
```

Each component owns its own piece of the UI. Changes to `Avatar` do not require touching `UserCard` or `UserList`. This isolation is the core value of the component model.

{`// Simulate React.createElement and a basic render function.
// createElement returns a plain JS object describing the UI.
// render converts that object to an HTML string.

function createElement(type, props, ...children) {
// Return an object with type, props (including children)
// If there's only one child, children should not be an array
const resolvedChildren = children.length === 1 ? children[0] : children;
return {
type,
props: {
...props,
children: resolvedChildren,
},
};
}

function render(element) {
// If element is a string or number, return it as-is (it's a text node)
if (typeof element === "string" || typeof element === "number") {
return String(element);
}

const { type, props } = element;
const { children, ...attrs } = props || {};

// Build the attribute string (e.g., class="card" id="main")
const attrString = Object.entries(attrs)
.map(([key, val]) => {
// Convert camelCase props to HTML attributes
const htmlKey = key === "className" ? "class" : key;
return \`\${htmlKey}="\${val}"\`;
})
.join(" ");

const openTag = attrString ? \`<\${type} \${attrString}>\` : \`<\${type}>\`;

// Render children recursively
const childrenArray = Array.isArray(children) ? children : [children].filter(Boolean);
const childrenHtml = childrenArray.map(render).join("");

return \`\${openTag}\${childrenHtml}</\${type}>\`;
}

// Test it — build a component tree using createElement
const element = createElement(
"div",
{ className: "card" },
createElement("h1", null, "Hello, React!"),
createElement("p", { className: "subtitle" }, "Built with createElement")
);

console.log("Element object:");
console.log(JSON.stringify(element, null, 2));

console.log("\\nRendered HTML:");
console.log(render(element));`}

## What you learned

- JSX compiles to `React.createElement` calls — it is JavaScript, not HTML, which is why rules like `className` and self-closing tags apply.
- **Fragments** (`<>...</>`) let you return multiple elements without adding DOM nodes.
- **Props** are read-only inputs to a component, passed as attributes in JSX and received as a single object parameter.
- The **`children` prop** is whatever JSX you place between a component's opening and closing tags — the foundation of composition.
