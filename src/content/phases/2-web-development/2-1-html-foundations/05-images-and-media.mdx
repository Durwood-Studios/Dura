## Why this matters

The web is not just text. Images, video, and audio make content engaging — but only when embedded correctly. A missing `alt` attribute means a screen reader user gets nothing. A video without controls means a mobile user cannot pause it. Media done right is inclusive. Media done wrong excludes people.

## Images

The `<img>` tag embeds an image. It is self-closing — no content goes between tags.

```html
<img src="photo.jpg" alt="A laptop on a wooden desk" width="800" height="600" />
```

Required attributes:

- **`src`** — the path to the image file (relative or absolute).
- **`alt`** — a text description of the image. This is not optional.

Optional but recommended:

- **`width`** and **`height`** — prevent layout shift while the image loads. The browser reserves space before the image downloads.
- **`loading="lazy"`** — delays loading until the image is near the viewport. Saves bandwidth.

```html
<img
  src="/images/hero.webp"
  alt="Student writing code on a laptop"
  width="1200"
  height="800"
  loading="lazy"
/>
```

## Alt text that works

Alt text is not a caption. It is a replacement for the image when the image cannot be seen — by a blind user, a slow connection, or a broken link.

Good alt text is **specific and functional**:

- Bad: `alt="image"` — useless.
- Bad: `alt="photo of a thing"` — vague.
- Good: `alt="Bar chart showing 40% growth in Q3"` — descriptive.
- Decorative image: `alt=""` — an empty alt tells screen readers to skip it entirely.

Never omit the `alt` attribute. An empty `alt=""` is intentional silence. A missing `alt` forces the screen reader to announce the file name, which is almost never helpful.

## Video

The `<video>` tag embeds video with built-in browser controls.

```html
<video controls width="640" height="360">
  <source src="lesson.mp4" type="video/mp4" />
  <source src="lesson.webm" type="video/webm" />
  Your browser does not support the video tag.
</video>
```

- **`controls`** — shows play, pause, volume, and fullscreen buttons. Always include this.
- **`<source>`** — multiple formats let the browser pick the best one it supports.
- The text between the tags is a **fallback** for browsers that do not support `<video>`.

## Audio

Audio works the same way:

```html
<audio controls>
  <source src="podcast.mp3" type="audio/mpeg" />
  <source src="podcast.ogg" type="audio/ogg" />
  Your browser does not support the audio tag.
</audio>
```

Avoid `autoplay` on both video and audio. It is disruptive, wastes bandwidth, and many browsers block it by default.

## Responsive images

The `srcset` attribute lets the browser choose the best image size for the device:

```html
<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, 800px"
  alt="Workshop participants collaborating"
/>
```

The browser picks the smallest image that fits. This means faster loads on mobile and sharp images on desktop — without you serving a 2MB file to a phone.

## What you learned

- The `<img>` tag requires both `src` and `alt` — alt text is a **replacement**, not a caption.
- `<video>` and `<audio>` use `<source>` elements for format fallbacks and should always include **controls**.
- `loading="lazy"` and `srcset` optimize media delivery for different devices and connection speeds.
