## Why this matters

Data has structure. A list of ingredients is not the same as a table of prices. HTML gives you distinct elements for each because the structure communicates meaning to browsers, screen readers, and search engines. Using the wrong element — like a table for layout or a paragraph for a list — hides that meaning.

## Unordered lists

Use `<ul>` when the order does not matter. Navigation menus, feature lists, and tags are all unordered.

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

The browser renders bullet points by default. CSS can change the appearance, but the semantic meaning stays: these items are a group with no inherent sequence.

## Ordered lists

Use `<ol>` when sequence matters. Steps in a process, rankings, or instructions.

```html
<ol>
  <li>Open your editor</li>
  <li>Create index.html</li>
  <li>Write your first tag</li>
</ol>
```

The browser numbers these automatically. You can change the starting number with the `start` attribute or reverse the order with `reversed`:

```html
<ol start="5" reversed>
  <li>Fifth item (displayed first)</li>
  <li>Fourth item</li>
</ol>
```

## Nesting lists

Lists nest inside each other. Each nested list goes inside an `<li>`:

```html
<ul>
  <li>
    Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>
```

Screen readers announce the nesting level, so users know they have entered a sub-list. This structure is how accessible navigation menus with dropdowns work.

## Tables

Tables display **tabular data** — rows and columns of related information. Never use tables for page layout.

```html
<table>
  <thead>
    <tr>
      <th>Language</th>
      <th>Year Created</th>
      <th>Creator</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>HTML</td>
      <td>1993</td>
      <td>Tim Berners-Lee</td>
    </tr>
    <tr>
      <td>JavaScript</td>
      <td>1995</td>
      <td>Brendan Eich</td>
    </tr>
  </tbody>
</table>
```

Key elements:

- **`<thead>`** — groups the header row. Screen readers use this to associate data cells with their column headers.
- **`<tbody>`** — groups the data rows.
- **`<th>`** — a header cell. By default bold and centered. More importantly, it tells assistive technology what each column means.
- **`<tr>`** — a table row.
- **`<td>`** — a data cell.

## When to use lists vs tables

**Use a list** when you have a collection of items. Even if they have details, if there is no column structure, use a list.

**Use a table** when your data has consistent attributes across items — like a spreadsheet. If every item has the same three properties, a table communicates that pattern clearly.

Using a table for layout is a relic of the 1990s. CSS Grid and Flexbox handle layout. Tables handle data.

## What you learned

- `<ul>` groups items with no inherent order; `<ol>` groups items where **sequence matters**.
- Tables require `<thead>`, `<tbody>`, `<th>`, and `<td>` to be **accessible and semantic**.
- Tables are for **data**, never for layout — CSS handles layout now.
