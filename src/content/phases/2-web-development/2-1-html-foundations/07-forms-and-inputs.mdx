## Why this matters

Every login screen, search bar, checkout form, and survey on the web is an HTML form. Forms are how users send data to a server. Understanding form elements, input types, and built-in validation means you can collect data correctly, accessibly, and securely — before writing a single line of JavaScript.

## The form element

A `<form>` wraps all the inputs that belong together. It has two key attributes:

```html
<form action="/submit" method="POST">
  <!-- inputs go here -->
</form>
```

- **`action`** — the URL where the form data is sent.
- **`method`** — `GET` appends data to the URL (for searches). `POST` sends data in the request body (for submissions with sensitive data).

## Labels and inputs

Every input needs a label. This is non-negotiable for accessibility.

```html
<label for="email">Email address</label> <input type="email" id="email" name="email" required />
```

The `for` attribute on the label must match the `id` on the input. This association does three things:

1. Screen readers announce the label when the input is focused.
2. Clicking the label focuses the input — a bigger click target on mobile.
3. Automated testing tools can find inputs by their label.

## Input types

HTML5 added input types that give you validation and mobile keyboard optimization for free:

```html
<input type="text" />
<!-- Generic text -->
<input type="email" />
<!-- Validates email format, shows @ keyboard on mobile -->
<input type="number" />
<!-- Numeric keyboard, min/max/step attributes -->
<input type="password" />
<!-- Hides characters -->
<input type="url" />
<!-- Validates URL format -->
<input type="tel" />
<!-- Phone keyboard on mobile -->
<input type="date" />
<!-- Native date picker -->
<input type="search" />
<!-- Styled as search, clear button on some browsers -->
```

Using the right type means the browser does validation work you would otherwise write in JavaScript.

## Checkboxes and radio buttons

Checkboxes allow multiple selections. Radio buttons allow exactly one within a group.

```html
<!-- Checkboxes: independent -->
<label><input type="checkbox" name="skills" value="html" /> HTML</label>
<label><input type="checkbox" name="skills" value="css" /> CSS</label>

<!-- Radio buttons: grouped by name -->
<label><input type="radio" name="level" value="beginner" /> Beginner</label>
<label><input type="radio" name="level" value="advanced" /> Advanced</label>
```

Radio buttons with the same `name` attribute form a group. Selecting one deselects the others. Wrapping the input inside the label eliminates the need for `for` and `id` — the association is implicit.

## Built-in validation

HTML attributes validate input before any JavaScript runs:

```html
<input type="text" required minlength="3" maxlength="100" />
<input type="number" min="1" max="10" step="1" />
<input type="email" required />
<input type="text" pattern="[A-Za-z]{3,}" title="At least 3 letters" />
```

- **`required`** — the form will not submit if this field is empty.
- **`minlength` / `maxlength`** — character count constraints.
- **`min` / `max`** — numeric range constraints.
- **`pattern`** — a regex the value must match.

The browser shows native error messages. These are accessible, localized, and require zero JavaScript.

## Submitting a form

```html
<form action="/signup" method="POST">
  <label for="name">Name</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required />

  <button type="submit">Sign Up</button>
</form>
```

Use `<button type="submit">` — not `<input type="submit">`. The `<button>` element is more flexible (it can contain icons, spans, and other HTML) and is easier to style.

## What you learned

- Every form input **must** have a `<label>` with a matching `for`/`id` pair for accessibility.
- HTML5 input types like `email`, `number`, and `date` provide **free validation and mobile keyboards**.
- Built-in attributes like `required`, `pattern`, and `min`/`max` validate data **before JavaScript runs**.
