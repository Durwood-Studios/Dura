## Why this matters

Links are the connective tissue of the web. Without the `<a>` tag, every page would be a dead end. Understanding how links work — where they point, how they open, and what makes a good link — is foundational to building anything on the web.

## The anchor element

The `<a>` tag creates a hyperlink. The `href` attribute tells the browser where to go.

```html
<a href="https://dura.dev">Visit DURA</a>
```

The text between the tags is the **link text** — what the user sees and clicks. Screen readers announce it, so "click here" is never acceptable. Write link text that makes sense out of context: "Visit DURA" tells you where you are going. "Click here" tells you nothing.

## Absolute vs relative paths

**Absolute paths** include the full URL — protocol, domain, and path:

```html
<a href="https://dura.dev/lessons/html">HTML Lessons</a>
```

**Relative paths** are relative to the current page's location:

```html
<!-- Same directory -->
<a href="about.html">About</a>

<!-- One directory up -->
<a href="../index.html">Home</a>

<!-- From the root of the site -->
<a href="/contact">Contact</a>
```

Use absolute paths for external links. Use relative paths for internal navigation — they survive domain changes and work in development without configuration.

## The target attribute

By default, links open in the same tab. The `target` attribute changes this.

```html
<!-- Opens in a new tab -->
<a href="https://github.com" target="_blank" rel="noopener noreferrer">GitHub</a>
```

When using `target="_blank"`, always add `rel="noopener noreferrer"`. Without it, the new page can access your page's `window.opener` object — a security risk called **tabnapping**. Modern browsers mitigate this, but the `rel` attribute is still best practice.

## Fragment links

Links can point to a specific section on the same page using an `id`:

```html
<a href="#contact">Jump to Contact</a>

<!-- Elsewhere on the page -->
<section id="contact">
  <h2>Contact Us</h2>
</section>
```

This is how "skip to content" links and table-of-contents navigation work. The `#` tells the browser to scroll to the element with that `id` instead of loading a new page.

## Email and telephone links

The `href` attribute supports more than HTTP:

```html
<a href="mailto:hello@dura.dev">Send Email</a> <a href="tel:+15551234567">Call Us</a>
```

These trigger the user's default email client or phone app. On mobile, `tel:` links are especially useful.

## What you learned

- The `<a>` tag with `href` creates hyperlinks — the web's core navigation mechanism.
- **Relative paths** navigate within your site; **absolute paths** link to external resources.
- Always add `rel="noopener noreferrer"` when using `target="_blank"` to prevent security vulnerabilities.
