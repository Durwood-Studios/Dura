## Why this matters

Search engines, screen readers, and browsers all rely on heading levels to understand your page. A visually large font means nothing to a machine — but an `<h1>` tells every tool exactly what your page is about. Getting text hierarchy right is the difference between a page that works and a page that just looks like it works.

## Headings create structure

HTML gives you six heading levels: `<h1>` through `<h6>`. They form a hierarchy, not a styling choice.

```html
<h1>Web Development Fundamentals</h1>
<h2>HTML Basics</h2>
<h3>Text Elements</h3>
<h3>Link Elements</h3>
<h2>CSS Basics</h2>
```

Rules that matter:

- **One `<h1>` per page.** It is the page title. Search engines use it as a primary signal.
- **Never skip levels.** Going from `<h2>` to `<h4>` confuses assistive technology. If you need smaller text, use CSS — not a lower heading.
- **Headings are not for bold text.** If you want bold, use `<strong>`. Headings announce sections.

## Paragraphs and line breaks

The `<p>` element wraps a block of text. Browsers add space above and below paragraphs automatically.

```html
<p>This is the first paragraph. It can be as long as you need.</p>
<p>This is a second paragraph. The browser separates them visually.</p>
```

Need a line break without starting a new paragraph? Use `<br />`. But reach for it rarely — most of the time, a new `<p>` is more semantic.

## Inline text elements

These elements mark up text inside a paragraph without breaking the flow.

```html
<p>Use <strong>strong</strong> for important text.</p>
<p>Use <em>em</em> for emphasis or a shift in tone.</p>
<p>Use <span>span</span> when you need a hook for styling but no semantic meaning.</p>
```

`<strong>` vs `<b>`: both render bold, but `<strong>` carries semantic weight — screen readers may announce it differently. Use `<strong>` for meaning, `<b>` only for visual styling with no importance attached.

`<em>` vs `<i>`: same distinction. `<em>` means emphasis. `<i>` means alternate voice (like a foreign word or technical term). When in doubt, use `<em>`.

## Combining elements

Text elements nest naturally inside headings and paragraphs:

```html
<h2>Getting Started with <em>Real</em> HTML</h2>
<p>
  The <strong>most common mistake</strong> beginners make is using
  <span class="highlight">divs for everything</span> instead of choosing the right text element.
</p>
```

Every element you pick communicates meaning to the browser. A `<span>` says "I have no meaning — style me." A `<strong>` says "this part is important." Choose deliberately.

## What you learned

- Headings `<h1>` through `<h6>` define **document hierarchy**, not font size.
- `<strong>` and `<em>` carry **semantic meaning** that assistive tools use.
- `<span>` is a **generic inline container** with no meaning — use it only for styling hooks.
