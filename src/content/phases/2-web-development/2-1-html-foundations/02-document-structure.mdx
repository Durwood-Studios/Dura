## Why this matters

Browsers are forgiving — they will render broken HTML as best they can. But "works" is not the same as "correct." A properly structured document loads faster, is accessible, and behaves consistently across browsers.

## The skeleton

Every HTML document has the same shape:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Page Title</title>
  </head>
  <body>
    <!-- visible content goes here -->
  </body>
</html>
```

## DOCTYPE

`<!DOCTYPE html>` tells the browser "this is HTML5." Without it, browsers enter **quirks mode** — an older rendering mode that can break modern CSS. Always include it. It is not an HTML tag; it is a processing instruction.

## The html element

`<html lang="en">` is the root element. The `lang` attribute tells screen readers and search engines what language the content is in. Set it to the primary language of your page.

## The head

The `<head>` contains metadata — information about the page that is not displayed directly.

- `<meta charset="UTF-8">` — character encoding. Always UTF-8.
- `<meta name="viewport" ...>` — tells mobile browsers how to scale the page.
- `<title>` — the text in the browser tab and search results.
- `<link>` — stylesheets, favicons.
- `<script>` — JavaScript (or a reference to a JS file).

## The body

Everything visible lives in `<body>`. Text, images, forms, buttons — all of it.

## Comments

```html
<!-- This is a comment. The browser ignores it. -->
```

Comments are for developers. Use them to explain why, not what.

## What you learned

- Every HTML document needs `<!DOCTYPE html>`, `<html>`, `<head>`, and `<body>`.
- The `<head>` holds metadata (charset, viewport, title). The `<body>` holds content.
- `lang` on `<html>` sets the page language for accessibility and SEO.
