## Why this matters

A page built entirely with `<div>` and `<span>` is like a book with no chapters, no table of contents, and no page numbers. It renders fine visually, but machines cannot understand its structure. Screen readers, search engines, and browser features like Reader Mode all depend on semantic HTML to know what your content means — not just how it looks.

## The problem with div soup

```html
<!-- This tells a machine nothing -->
<div class="top-bar">
  <div class="logo">DURA</div>
  <div class="links">
    <div class="link">Home</div>
    <div class="link">Lessons</div>
  </div>
</div>
<div class="content">
  <div class="post">
    <div class="post-title">Learning HTML</div>
    <div class="post-body">HTML is the foundation...</div>
  </div>
</div>
<div class="bottom">
  <div class="copyright">2026 Durwood Studios</div>
</div>
```

A screen reader sees a flat list of generic containers. There is no way to jump to the main content, identify the navigation, or find the article. Every `<div>` says "I am a division with no specific meaning."

## Semantic elements

HTML5 introduced elements that describe their purpose:

```html
<header>
  <h1>DURA</h1>
  <nav>
    <a href="/">Home</a>
    <a href="/lessons">Lessons</a>
  </nav>
</header>
<main>
  <article>
    <h2>Learning HTML</h2>
    <p>HTML is the foundation...</p>
  </article>
</main>
<footer>
  <p>2026 Durwood Studios</p>
</footer>
```

Same visual result. Completely different meaning to machines.

## The elements and when to use them

**`<header>`** — introductory content for its parent. Usually contains a heading, logo, or navigation. A page can have multiple headers (one per section), but most pages have one at the top.

**`<nav>`** — a group of navigation links. Use it for primary navigation, not every group of links. A footer with three links does not need `<nav>`.

**`<main>`** — the primary content of the page. There must be **exactly one** `<main>` per page. Skip links target this element. Content inside `<main>` is what Reader Mode extracts.

**`<section>`** — a thematic grouping of content, usually with a heading. Use it when content belongs together conceptually but is not a standalone piece.

**`<article>`** — self-contained content that makes sense on its own. Blog posts, news stories, lesson cards, and comments are all articles. The test: would this content make sense syndicated in an RSS feed?

**`<footer>`** — closing content for its parent. Copyright notices, related links, and contact information.

**`<aside>`** — tangentially related content. Sidebars, callout boxes, and supplementary info.

## Choosing the right element

Ask these questions in order:

1. Is it navigation? Use `<nav>`.
2. Is it the primary page content? Use `<main>`.
3. Is it self-contained and reusable? Use `<article>`.
4. Is it a thematic group with a heading? Use `<section>`.
5. Is it supplementary or tangential? Use `<aside>`.
6. Is it introductory content? Use `<header>`.
7. Is it closing content? Use `<footer>`.
8. None of the above? **Now** use `<div>`.

The `<div>` is not bad — it is a last resort. It means "I need a container for styling, but this content has no special semantic role."

## A complete page structure

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>DURA - HTML Foundations</title>
  </head>
  <body>
    <header>
      <h1>DURA</h1>
      <nav>
        <a href="/">Home</a>
        <a href="/phases">Phases</a>
      </nav>
    </header>
    <main>
      <article>
        <h2>Semantic HTML</h2>
        <section>
          <h3>Why It Matters</h3>
          <p>Machines need structure...</p>
        </section>
      </article>
      <aside>
        <h3>Related Lessons</h3>
        <ul>
          <li><a href="/lessons/accessibility">Accessibility</a></li>
        </ul>
      </aside>
    </main>
    <footer>
      <p>2026 Durwood Studios LLC</p>
    </footer>
  </body>
</html>
```

## What you learned

- Semantic elements like `<header>`, `<nav>`, `<main>`, and `<article>` tell machines **what content means**.
- `<div>` is a last resort — use it only when no semantic element fits.
- A page needs exactly **one `<main>`** element, which screen readers and Reader Mode use to find the primary content.
