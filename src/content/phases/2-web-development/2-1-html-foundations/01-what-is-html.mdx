## Why this matters

Every webpage you have ever seen is HTML underneath. Buttons, headings, images, forms — they are all HTML elements rendered by a browser. If JavaScript is the brain and CSS is the clothes, HTML is the skeleton. Without it, nothing stands up.

## Tags and elements

HTML is made of **tags** — keywords wrapped in angle brackets.

```html
<p>This is a paragraph.</p>
```

`<p>` is the **opening tag**. `</p>` is the **closing tag**. Together with the content between them, they form an **element**. Most HTML elements have an opening tag, content, and a closing tag.

Some elements are **self-closing** — they have no content and no closing tag:

```html
<br />
<img src="photo.jpg" alt="A photo" />
```

## Attributes

Attributes add information to an element. They sit inside the opening tag.

```html
<a href="https://dura.dev" target="_blank">Visit DURA</a>
```

- `href` tells the link where to go.
- `target="_blank"` opens it in a new tab.

Attributes always follow the pattern `name="value"`. The order does not matter, but the value must be in quotes.

## Nesting

Elements go inside other elements. This creates a tree — the structure the browser parses into the DOM.

```html
<div>
  <h1>Hello</h1>
  <p>Welcome to the <strong>web</strong>.</p>
</div>
```

Proper nesting means every tag that opens inside another element closes inside it too. Overlapping tags (`<b><i></b></i>`) are invalid.

## Your first page

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is a webpage.</p>
  </body>
</html>
```

Save this as `index.html`, open it in a browser, and you have a webpage. That is all it takes.

## What you learned

- HTML is made of **elements**: an opening tag, content, and a closing tag.
- **Attributes** add information inside the opening tag (`name="value"`).
- Elements **nest** inside each other to form a tree structure the browser renders.
