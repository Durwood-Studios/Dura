## Why this matters

The browser is not just a DOM renderer. It's a platform with APIs for navigation, observing scroll, reading device location, accessing the clipboard, sharing content, and much more. Knowing what the browser can do natively — before reaching for a library — makes your apps smaller, faster, and more capable. It also means understanding one of the most important skills in browser development: feature detection.

## The History API

Single-page applications navigate between views without full page reloads. The History API is what makes URL changes possible without the browser sending a new HTTP request.

```javascript
// Change the URL and push a new entry onto the browser history stack
history.pushState({ page: "home" }, "", "/home");

// Change the URL without adding to the history stack (replace current entry)
history.replaceState({ page: "home" }, "", "/home");

// Navigate backward/forward (same as browser buttons)
history.back();
history.forward();
history.go(-2); // Go back 2 entries
```

The first argument is **state** — a plain object stored alongside the URL entry, retrievable when the user navigates back. The third argument is the new URL (must be same origin).

Listen for the user pressing the back/forward button:

```javascript
window.addEventListener("popstate", (event) => {
  // event.state is the state object from pushState/replaceState
  navigateTo(event.state?.page ?? "home");
});
```

This is the foundation of every client-side router — React Router, Vue Router, and Next.js's router all use `pushState` internally.

## Intersection Observer

`IntersectionObserver` tells you when an element enters or leaves the viewport — without scroll event polling. It's the modern way to implement lazy loading, scroll animations, and infinite scroll.

```javascript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        // Element is visible in the viewport
        entry.target.classList.add("visible");
        observer.unobserve(entry.target); // Stop watching after it animates in
      }
    });
  },
  {
    threshold: 0.1, // Trigger when 10% of the element is visible
    rootMargin: "0px 0px -50px 0px", // Shrink the viewport bottom by 50px
  }
);

// Watch multiple elements
document.querySelectorAll(".fade-in").forEach((el) => observer.observe(el));
```

For lazy-loading images:

```javascript
const imageObserver = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src; // Move the real URL from data-src to src
      imageObserver.unobserve(img);
    }
  });
});

document.querySelectorAll("img[data-src]").forEach((img) => imageObserver.observe(img));
```

IntersectionObserver is vastly more performant than listening to the `scroll` event, which fires dozens of times per second.

## Geolocation API

The Geolocation API requests the user's physical location. It always requires explicit user permission.

```javascript
// One-time position read
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude, accuracy } = position.coords;
    console.log(`You are at ${latitude}, ${longitude} (±${accuracy}m)`);
  },
  (error) => {
    // error.code: 1 = PERMISSION_DENIED, 2 = POSITION_UNAVAILABLE, 3 = TIMEOUT
    console.error("Geolocation error:", error.message);
  },
  { timeout: 10000, maximumAge: 60000 } // Options
);

// Watch for continuous updates (like for navigation apps)
const watchId = navigator.geolocation.watchPosition(successFn, errorFn);
navigator.geolocation.clearWatch(watchId); // Stop watching
```

Because it requires permission and can fail for multiple reasons, always handle both callbacks. Never assume geolocation will succeed.

## Clipboard API

`navigator.clipboard` provides async read/write access to the clipboard. It requires user permission for reading, and write access requires the page to be focused.

```javascript
// Write text to clipboard (common — "Copy link" button)
async function copyToClipboard(text) {
  try {
    await navigator.clipboard.writeText(text);
    showToast("Copied!");
  } catch {
    showToast("Couldn't copy — try selecting and copying manually.");
  }
}

// Read text from clipboard (requires explicit permission prompt)
async function pasteFromClipboard() {
  try {
    const text = await navigator.clipboard.readText();
    processText(text);
  } catch {
    showError("Clipboard access denied.");
  }
}
```

Always wrap clipboard calls in try/catch. The user may deny permission, or the page may not be focused.

## Web Share API

On mobile devices, the Web Share API lets users share content through the OS native share sheet:

```javascript
async function shareContent() {
  if (!navigator.share) {
    // Fallback for browsers that don't support it
    copyToClipboard(window.location.href);
    return;
  }

  try {
    await navigator.share({
      title: "Check out this lesson",
      text: "I just learned about Browser APIs in DURA!",
      url: window.location.href,
    });
  } catch (error) {
    if (error.name !== "AbortError") {
      // User cancelled — no action needed. Other errors: log them.
      console.error("Share failed:", error);
    }
  }
}
```

## Feature detection

Never assume a browser API exists. Feature detection checks at runtime before calling:

```javascript
// Check with the "in" operator — does this property exist on the object?
if ("geolocation" in navigator) {
  // Safe to use
}

if ("share" in navigator) {
  shareButton.style.display = "block";
} else {
  copyButton.style.display = "block"; // Show fallback
}

// Optional chaining for nested checks
const supported = navigator.clipboard?.writeText !== undefined;
```

The `"feature" in object` pattern returns `true` even if the value is `undefined` or `null`, making it more reliable than checking truthiness alone. This is how you build apps that degrade gracefully on older browsers without crashing.

## What you learned

- The History API (`pushState`, `replaceState`, `popstate` event) is the foundation of client-side routing — it changes the URL and state without a server round-trip
- `IntersectionObserver` efficiently detects when elements enter the viewport, replacing scroll event polling for lazy loading and scroll animations
- Always feature-detect APIs with `"featureName" in object` before using them, and provide fallbacks — browser support is never universal
