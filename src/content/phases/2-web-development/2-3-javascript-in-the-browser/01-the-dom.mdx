## Why this matters

Every interactive web page — dropdown menus, live search results, dark mode toggles — works by reading and changing the DOM. Before you can write a single line of interactive JavaScript, you need to understand what the browser gives you to work with: a live, structured tree of objects that represents your page.

## What the DOM is

When a browser loads an HTML file, it doesn't display the raw text. It parses the HTML and builds a data structure called the **Document Object Model** (DOM). The DOM is a tree of **nodes**, where each node represents a piece of the document.

Your HTML becomes a tree. This `<body>`:

```html
<body>
  <header>
    <h1>Hello</h1>
  </header>
  <main>
    <p>World</p>
  </main>
</body>
```

Becomes a tree that looks like:

```
document
└── html
    ├── head
    └── body
        ├── header
        │   └── h1
        │       └── "Hello"  (text node)
        └── main
            └── p
                └── "World"  (text node)
```

The DOM is **not** your HTML source code. It's a live object tree that exists in memory. When JavaScript changes the DOM, the browser immediately reflects those changes on screen — without reloading the page. The source file on disk stays the same; the DOM in memory changes.

## Nodes and elements

The DOM tree contains different types of nodes:

- **Element nodes** — represent HTML tags (`<div>`, `<p>`, `<button>`)
- **Text nodes** — the raw text inside an element
- **Comment nodes** — HTML comments (`<!-- like this -->`)
- **The document node** — the root, accessible as `document`

In practice, you mostly work with element nodes. Every element node has properties that mirror its HTML attributes, and methods that let you manipulate it.

## Selecting elements

To work with an element, you first need to get a reference to it. JavaScript provides several ways to do this.

### querySelector

`document.querySelector()` accepts any valid CSS selector and returns the **first matching element**, or `null` if nothing matches.

```javascript
// Select by tag name
const heading = document.querySelector("h1");

// Select by class
const card = document.querySelector(".card");

// Select by ID
const nav = document.querySelector("#main-nav");

// Select with a complex selector
const firstButton = document.querySelector("form button[type='submit']");
```

`document.querySelectorAll()` returns a `NodeList` of **all** matching elements:

```javascript
const allParagraphs = document.querySelectorAll("p");
// allParagraphs is not an Array, but you can iterate it
allParagraphs.forEach((p) => console.log(p.textContent));
```

### getElementById

`document.getElementById()` is a legacy method that's slightly faster for ID lookups. It does not take a `#` prefix — just the bare ID string.

```javascript
// These do the same thing:
document.querySelector("#main-nav");
document.getElementById("main-nav");
```

Prefer `querySelector` in new code — it's consistent with CSS selectors you already know, and it works for any selector, not just IDs.

## Traversing the tree

Once you have an element, you can navigate to related elements using traversal properties:

```javascript
const item = document.querySelector(".list-item");

// Parent
item.parentElement; // The direct parent element

// Children
item.children; // HTMLCollection of child elements (excludes text nodes)
item.firstElementChild; // First child element
item.lastElementChild; // Last child element

// Siblings
item.nextElementSibling; // The next element at the same level
item.previousElementSibling; // The previous element at the same level
```

There are also node-based equivalents (`parentNode`, `childNodes`, `nextSibling`) that include text nodes. You'll almost always want the `Element` versions, not the `Node` versions, because text nodes cause subtle bugs.

### When to traverse vs. when to query

Traversal is useful when you already have a reference and need a nearby element. Querying is more robust when the structure might change or when you need precision.

```javascript
// Fragile — breaks if you add a wrapper div
const label = button.previousElementSibling;

// Robust — works regardless of structure
const label = document.querySelector(`label[for="${button.id}"]`);
```

## The DOM is live

This is the most important thing to understand about the DOM: it's a live view of the page. When you change a DOM node, the page updates immediately. When the page updates (say, a form adds an error message), the DOM reflects that too.

```javascript
// This h1 reference always points to the current state of that element.
// If something else changes it, your reference sees the new state.
const title = document.querySelector("h1");
console.log(title.textContent); // "Hello"
// ... somewhere else changes the h1 ...
console.log(title.textContent); // "Goodbye" — same reference, updated state
```

This live nature is what makes DOM manipulation powerful. It's also what makes order of operations matter — selecting an element before it exists in the DOM returns `null`.

## What you learned

- The DOM is a live tree of node objects the browser builds from your HTML — changing it updates the page immediately without a reload
- `querySelector` accepts any CSS selector and returns the first match; `querySelectorAll` returns all matches as a NodeList
- Element traversal properties like `parentElement`, `children`, `nextElementSibling` let you navigate the tree from a known element
