## Why this matters

Selecting elements is passive. Manipulation is where the page comes alive. Every time a todo item appears on screen, a notification fades in, or a tab panel switches — DOM manipulation is doing the work. Understanding the right tools for the job also means understanding the security pitfalls that trip up developers every day.

## Creating and inserting elements

The browser gives you `document.createElement()` to create new elements in memory. The element doesn't appear on screen until you attach it to the document tree.

```javascript
// 1. Create an element
const li = document.createElement("li");

// 2. Set its content and attributes
li.textContent = "Buy groceries";
li.className = "todo-item";
li.setAttribute("data-id", "42");

// 3. Attach it to the DOM — now it appears on screen
const list = document.querySelector("#todo-list");
list.appendChild(li);
```

`appendChild` adds the new element as the **last child** of the target. To insert in a specific position, use `insertBefore`:

```javascript
// Insert before a specific existing child
const secondItem = list.children[1];
list.insertBefore(li, secondItem); // li goes before secondItem
```

The modern alternative is `element.insertAdjacentElement()`, which is more readable:

```javascript
const referenceItem = document.querySelector(".todo-item");

referenceItem.insertAdjacentElement("beforebegin", li); // Before the element
referenceItem.insertAdjacentElement("afterbegin", li); // First child inside
referenceItem.insertAdjacentElement("beforeend", li); // Last child inside
referenceItem.insertAdjacentElement("afterend", li); // After the element
```

## Reading and updating content

### textContent

`textContent` reads or sets the text inside an element. It treats everything as plain text — HTML tags are escaped, not parsed.

```javascript
const heading = document.querySelector("h1");
console.log(heading.textContent); // "Hello, World"

heading.textContent = "Welcome back!"; // Safe — no HTML parsing
```

### innerHTML — and why to be careful

`innerHTML` reads or sets the full HTML markup inside an element. It parses whatever string you give it as HTML.

```javascript
const card = document.querySelector(".card");
card.innerHTML = "<strong>Important:</strong> Read the docs."; // Works fine
```

The danger: if you inject user-supplied data into `innerHTML`, you open a **cross-site scripting (XSS)** vulnerability. A malicious user can inject a `<script>` tag or an event handler that steals data or hijacks sessions.

```javascript
// NEVER do this — a user could type: <img src=x onerror="stealCookies()">
const userInput = someFormField.value;
container.innerHTML = userInput; // XSS vulnerability

// ALWAYS do this instead:
container.textContent = userInput; // Plain text, nothing is parsed as HTML
```

The rule: **use `textContent` for user data, `innerHTML` only for trusted static strings**.

## Working with attributes and classes

### setAttribute and getAttribute

```javascript
const img = document.querySelector("img");

img.getAttribute("src"); // "/images/photo.jpg"
img.setAttribute("alt", "A mountain at sunrise");
img.removeAttribute("title");

// Boolean attributes work too
const input = document.querySelector("input");
input.setAttribute("disabled", ""); // Adds disabled
input.removeAttribute("disabled"); // Removes it
```

For standard attributes, you can also use them as direct properties:

```javascript
img.src = "/images/new-photo.jpg"; // Same as setAttribute
img.alt = "A forest";
```

### classList

Toggling CSS classes is how you apply visual state changes. The `classList` API makes this clean:

```javascript
const button = document.querySelector(".btn");

button.classList.add("active"); // Add a class
button.classList.remove("active"); // Remove a class
button.classList.toggle("active"); // Add if absent, remove if present
button.classList.contains("active"); // Returns true or false
button.classList.replace("btn-primary", "btn-secondary"); // Swap classes
```

`classList.toggle` is particularly useful for open/close states:

```javascript
menuButton.addEventListener("click", () => {
  navMenu.classList.toggle("is-open");
});
```

### dataset attributes

`data-*` attributes store custom data on elements. They're accessible through the `dataset` property, with the `data-` prefix stripped and the name converted to camelCase:

```javascript
// HTML: <li data-id="42" data-priority="high">...</li>
const item = document.querySelector("li");

item.dataset.id; // "42" — always a string
item.dataset.priority; // "high"

item.dataset.completed = "true"; // Sets data-completed="true" on the element
```

Dataset attributes are the standard way to embed IDs or state into HTML so event handlers can read them without a separate lookup.

## Removing elements

```javascript
const item = document.querySelector(".todo-item");

// Modern — remove the element from its parent
item.remove();

// Legacy (still common) — remove a child via its parent
item.parentElement.removeChild(item);
```

## What you learned

- `createElement` creates elements in memory; `appendChild` (or `insertAdjacentElement`) attaches them to the DOM so they appear on screen
- Always use `textContent` for user-provided data — `innerHTML` parses HTML and creates XSS vulnerabilities
- `classList.add/remove/toggle/contains` manages CSS classes without string manipulation; `dataset` gives clean access to `data-*` attributes
