## Why this matters

Real applications have hundreds of pages — blog posts, user profiles, product listings — that share a template but differ in content. You can't create a file for every blog post. Dynamic routing solves this, and Next.js makes it as simple as naming a folder `[slug]`. Understanding the full routing toolkit lets you model any URL structure your product needs.

## Dynamic route segments

A folder name wrapped in square brackets creates a **dynamic segment** — a placeholder that matches any value at that position in the URL.

```
app/
└── blog/
    └── [slug]/
        └── page.tsx    ← matches /blog/anything
```

The matched value is passed to the page as `params`:

```typescript
// app/blog/[slug]/page.tsx
interface Props {
  params: Promise<{ slug: string }>;
}

export default async function BlogPost({ params }: Props) {
  const { slug } = await params; // "hello-world", "my-first-post", etc.
  const post = await fetchPost(slug);
  return <article>{post.content}</article>;
}
```

You can have multiple dynamic segments at different levels:

```
app/shop/[category]/[productId]/page.tsx
// matches: /shop/electronics/42
// params: { category: "electronics", productId: "42" }
```

## Catch-all segments

A `[...slug]` folder matches one or more path segments and captures them as an array:

```
app/docs/[...path]/page.tsx
// matches: /docs/getting-started
// matches: /docs/api/reference/auth
// params.path: ["getting-started"] or ["api", "reference", "auth"]
```

An **optional catch-all** `[[...slug]]` also matches the route itself (with no segments):

```
app/docs/[[...path]]/page.tsx
// also matches: /docs  (path is undefined)
```

Use catch-all routes for documentation trees, file browsers, or any URL hierarchy with variable depth.

## The `Link` component

The `<Link>` component from `next/link` is how you navigate between pages in Next.js. It renders an `<a>` tag but intercepts the click to perform **client-side navigation** — the browser doesn't do a full page reload.

```typescript
import Link from "next/link";

export function Nav() {
  return (
    <nav>
      <Link href="/">Home</Link>
      <Link href="/about">About</Link>
      <Link href="/blog/hello-world">Read Post</Link>
    </nav>
  );
}
```

`Link` also **prefetches** routes automatically: when a `<Link>` scrolls into the viewport, Next.js silently fetches the destination page in the background. By the time the user clicks, the page is already loaded — navigation feels instant.

Useful props on `<Link>`:

```typescript
// Scroll to top on navigation (default: true)
<Link href="/blog" scroll={true}>Blog</Link>

// Replace instead of push in browser history
<Link href="/login" replace>Login</Link>

// Pass dynamic values
<Link href={`/blog/${post.slug}`}>{post.title}</Link>
```

## Programmatic navigation with `useRouter`

When you need to navigate in response to an event (form submission, timer, business logic), use the `useRouter` hook. It's a Client Component hook:

```typescript
"use client";

import { useRouter } from "next/navigation";

export function LoginForm() {
  const router = useRouter();

  async function handleSubmit(event: React.FormEvent) {
    event.preventDefault();
    await loginUser(/* ... */);
    router.push("/dashboard"); // navigate after login
  }

  return <form onSubmit={handleSubmit}>{/* ... */}</form>;
}
```

Key methods on the router object:

- `router.push(href)` — navigate and add to history stack
- `router.replace(href)` — navigate without adding to history (user can't go back)
- `router.back()` — go back in history
- `router.refresh()` — re-fetch server data for the current page without a full reload

## Reading URL state

Two hooks let you read the current URL in Client Components:

```typescript
"use client";

import { usePathname, useSearchParams } from "next/navigation";

export function BreadCrumb() {
  const pathname = usePathname();    // "/blog/hello-world"
  const searchParams = useSearchParams(); // URLSearchParams instance

  const page = searchParams.get("page"); // null or "2"

  return <span>{pathname}</span>;
}
```

For reading and updating search params (filters, pagination), combine these with `useRouter`:

```typescript
function updateFilter(value: string) {
  const params = new URLSearchParams(searchParams.toString());
  params.set("filter", value);
  router.push(`?${params.toString()}`);
}
```

## Parallel routes

Parallel routes let you render multiple pages in the same layout simultaneously, each in its own "slot". Define a slot by creating a folder prefixed with `@`:

```
app/
├── layout.tsx
├── page.tsx
├── @modal/
│   └── page.tsx   ← rendered alongside the main page
└── @sidebar/
    └── page.tsx   ← another parallel slot
```

The layout receives each slot as a named prop:

```typescript
// app/layout.tsx
export default function Layout({
  children,
  modal,
  sidebar,
}: {
  children: React.ReactNode;
  modal: React.ReactNode;
  sidebar: React.ReactNode;
}) {
  return (
    <div>
      {sidebar}
      {children}
      {modal}
    </div>
  );
}
```

Parallel routes are ideal for modals, split views, and dashboards where multiple independent sections load simultaneously.

## Generating static params

When you have a dynamic route and want to pre-render all possible values at build time (SSG), export `generateStaticParams`:

```typescript
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await fetchAllPosts();
  return posts.map((post) => ({
    slug: post.slug,
  }));
}
```

At build time, Next.js calls this function, gets all the slugs, and pre-renders a static HTML file for each one. No server work happens at request time — it's instant.

## Try it

<SandboxExercise language="javascript">
{`// Write a parseRoutePattern(pattern) function.
// Given a Next.js route pattern, return a match(url) function.
//
// Patterns to support:
//   "/blog/[slug]"    -> match("/blog/hello") -> { slug: "hello" }
//   "/docs/[...path]" -> match("/docs/a/b/c") -> { path: ["a","b","c"] }
//   "/about"          -> match("/about")       -> {}
//   "/about"          -> match("/contact")     -> null

function parseRoutePattern(pattern) {
const patternParts = pattern.split("/").filter(Boolean);

return function match(url) {
const urlParts = url.split("/").filter(Boolean);
const params = {};

    for (let i = 0; i < patternParts.length; i++) {
      const p = patternParts[i];
      const u = urlParts[i];

      // Catch-all: [...slug]
      if (p.startsWith("[...") && p.endsWith("]")) {
        const key = p.slice(4, -1);
        if (urlParts.length <= i) return null; // must have at least one segment
        params[key] = urlParts.slice(i);
        return params; // consume the rest
      }

      // Dynamic: [slug]
      if (p.startsWith("[") && p.endsWith("]")) {
        const key = p.slice(1, -1);
        if (u === undefined) return null;
        params[key] = u;
        continue;
      }

      // Static segment
      if (p !== u) return null;
    }

    // URL must not have extra segments (unless catch-all consumed them)
    if (urlParts.length !== patternParts.length && !pattern.includes("[...")) {
      return null;
    }

    return params;

};
}

// Tests
const blogMatch = parseRoutePattern("/blog/[slug]");
console.log(blogMatch("/blog/hello-world")); // { slug: "hello-world" }
console.log(blogMatch("/blog/")); // null
console.log(blogMatch("/about")); // null

const docsMatch = parseRoutePattern("/docs/[...path]");
console.log(docsMatch("/docs/getting-started")); // { path: ["getting-started"] }
console.log(docsMatch("/docs/api/reference/auth")); // { path: ["api","reference","auth"] }

const aboutMatch = parseRoutePattern("/about");
console.log(aboutMatch("/about")); // {}
console.log(aboutMatch("/contact")); // null`}

</SandboxExercise>

## What you learned

- Dynamic segments (`[slug]`) match any single URL value; catch-all segments (`[...slug]`) match one or more values as an array; optional catch-all (`[[...slug]]`) also matches the base route
- The `Link` component performs client-side navigation and prefetches routes automatically when they enter the viewport
- `useRouter` provides programmatic navigation (`push`, `replace`, `back`, `refresh`) and must be used in Client Components
- `generateStaticParams` pre-renders all dynamic routes at build time for maximum performance
