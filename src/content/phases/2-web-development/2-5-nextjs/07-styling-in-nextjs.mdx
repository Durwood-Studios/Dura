## Why this matters

Styling decisions compound. Pick the wrong approach early and you spend months fighting specificity wars, fighting class name collisions, fighting bundle bloat. Next.js is opinionated enough to give you great defaults — Tailwind CSS for utility classes, `next/font` for zero-layout-shift fonts, `next/image` for optimized images — but flexible enough to support the full CSS ecosystem. Know your tools and you'll ship polished UIs fast.

## Tailwind CSS v4

Tailwind CSS v4 is a significant departure from v3. The biggest change: **there is no `tailwind.config.js` file**. Everything that used to live in that file now lives in your CSS, inside an `@theme` block in `globals.css`.

```css
/* app/globals.css */
@import "tailwindcss";

@theme {
  --color-brand: #10b981;
  --color-surface: rgba(255, 255, 255, 0.04);
  --font-sans: "DM Sans", sans-serif;
  --font-mono: "JetBrains Mono", monospace;
  --radius-card: 12px;
  --spacing-content: 700px;
}
```

Once defined, these tokens become Tailwind utility classes automatically. `--color-brand` becomes `bg-brand`, `text-brand`, and `border-brand`. `--radius-card` becomes `rounded-card`. Your design system lives in one place, works everywhere.

Tailwind v4 uses CSS cascade layers internally, which means specificity is predictable and overrides are clean. It also processes faster than v3 because it uses a native CSS engine rather than a PostCSS plugin.

## Global styles

`app/globals.css` is imported in your root layout and applies to every page. It's the right place for:

- Base resets and default element styles
- Font face declarations
- CSS custom properties (design tokens)
- The `@theme` block for Tailwind tokens
- Any styles that truly need to be global

```css
/* app/globals.css */
@import "tailwindcss";

@theme {
  --color-bg: #08080d;
  --color-text: #f0f0f0;
}

/* Base resets */
*,
*::before,
*::after {
  box-sizing: border-box;
}

body {
  background-color: var(--color-bg);
  color: var(--color-text);
  font-family: var(--font-sans);
}
```

```typescript
// app/layout.tsx — import globals.css once here
import "./globals.css";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

## CSS Modules

When you need styles that are scoped to a single component — no global pollution, no name collisions — CSS Modules are the answer. Create a file with the `.module.css` extension alongside your component:

```css
/* components/Card.module.css */
.card {
  background: var(--color-surface);
  border-radius: 12px;
  padding: 1.5rem;
}

.card:hover {
  background: rgba(255, 255, 255, 0.06);
}

.title {
  font-size: 1.25rem;
  font-weight: 600;
}
```

```typescript
// components/Card.tsx
import styles from "./Card.module.css";

export function Card({ title }: { title: string }) {
  return (
    <div className={styles.card}>
      <h2 className={styles.title}>{title}</h2>
    </div>
  );
}
```

At build time, Next.js transforms `.card` into a unique class like `Card_card__x7y3z`. The name won't collide with any other `.card` class in your codebase. CSS Modules are excellent for complex component-specific styles where Tailwind utilities would get unwieldy.

## The `cn()` utility

Tailwind classes pile up quickly — especially when you need conditional classes based on props or state. The `cn()` utility, built from `clsx` + `tailwind-merge`, solves this cleanly:

```typescript
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]): string {
  return twMerge(clsx(inputs));
}
```

```typescript
// Usage
<button
  className={cn(
    "rounded-lg px-4 py-2 font-medium",          // base styles
    isPrimary && "bg-emerald-500 text-white",      // conditional
    isDisabled && "opacity-50 cursor-not-allowed", // conditional
    className,                                     // prop override
  )}
>
  {children}
</button>
```

`clsx` handles the conditional logic (filters out `false`, `null`, `undefined`). `twMerge` resolves Tailwind conflicts — if both `px-4` and `px-8` end up in the class list, it keeps only the last one instead of letting the one with higher specificity win unpredictably.

## Font optimization with `next/font`

Web fonts are a common source of layout shift — the page renders with a system font, then jumps when the web font loads. `next/font/google` eliminates this by downloading fonts at build time and self-hosting them:

```typescript
// app/layout.tsx
import { DM_Sans, JetBrains_Mono } from "next/font/google";

const dmSans = DM_Sans({
  subsets: ["latin"],
  variable: "--font-sans",
  display: "swap",
});

const jetbrainsMono = JetBrains_Mono({
  subsets: ["latin"],
  variable: "--font-mono",
  display: "swap",
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={`${dmSans.variable} ${jetbrainsMono.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

The `variable` option creates a CSS custom property you reference in your CSS: `font-family: var(--font-sans)`. No Google Fonts CDN request in production — fonts are bundled with your app. Zero layout shift, better privacy, faster load on repeat visits.

## Image optimization with `next/image`

The `<Image>` component from `next/image` wraps the standard `<img>` tag with automatic optimization:

```typescript
import Image from "next/image";

export function Avatar({ src, name }: { src: string; name: string }) {
  return (
    <Image
      src={src}
      alt={name}
      width={64}
      height={64}
      className="rounded-full"
    />
  );
}
```

What happens automatically:

- **Format conversion** — images are served as WebP or AVIF when the browser supports it
- **Responsive sizing** — the right resolution is served for the user's screen
- **Lazy loading** — images below the fold are not fetched until the user scrolls near them
- **Aspect ratio** — `width` and `height` reserve space so the page doesn't jump when the image loads

For images that should load immediately (hero images, above-the-fold content), add `priority`:

```typescript
<Image src="/hero.jpg" alt="Hero" width={1200} height={600} priority />
```

{`// Write the cn() utility function.
// cn(...args) accepts any mix of:
//   - strings: "foo bar"
//   - undefined / null / false (ignored)
//   - objects: { "class-name": true, "other-class": false }
//
// It should filter out all falsy values and return a single
// space-joined string of all the active class names.
//
// (This is the core logic of the clsx library.)

function cn(...args) {
const classes = [];

for (const arg of args) {
if (!arg) continue; // skip falsy: false, null, undefined, 0, ""

    if (typeof arg === "string") {
      classes.push(arg);
    } else if (typeof arg === "object" && !Array.isArray(arg)) {
      // Object: include keys where value is truthy
      for (const [key, value] of Object.entries(arg)) {
        if (value) classes.push(key);
      }
    } else if (Array.isArray(arg)) {
      // Flatten arrays recursively
      const nested = cn(...arg);
      if (nested) classes.push(nested);
    }

}

return classes.join(" ");
}

// Tests
console.log(cn("btn", "btn-primary"));
// "btn btn-primary"

console.log(cn("btn", false && "btn-disabled", null, undefined));
// "btn"

console.log(cn("btn", { "btn-active": true, "btn-disabled": false }));
// "btn btn-active"

console.log(cn("base", { active: true }, "extra"));
// "base active extra"

console.log(cn());
// ""

const isPrimary = true;
const isDisabled = false;
console.log(cn("rounded px-4 py-2", isPrimary && "bg-emerald-500", isDisabled && "opacity-50"));
// "rounded px-4 py-2 bg-emerald-500"`}

## What you learned

- Tailwind CSS v4 has no config file — custom design tokens go in an `@theme` block in `globals.css` and become utility classes automatically
- CSS Modules scope class names to their file at build time, making common names like `.card` safe to reuse across components
- The `cn()` utility (built from `clsx` + `tailwind-merge`) handles conditional class logic and resolves Tailwind conflicts cleanly
- `next/font/google` self-hosts fonts at build time for zero layout shift; `next/image` optimizes images with automatic format conversion, lazy loading, and responsive sizing
