## Why this matters

Most bugs fall into a small number of patterns. If you can recognize them on sight, you will fix them in seconds instead of hours.

## Off-by-one errors

```javascript
// Bug: <= instead of <
for (let i = 0; i <= arr.length; i++) {
  console.log(arr[i]); // last iteration: arr[arr.length] is undefined
}
```

Fix: `i < arr.length`. This is the most common loop bug in all of programming.

## Accidental mutation

```javascript
function addTag(tags, newTag) {
  tags.push(newTag); // mutates the original
  return tags;
}
```

The caller does not expect `tags` to change. Fix: `return [...tags, newTag]`.

## Type coercion surprises

```javascript
if (userInput == 0) // true for "", false, null
if (userInput === 0) // only true for 0
```

Fix: always use `===`.

## Forgetting await

```javascript
const data = fetch("/api"); // Promise, not data!
console.log(data.name); // undefined
```

Fix: `const response = await fetch("/api"); const data = await response.json();`

## Variable shadowing

```javascript
const name = "global";
function greet() {
  const name = "local"; // shadows the outer name
  console.log(name); // "local" — maybe unintentional
}
```

Not always a bug, but a source of confusion. Use unique names or be explicit.

## Reference vs value

```javascript
const a = { x: 1 };
const b = a; // b points to the SAME object
b.x = 99;
console.log(a.x); // 99 — a was "changed" too
```

Fix: `const b = { ...a }` for a shallow copy.

## What you learned

- Most bugs are off-by-one, accidental mutation, forgotten await, or type coercion.
- Use `===`, copy before mutating, `await` every async call, and check loop bounds.
- If you can name the pattern, you can fix the bug in seconds.
