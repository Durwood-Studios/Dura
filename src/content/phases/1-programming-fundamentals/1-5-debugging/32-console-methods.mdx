## Why this matters

`console.log` is the most-used debugging tool in the world. But the console object has a dozen other methods that can save you minutes of formatting output by hand.

## console.log — the basics

```javascript
console.log("Hello");
console.log("count:", 42);
console.log({ name: "Ada", age: 30 });
```

Multiple arguments are printed space-separated. Objects are expandable in the browser console.

## console.table — structured data

```javascript
const users = [
  { name: "Ada", age: 30 },
  { name: "Bob", age: 25 },
];
console.table(users);
```

Renders as a formatted table. Best way to inspect arrays of objects.

## console.warn and console.error

```javascript
console.warn("This might be a problem.");
console.error("This IS a problem.");
```

They appear with yellow (warn) and red (error) styling and include stack traces. Use them to distinguish severity — your future self will thank you.

## console.group / groupEnd

```javascript
console.group("User processing");
console.log("Loading...");
console.log("Done.");
console.groupEnd();
```

Groups related messages with an indent. Collapsed variant: `console.groupCollapsed`.

## console.time / timeEnd

```javascript
console.time("sort");
const sorted = bigArray.sort();
console.timeEnd("sort"); // sort: 12.345ms
```

Quick benchmarking without importing a library.

## console.trace

```javascript
function inner() {
  console.trace("Where am I?");
}
function outer() {
  inner();
}
outer();
```

Prints a stack trace without throwing an error. Useful when you want to see who called a function.

## What you learned

- `console.table` is the best way to inspect arrays of objects.
- `console.warn`/`error` add severity and stack traces. Use them.
- `console.time`/`timeEnd` measures elapsed time; `console.trace` shows the call stack.
