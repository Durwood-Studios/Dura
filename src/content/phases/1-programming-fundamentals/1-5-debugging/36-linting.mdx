## Why this matters

A linter finds bugs in your code before you run it. A formatter makes your code look consistent without you thinking about it. Together they eliminate entire categories of review comments and runtime surprises.

## ESLint: the linter

ESLint scans your JavaScript/TypeScript files and flags suspicious patterns.

```text
warning  'x' is assigned a value but never used  @typescript-eslint/no-unused-vars
error    Expected '===' and instead saw '=='      eqeqeq
```

Some rules are stylistic (quotes, semicolons). Some catch real bugs (unused variables, missing `await`). The best ESLint configs lean toward bug-catching and leave style to the formatter.

## Prettier: the formatter

Prettier reformats your code on every save. It handles indentation, line length, trailing commas, quote style — everything you would otherwise argue about in a review.

```javascript
// Before Prettier (messy)
const x = { a: 1, b: 2, c: 3 };

// After Prettier (consistent)
const x = { a: 1, b: 2, c: 3 };
```

The deal: you give up control over formatting. In return, every file in the project looks the same, every contributor's code looks the same, and nobody spends time on style arguments.

## How they work together

- **Prettier** handles formatting (where the spaces go, how lines wrap).
- **ESLint** handles logic (unused variables, missing return, bad patterns).
- Run both: `eslint-config-prettier` disables the ESLint rules that conflict with Prettier.

DURA uses both — every file in this codebase is auto-formatted on commit via Husky + lint-staged.

## Fixing automatically

```bash
npx eslint --fix .          # auto-fix fixable lint issues
npx prettier --write .      # reformat all files
```

Most lint issues and all formatting issues can be fixed automatically. Always run these before committing.

## What you learned

- **ESLint** finds bugs and bad patterns in static code.
- **Prettier** auto-formats everything. You stop thinking about style.
- Together they eliminate style arguments and catch mistakes before runtime.
