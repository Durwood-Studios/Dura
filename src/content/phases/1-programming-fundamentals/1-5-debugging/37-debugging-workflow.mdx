## Why this matters

Debugging is not guessing. It is a skill with a repeatable process. Developers who follow a systematic workflow find bugs faster, waste less time, and introduce fewer new bugs when they fix the old one.

## The four-step process

### 1. Reproduce

Before you fix anything, reproduce the bug reliably. "It sometimes crashes" is not a bug report — "Click Submit with an empty email field" is.

Write down the exact steps to trigger it. If you cannot reproduce it, you cannot verify a fix.

### 2. Isolate

Narrow down where the bug lives. Binary search through the code:

- Does the bug happen if you remove the last function call?
- Does it happen with a hardcoded value instead of user input?
- Does it happen in one browser but not another?

The goal is the **smallest possible reproducing case**. Comment out code, simplify inputs, strip away layers until you find the exact line.

### 3. Fix

Once you know the exact cause, fix it. A good fix:

- Changes the minimum amount of code.
- Does not break anything else.
- Has a clear reason (you can explain why the change works).

Resist the urge to refactor the surrounding code at the same time. Fix the bug, commit, then refactor separately.

### 4. Verify

Confirm the fix works by running the reproduction steps again. Then ask:

- Does it work with edge cases (empty input, null, extreme values)?
- Did I break anything nearby?
- Should I add a test so this bug never comes back?

## Writing a good bug report

When you find a bug — in your own code or someone else's — describe it as:

1. **What I expected** to happen.
2. **What actually happened.**
3. **Steps to reproduce** (numbered, specific).
4. **Environment** (browser, OS, relevant versions).

This format saves everyone time, including your future self.

    .split(" ")

  solution={`function titleCase(str) {
return str
.split(" ")
.map((word) => {
if (word.length === 0) return "";
return word[0].toUpperCase() + word.slice(1);
})
.join(" ");
}

console.log(titleCase("hello world"));
console.log(titleCase("a b c"));
console.log(titleCase("hello world"));`}
testCases={["titleCase handles double spaces"]}

## What you learned

- The debugging workflow: **reproduce → isolate → fix → verify**.
- Fix the minimum. Don't refactor while debugging.
- Write bug reports with: expected, actual, steps, environment.
