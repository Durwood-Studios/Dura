## Why this matters

`console.log` is good for quick checks. But when you need to see every variable in scope, step through code line by line, and test expressions on the fly, you need a debugger. Every browser and every IDE has one. Learning to use it will save you hours per week.

## The debugger statement

Drop the `debugger` keyword into your code and open your browser's DevTools. When JavaScript hits that line, it pauses.

```javascript
function processItems(items) {
  debugger; // execution stops here
  return items.map((item) => item.toUpperCase());
}
```

While paused you can:

- Hover any variable to see its current value.
- Type expressions in the console panel to test hypotheses.
- Step forward one line at a time.
- Resume (continue) execution until the next breakpoint.

## DevTools breakpoints

Instead of editing code, you can click a line number in the browser's Sources panel to set a breakpoint. The effect is the same — execution pauses when that line is reached — but you do not need to change your source.

**Conditional breakpoints** only pause when a condition is true. Right-click the line number → "Add conditional breakpoint" → enter `user.role === "admin"`.

## Step controls

When paused, four buttons control execution:

- **Resume (▶)** — continue until the next breakpoint.
- **Step Over (⏭)** — run the current line and pause on the next line in the same function.
- **Step Into (↓)** — if the current line calls a function, enter it.
- **Step Out (↑)** — run the rest of the current function and pause in the caller.

Step Over is the most common. Use Step Into only when you want to see inside a specific function call.

## Watch expressions

Add a variable name or expression to the Watch panel. Its value updates every time execution pauses. Useful for tracking a value across many steps without re-typing.

## VS Code debugging

VS Code has a built-in debugger that works with Node.js and browser JavaScript. You can set breakpoints by clicking the gutter, inspect variables in the sidebar, and use the debug console — all without leaving the editor.

## What you learned

- The `debugger` statement pauses execution. DevTools breakpoints do the same without editing code.
- Step Over, Step Into, Step Out, and Resume give you precise control over execution.
- Watch expressions let you track values across steps without re-typing.
