## Why this matters

Every developer sees errors. The difference between a beginner and an intermediate is how fast they read the error message instead of panicking. Most errors tell you exactly what went wrong and exactly where. You just have to look.

## Anatomy of an error

```text
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (app.js:12:15)
    at main (app.js:24:3)
```

Three pieces of information:

1. **Type:** `TypeError` — the category of what went wrong.
2. **Message:** `Cannot read properties of undefined (reading 'name')` — what happened.
3. **Stack trace:** where it happened, with file names and line numbers, most recent call first.

## Common error types

- **ReferenceError** — you used a variable that does not exist.
- **TypeError** — you used a value in a way its type does not support (like calling `.name` on `undefined`).
- **SyntaxError** — the code is structurally wrong (missing bracket, extra comma). The program does not run at all.
- **RangeError** — a value is outside the allowed range (e.g., infinite recursion blows the call stack).

## Reading the stack trace

```text
Error: User not found
    at findUser (src/users.js:45:11)
    at handleRequest (src/server.js:102:24)
    at processRequest (src/server.js:88:5)
```

Read from top to bottom. The top line is where the error was thrown. The lines below show who called whom. `handleRequest` called `findUser`, and `findUser` threw.

## The workflow

1. **Read the error type and message.** Don't skip this.
2. **Read the top of the stack trace.** Open that file, go to that line.
3. **Look at the value that caused it.** Is it `undefined`? `null`? The wrong type?
4. **Walk backwards.** Why is that value wrong? Where did it come from?
5. **Only then Google.** And paste the error message, not a description of your problem.

## What you learned

- Error messages have a **type**, a **message**, and a **stack trace**. Read all three.
- Common types: ReferenceError (undeclared), TypeError (wrong type), SyntaxError (bad structure).
- Read the error before Googling. Walk backwards from the throw site to the root cause.
