## Why this matters

Professional code is full of `||`, `&&`, `??`, and ternary patterns that look cryptic until you see the logic. This lesson breaks down the four most common patterns so you can read and write them confidently.

## Pattern 1: Default value with ||

```javascript
const name = userName || "Anonymous";
```

`||` returns the first **truthy** operand, or the last one. If `userName` is `""`, `null`, `undefined`, `0`, or `false`, name becomes `"Anonymous"`.

Problem: `0` and `""` are legitimate values that `||` treats as falsy.

## Pattern 2: Default value with ?? (better)

```javascript
const count = userCount ?? 10;
```

`??` returns the left side unless it is `null` or `undefined`. `0` and `""` pass through. Use `??` when zero or empty string are valid values.

## Pattern 3: Conditional execution with &&

```javascript
isAdmin && showAdminPanel();
```

`&&` returns the first **falsy** operand, or the last one. If `isAdmin` is falsy, `showAdminPanel()` never runs. This is the short-circuit pattern — the right side is only evaluated if the left side is truthy.

In React JSX you see this constantly:

```javascript
{
  isLoggedIn && <Dashboard />;
}
```

## Pattern 4: Ternary for inline values

```javascript
const label = count === 1 ? "item" : "items";
```

Clean for a single choice. Do not chain ternaries — they become unreadable.

```javascript
// Don't do this:
const color = status === "ok" ? "green" : status === "warn" ? "yellow" : "red";
// Do this instead:
function statusColor(status) {
  if (status === "ok") return "green";
  if (status === "warn") return "yellow";
  return "red";
}
```

## What you learned

- `||` defaults on any falsy value; `??` defaults only on null/undefined. Prefer `??`.
- `&&` short-circuits: the right side only runs if the left side is truthy.
- Ternary is great for one-line choices; don't chain them.
