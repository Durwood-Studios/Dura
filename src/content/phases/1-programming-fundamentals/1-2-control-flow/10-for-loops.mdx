## Why this matters

Anything you need to do once, you can do with a single statement. Anything you need to do ten times — or a thousand — you need a loop. The classic `for` loop is the foundation; every other loop builds on the same idea.

## The classic for loop

```javascript
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// 0, 1, 2, 3, 4
```

Three parts separated by semicolons:

1. **Init:** `let i = 0` — runs once before the loop starts.
2. **Condition:** `i < 5` — checked before each iteration. If false, the loop ends.
3. **Update:** `i++` — runs after each iteration.

## Looping through arrays

```javascript
const colors = ["red", "green", "blue"];

for (let i = 0; i < colors.length; i++) {
  console.log(colors[i]);
}
```

This is the pattern behind every array iteration: start at 0, go up to (not including) `length`, and use `i` as the index.

## for...of — the modern alternative

When you just want the values and do not need the index:

```javascript
const fruits = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
  console.log(fruit);
}
```

`for...of` is cleaner for most use cases. Use the classic `for` when you need the index or when you need to skip/modify the counter.

## break and continue

`break` exits the loop entirely. `continue` skips to the next iteration.

```javascript
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  if (i % 2 === 0) continue;
  console.log(i); // 1, 3
}
```

Use sparingly. Most loops are clearer without `break` or `continue`.

## Counting backwards

```javascript
for (let i = 5; i >= 0; i--) {
  console.log(i);
}
// 5, 4, 3, 2, 1, 0
```

## What you learned

- `for (init; condition; update)` repeats a block a controlled number of times.
- `for...of` is the cleaner alternative when you only need values, not indexes.
- `break` exits the loop; `continue` skips to the next iteration.
