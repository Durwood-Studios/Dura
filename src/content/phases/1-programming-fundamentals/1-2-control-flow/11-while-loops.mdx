## Why this matters

`for` loops work when you know the count ahead of time. But sometimes you do not: retry until the server responds, read lines until the file ends, prompt the user until they enter valid input. `while` loops handle that.

## while

```javascript
let attempts = 0;

while (attempts < 3) {
  console.log(`Attempt ${attempts + 1}`);
  attempts++;
}
```

The condition is checked **before** each iteration. If it is false from the start, the body never runs.

## do...while

```javascript
let input;
do {
  input = Math.random() > 0.7 ? "valid" : "invalid";
  console.log(input);
} while (input !== "valid");
```

`do...while` checks the condition **after** the body runs. This guarantees the body runs at least once — useful when you need to perform the action before you can test whether to continue.

## Infinite loops

A `while (true)` loop runs forever — or until a `break` inside it exits.

```javascript
let count = 0;
while (true) {
  if (count >= 5) break;
  console.log(count);
  count++;
}
```

This pattern is common in event loops and servers, but dangerous in ordinary code. If you forget the exit condition, the program hangs.

## Choosing between for and while

- Know the count? Use `for`.
- Know the exit condition but not the count? Use `while`.
- Need to run the body at least once? Use `do...while`.

Most real code uses `for` and `while`. `do...while` is rare but worth knowing.

## What you learned

- `while` repeats as long as its condition is truthy; the body may never run.
- `do...while` runs the body first, then checks — guaranteeing at least one execution.
- Use `for` when you know the count, `while` when you know the condition.
