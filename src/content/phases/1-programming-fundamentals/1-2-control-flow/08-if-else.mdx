## Why this matters

Until now, every program ran every line, top to bottom. Real programs need to choose: show a login form OR a dashboard. Charge a fee OR waive it. Retry a request OR give up. The `if` statement is the simplest way to make that choice.

## The basic shape

```javascript
const temperature = 35;

if (temperature > 30) {
  console.log("It's hot.");
}
```

The part inside the `()` is the **condition** — any expression that evaluates to true or false. If it is truthy, the block inside `{}` runs. If it is falsy, the block is skipped.

## Adding else

```javascript
const score = 72;

if (score >= 80) {
  console.log("Pass.");
} else {
  console.log("Try again.");
}
```

`else` catches everything the `if` condition did not. Exactly one of the two blocks runs — never both, never neither.

## Chaining with else if

```javascript
const grade = 85;

if (grade >= 90) {
  console.log("A");
} else if (grade >= 80) {
  console.log("B");
} else if (grade >= 70) {
  console.log("C");
} else {
  console.log("F");
}
```

JavaScript evaluates from top to bottom and runs the first block whose condition is true. The `else` at the end is a catch-all. The order of conditions matters: put the most specific or highest threshold first.

## The ternary operator

For simple either/or assignments, JavaScript has a shorthand:

```javascript
const age = 20;
const label = age >= 18 ? "adult" : "minor";
console.log(label); // "adult"
```

`condition ? valueIfTrue : valueIfFalse`. Use this for single expressions, not for complex logic. If you need more than one line per branch, use a regular `if`.

## Common mistakes

1. **Using `=` instead of `===`** in the condition. `if (x = 5)` assigns 5 to x (and is always truthy). You meant `if (x === 5)`.
2. **Forgetting braces.** Without `{}`, only the very next statement belongs to the `if`. This works fine for one-liners, but adding a second line later will not behave as you expect.

## What you learned

- `if` runs a block when its condition is truthy. `else` catches the rest. `else if` chains multiple conditions.
- The ternary operator `? :` is a single-expression shorthand for if/else.
- Order your conditions most-specific-first; use `===` not `=` in conditions.
