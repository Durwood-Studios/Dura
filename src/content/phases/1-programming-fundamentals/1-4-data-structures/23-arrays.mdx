## Why this matters

An array is an ordered list of values. Shopping carts, search results, chat messages, pixels on a screen — if data comes in a sequence, it lives in an array.

## Creating arrays

```javascript
const colors = ["red", "green", "blue"];
const numbers = [1, 2, 3, 4, 5];
const mixed = [42, "hello", true, null];
const empty = [];
```

Arrays can hold any type, including other arrays.

## Access and length

```javascript
const fruits = ["apple", "banana", "cherry"];
console.log(fruits[0]); // "apple"
console.log(fruits[2]); // "cherry"
console.log(fruits.length); // 3
console.log(fruits.at(-1)); // "cherry"
```

Zero-indexed. `.at(-1)` gives the last element without computing `length - 1`.

## Mutating methods

These change the array in place:

```javascript
const items = ["a", "b"];
items.push("c"); // ["a", "b", "c"]
items.pop(); // ["a", "b"] — removes last
items.unshift("z"); // ["z", "a", "b"] — adds to front
items.shift(); // ["a", "b"] — removes from front
items.splice(1, 0, "X"); // ["a", "X", "b"] — insert at index 1
```

## Non-mutating methods

These return a new array:

```javascript
const nums = [3, 1, 4, 1, 5];
console.log(nums.slice(1, 3)); // [1, 4]
console.log(nums.concat([9, 2])); // [3, 1, 4, 1, 5, 9, 2]
console.log(nums.includes(4)); // true
console.log(nums.indexOf(1)); // 1
console.log([...nums].sort()); // [1, 1, 3, 4, 5] — spread to avoid mutating
```

## Iterating

```javascript
const names = ["Ada", "Bob", "Cal"];

// for...of
for (const name of names) {
  console.log(name);
}

// forEach
names.forEach((name, i) => console.log(i, name));

// map (returns new array)
const upper = names.map((n) => n.toUpperCase());
```

## What you learned

- Arrays are ordered, zero-indexed lists. Access with `[index]` or `.at()`.
- Mutating methods: `push`, `pop`, `unshift`, `shift`, `splice`. Non-mutating: `slice`, `concat`, `map`, `filter`.
- Spread (`[...arr]`) creates a shallow copy — safe to sort or modify without affecting the original.
