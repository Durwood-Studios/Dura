## Why this matters

You know how to create arrays, objects, maps, and sets. Now you need to walk through them. JavaScript has several iteration tools — and using the wrong one causes subtle bugs.

## for...of — values

Works on arrays, strings, Maps, Sets, and any iterable.

```javascript
for (const item of [10, 20, 30]) {
  console.log(item); // 10, 20, 30
}

for (const char of "abc") {
  console.log(char); // "a", "b", "c"
}
```

**Does NOT work on plain objects.** An object is not iterable by default.

## for...in — keys (usually avoid)

Iterates over an object's **enumerable string keys**, including inherited ones.

```javascript
const obj = { a: 1, b: 2 };
for (const key in obj) {
  console.log(key, obj[key]); // "a" 1, "b" 2
}
```

On arrays, `for...in` iterates index strings (`"0"`, `"1"`, ...) — not values. It also picks up prototype properties. **Avoid for arrays.** Use `Object.keys()` or `Object.entries()` for objects.

## forEach — callback per element

```javascript
["a", "b", "c"].forEach((item, index) => {
  console.log(index, item);
});
```

No `break` or `continue` support. Cannot be `await`-ed in an async context. Use `for...of` when you need either.

## Object.entries + for...of

The standard pattern for iterating an object's key-value pairs:

```javascript
const scores = { math: 90, english: 85 };
for (const [subject, score] of Object.entries(scores)) {
  console.log(`${subject}: ${score}`);
}
```

## Map and Set iteration

```javascript
const map = new Map([
  ["a", 1],
  ["b", 2],
]);
for (const [key, value] of map) {
  console.log(key, value);
}

const set = new Set([10, 20, 30]);
for (const value of set) {
  console.log(value);
}
```

## What you learned

- `for...of` iterates values — works on arrays, strings, Maps, Sets. **Not** plain objects.
- `for...in` iterates keys — avoid on arrays; use `Object.keys/values/entries` for objects.
- `forEach` has no `break` or `await`. Prefer `for...of` when you need either.
