## Why this matters

JSON (JavaScript Object Notation) is the format nearly every web API uses to send and receive data. Config files, databases, local storage — JSON is everywhere. It looks like JavaScript objects, but it is stricter in the details.

## The format

```json
{
  "name": "Dustin",
  "age": 37,
  "active": true,
  "skills": ["typescript", "rust"],
  "address": null
}
```

Rules:

- Keys must be double-quoted strings.
- Values: string, number, boolean, null, array, object. No functions, no `undefined`.
- No trailing commas. No comments.

## JSON.stringify

Converts a JavaScript value to a JSON string.

```javascript
const data = { name: "Ada", score: 95 };
const json = JSON.stringify(data);
console.log(json); // '{"name":"Ada","score":95}'
```

Pretty-print with a spacer:

```javascript
console.log(JSON.stringify(data, null, 2));
```

## JSON.parse

Converts a JSON string back to a JavaScript value.

```javascript
const text = '{"name":"Ada","score":95}';
const obj = JSON.parse(text);
console.log(obj.name); // "Ada"
```

If the string is not valid JSON, `JSON.parse` throws.

```javascript
JSON.parse("not json"); // SyntaxError
```

Always wrap parse in try/catch when the input is not guaranteed valid.

## Round-trip gotchas

Some JavaScript values do not survive the JSON round-trip:

- `undefined` → stripped entirely
- `NaN`, `Infinity` → `null`
- `Date` objects → ISO string (and do not parse back to Date)
- `Map`, `Set`, `RegExp` → empty object `{}`

## What you learned

- JSON is a strict subset of JavaScript: double-quoted keys, no undefined, no trailing commas.
- `JSON.stringify` serializes; `JSON.parse` deserializes. Parse can throw — wrap it.
- The JSON round-trip drops `undefined`, Dates, Maps, and Sets. Know the limits.
