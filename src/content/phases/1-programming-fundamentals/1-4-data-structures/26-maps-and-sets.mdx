## Why this matters

Objects work fine as key-value stores when keys are strings. But what if you need non-string keys, care about insertion order, or need to check "has this been seen before" efficiently? `Map` and `Set` are built for those jobs.

## Map

A `Map` is a key-value collection where keys can be any type.

```javascript
const cache = new Map();
cache.set("user-1", { name: "Ada" });
cache.set(42, "the answer");

console.log(cache.get("user-1")); // { name: "Ada" }
console.log(cache.has(42)); // true
console.log(cache.size); // 2
```

Methods: `set`, `get`, `has`, `delete`, `clear`, `size`. Iterates in insertion order.

## When to use Map over Object

- Non-string keys (numbers, objects, functions as keys).
- Frequent additions/deletions — `Map` is optimized for it.
- You need `.size` without calling `Object.keys(obj).length`.
- You need guaranteed insertion-order iteration (objects maintain it in modern engines, but `Map` makes it explicit).

## Set

A `Set` is a collection of unique values. Duplicates are silently ignored.

```javascript
const seen = new Set();
seen.add("apple");
seen.add("banana");
seen.add("apple"); // ignored — already present

console.log(seen.size); // 2
console.log(seen.has("banana")); // true
```

## Deduplicating an array

```javascript
const dupes = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(dupes)];
console.log(unique); // [1, 2, 3]
```

This one-liner is the most common use of `Set` in everyday code.

## What you learned

- `Map` is a key-value store with any key type, guaranteed order, and a `.size` property.
- `Set` holds unique values. `[...new Set(arr)]` is the standard deduplication one-liner.
- Use Map over Object when you need non-string keys or frequent mutations.
