## Why this matters

You can solve any array problem with a `for` loop. But `map`, `filter`, `reduce`, `find`, `some`, and `every` say **what** you are doing, not just **how**. They make intent readable at a glance and compose into clean pipelines.

## map — transform every element

```javascript
const names = ["ada", "bob", "cal"];
const upper = names.map((n) => n.toUpperCase());
// ["ADA", "BOB", "CAL"]
```

Returns a **new array** of the same length.

## filter — keep matching elements

```javascript
const nums = [1, 2, 3, 4, 5, 6];
const evens = nums.filter((n) => n % 2 === 0);
// [2, 4, 6]
```

Returns a new array with a subset.

## reduce — accumulate to a single value

```javascript
const total = [10, 20, 30].reduce((sum, n) => sum + n, 0);
// 60
```

The second argument (`0`) is the initial accumulator. Each callback returns the next accumulator.

## find — first match

```javascript
const users = [
  { name: "Ada", admin: false },
  { name: "Bob", admin: true },
];
const admin = users.find((u) => u.admin);
console.log(admin.name); // "Bob"
```

Returns the first element that passes the test, or `undefined`.

## some and every

```javascript
const ages = [22, 17, 30];
console.log(ages.some((a) => a < 18)); // true — at least one
console.log(ages.every((a) => a >= 18)); // false — not all
```

Short-circuit: `some` stops on the first truthy; `every` stops on the first falsy.

## Chaining

```javascript
const transactions = [
  { amount: 50, type: "credit" },
  { amount: -20, type: "debit" },
  { amount: 100, type: "credit" },
  { amount: -10, type: "debit" },
];

const totalCredits = transactions
  .filter((t) => t.type === "credit")
  .map((t) => t.amount)
  .reduce((sum, a) => sum + a, 0);

console.log(totalCredits); // 150
```

## What you learned

- `map` transforms, `filter` selects, `reduce` accumulates. These three replace most loops.
- `find` gives the first match. `some`/`every` answer yes/no questions.
- Chain array methods for readable, composable data pipelines.
