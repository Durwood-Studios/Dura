## Why this matters

If arrays are lists, objects are records. A user has a name, email, and role. A product has a price, title, and stock count. Objects group related values under named keys.

## Creating objects

```javascript
const user = {
  name: "Dustin",
  email: "d@dura.dev",
  age: 37,
};
```

## Accessing properties

```javascript
console.log(user.name); // "Dustin"  — dot notation
console.log(user["email"]); // "d@dura.dev" — bracket notation
```

Use dot when the key is a valid identifier. Use brackets when the key is dynamic or has special characters.

```javascript
const key = "age";
console.log(user[key]); // 37
```

## Adding, changing, deleting

```javascript
user.role = "admin"; // add
user.age = 38; // change
delete user.email; // remove
console.log(user);
// { name: "Dustin", age: 38, role: "admin" }
```

Objects are mutable by default. `const` prevents reassignment of the variable, not mutation of the object.

## Checking keys

```javascript
console.log("name" in user); // true
console.log(user.hasOwnProperty("role")); // true
console.log(Object.keys(user)); // ["name", "age", "role"]
console.log(Object.values(user)); // ["Dustin", 38, "admin"]
console.log(Object.entries(user)); // [["name","Dustin"], ...]
```

## Shorthand properties

When the variable name matches the key:

```javascript
const name = "Ada";
const age = 30;
const person = { name, age }; // { name: "Ada", age: 30 }
```

## Spread and copying

```javascript
const defaults = { theme: "light", lang: "en" };
const prefs = { ...defaults, lang: "es" };
console.log(prefs); // { theme: "light", lang: "es" }
```

Spread creates a **shallow copy**. Nested objects still share the same reference.

## What you learned

- Objects hold key-value pairs. Access with `.key` or `["key"]`.
- `Object.keys/values/entries` iterate over an object's properties.
- Spread (`{...obj}`) creates a shallow copy — nested objects are still shared.
