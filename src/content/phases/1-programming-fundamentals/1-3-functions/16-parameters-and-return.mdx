## Why this matters

The difference between a useful function and a confusing one is usually how it handles its inputs and outputs. Defaults, rest parameters, and destructuring let you write functions that are easy to call correctly and hard to call incorrectly.

## Default parameters

```javascript
function greet(name = "stranger") {
  return `Hello, ${name}!`;
}

console.log(greet("Ada")); // "Hello, Ada!"
console.log(greet()); // "Hello, stranger!"
```

The default applies when the argument is `undefined` (including not passed at all). It does NOT apply when the argument is `null` or `""`.

## Rest parameters

When you do not know how many arguments there will be:

```javascript
function sum(...numbers) {
  let total = 0;
  for (const n of numbers) total += n;
  return total;
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(10, 20)); // 30
```

`...numbers` collects all arguments into a real array. It must be the last parameter.

## Destructuring parameters

When a function takes an object, destructuring in the parameter list makes the expected shape explicit:

```javascript
function createUser({ name, email, role = "member" }) {
  return { name, email, role, createdAt: Date.now() };
}

const user = createUser({ name: "Dustin", email: "d@dura.dev" });
console.log(user.role); // "member"
```

This is the standard pattern for functions with many optional fields. It replaces positional arguments (where you have to remember the order) with named arguments (where you can pass in any order).

## Return values

A function without a `return` statement returns `undefined`. A `return` with no value also returns `undefined`. Every function in JavaScript returns something — the question is whether you control what.

```javascript
function log(msg) {
  console.log(msg);
  // implicit return undefined
}

const result = log("test");
console.log(result); // undefined
```

## Returning early

You already saw early returns in control flow. They are the standard way to handle edge cases in functions:

```javascript
function divide(a, b) {
  if (b === 0) return null;
  return a / b;
}
```

## What you learned

- Default parameters apply when the argument is undefined. Rest parameters (`...`) collect extras into an array.
- Destructuring in the parameter list replaces positional arguments with named fields.
- Every function returns something — `undefined` if you don't explicitly `return`.
