## Why this matters

A pure function always returns the same output for the same input and changes nothing outside itself. This makes it predictable, testable, and safe to use anywhere. Understanding purity vs. side effects is the difference between code you can trust and code that surprises you.

## What pure means

A function is pure if:

1. **Same input → same output.** `add(2, 3)` always returns 5.
2. **No side effects.** It does not modify anything outside its own scope — no mutating arguments, no writing to globals, no logging, no network calls.

```javascript
// Pure
function add(a, b) {
  return a + b;
}

// Impure — modifies an external variable
let total = 0;
function addToTotal(n) {
  total += n; // side effect: changes external state
  return total;
}
```

## Why purity matters

- **Testing.** Pure functions need no setup — pass in values, check the output.
- **Reasoning.** You can read a pure function in isolation; you do not need to track what else it might change.
- **Concurrency.** Pure functions cannot cause race conditions because they do not share state.

## Side effects are necessary

You cannot build a useful program out of only pure functions. At some point you need to read input, display output, save data, or call an API. Those are side effects.

The strategy is: **keep the core logic pure and push side effects to the edges.**

```javascript
// Pure: the calculation
function calculateTax(price, rate) {
  return price * rate;
}

// Impure: the boundary
function displayInvoice(price) {
  const tax = calculateTax(price, 0.08);
  console.log(`Total: $${(price + tax).toFixed(2)}`);
}
```

`calculateTax` is pure — easy to test and reuse. `displayInvoice` is impure (it logs to the console) but it delegates the hard work to the pure function.

## Avoiding accidental mutation

```javascript
// BAD: mutates the input
function addItem(cart, item) {
  cart.push(item);
  return cart;
}

// GOOD: returns a new array
function addItem(cart, item) {
  return [...cart, item];
}
```

Spreading creates a copy. The original `cart` is untouched. This is how React state updates work — you always return a new object or array.

## What you learned

- A **pure function** has no side effects and always returns the same output for the same input.
- Side effects are necessary at boundaries — keep core logic pure.
- Use spread (`...`) to copy arrays/objects instead of mutating inputs.
