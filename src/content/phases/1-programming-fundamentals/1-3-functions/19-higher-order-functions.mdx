## Why this matters

A higher-order function (HOF) either takes a function as an argument or returns a function. You already used HOFs in the last lesson — `map`, `filter`, `reduce` are all higher-order. This lesson shows you how to write your own.

## Taking a function

```javascript
function applyToAll(items, transform) {
  const result = [];
  for (const item of items) {
    result.push(transform(item));
  }
  return result;
}

console.log(applyToAll([1, 2, 3], (n) => n * 10));
// [10, 20, 30]
```

`applyToAll` is a simple reimplementation of `map`. The `transform` parameter is a function — and `applyToAll` calls it once per item without knowing or caring what it does.

## Returning a function

```javascript
function multiplier(factor) {
  return (n) => n * factor;
}

const triple = multiplier(3);
const tenX = multiplier(10);

console.log(triple(5)); // 15
console.log(tenX(5)); // 50
```

`multiplier` is a **factory** — it produces specialized functions. Each returned function is a closure over `factor`.

## Composition

You can chain HOFs to build complex behavior from simple parts:

```javascript
const users = [
  { name: "Ada", age: 30 },
  { name: "Bob", age: 17 },
  { name: "Cal", age: 22 },
];

const adultNames = users
  .filter((u) => u.age >= 18)
  .map((u) => u.name)
  .sort();

console.log(adultNames); // ["Ada", "Cal"]
```

Each step is a small, testable transformation. The chain reads top-to-bottom like a pipeline.

## Writing your own: once()

```javascript
function once(fn) {
  let called = false;
  let result;
  return (...args) => {
    if (called) return result;
    called = true;
    result = fn(...args);
    return result;
  };
}

const init = once(() => {
  console.log("Initializing...");
  return 42;
});

console.log(init()); // "Initializing..." then 42
console.log(init()); // 42 (no log — fn not called again)
```

`once` is a real utility found in libraries like lodash. It wraps any function so it runs at most once.

## What you learned

- A **higher-order function** takes or returns a function. `map`, `filter`, `reduce` are built-in HOFs.
- **Factories** return specialized functions using closures over their parameters.
- Chaining HOFs creates readable, testable pipelines.
