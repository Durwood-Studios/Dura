## Why this matters

Network requests, database queries, and file I/O all take time. JavaScript is single-threaded — it cannot freeze the page while waiting. Promises and `async/await` are how JavaScript does one thing while waiting for another.

## The problem

```javascript
const data = fetch("/api/users"); // does NOT return data immediately
console.log(data); // Promise { <pending> }
```

`fetch` starts a network request and immediately returns a **Promise** — a placeholder for a value that will arrive later.

## Promises

A promise is an object with three states:

- **pending** — work in progress
- **fulfilled** — work done, value available
- **rejected** — work failed, error available

```javascript
fetch("/api/users")
  .then((response) => response.json())
  .then((users) => console.log(users))
  .catch((error) => console.error("Failed:", error));
```

`.then()` runs when the promise fulfills. `.catch()` runs when it rejects. They chain — each `.then` returns a new promise wrapping its return value.

## async/await — the modern way

`async/await` is syntactic sugar over promises. Same machinery, nicer syntax.

```javascript
async function loadUsers() {
  try {
    const response = await fetch("/api/users");
    const users = await response.json();
    console.log(users);
  } catch (error) {
    console.error("Failed:", error);
  }
}
```

`async` marks a function as returning a promise. `await` pauses the function until the promise settles, then gives you the resolved value. Error handling uses `try/catch` — the same pattern you already know.

## Key rules

1. `await` can only be used inside an `async` function.
2. An `async` function always returns a promise, even if you return a plain value.
3. If an awaited promise rejects and there is no `try/catch`, the error propagates as a rejected promise.

## Running things in parallel

```javascript
const [users, posts] = await Promise.all([
  fetch("/api/users").then((r) => r.json()),
  fetch("/api/posts").then((r) => r.json()),
]);
```

`Promise.all` starts both requests simultaneously and waits until both are done. Much faster than awaiting one, then the other.

## What you learned

- A **Promise** is a placeholder for a future value — pending, fulfilled, or rejected.
- `async/await` is sugar over `.then()/.catch()` — same thing, cleaner syntax.
- `Promise.all` runs multiple async operations in parallel.
