## Why this matters

"Call me back when you're done" — that is the callback pattern in one sentence. Event handlers, array methods, timers, and the entire Node.js ecosystem are built on it.

## What a callback is

A callback is a function you pass to another function, to be called later.

```javascript
function doTwice(fn) {
  fn();
  fn();
}

doTwice(() => console.log("Hi!"));
// "Hi!"
// "Hi!"
```

`doTwice` does not know what `fn` does. It just knows when to call it.

## Callbacks in array methods

```javascript
const nums = [1, 2, 3, 4, 5];

const doubled = nums.map((n) => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

const evens = nums.filter((n) => n % 2 === 0);
console.log(evens); // [2, 4]

const sum = nums.reduce((total, n) => total + n, 0);
console.log(sum); // 15
```

`map`, `filter`, and `reduce` each take a callback and call it once per element. The callback's job is different for each: transform, keep-or-discard, accumulate.

## Callbacks in timers

```javascript
setTimeout(() => {
  console.log("1 second later");
}, 1000);
```

`setTimeout` stores your callback and calls it after the delay. You do not call it yourself — the browser does.

## The callback pattern

```javascript
function fetchData(url, onSuccess, onError) {
  // pretend network call
  const ok = Math.random() > 0.3;
  if (ok) {
    onSuccess({ data: "hello" });
  } else {
    onError(new Error("Network failed"));
  }
}

fetchData(
  "/api",
  (result) => console.log(result.data),
  (err) => console.error(err.message)
);
```

Before promises, this was how every async operation worked. The caller provides a success callback and an error callback.

## What you learned

- A **callback** is a function you give to another function, which calls it at the right time.
- `map`, `filter`, and `reduce` are the essential array-callback trio.
- Before promises, callbacks were the only way to handle async operations.
