## Why this matters

The patterns below show up in real codebases, library source code, and interview questions. You do not need to use all of them daily, but you need to recognize them when you see them.

## IIFE: Immediately Invoked Function Expression

```javascript
(function () {
  const secret = "hidden";
  console.log(secret);
})();
```

The function is defined and called in one expression. Before modules existed, IIFEs were the only way to create a private scope. You still see them in build output and polyfills. Modern code uses modules instead.

## Currying

Currying transforms a function that takes multiple arguments into a series of functions that each take one.

```javascript
function add(a) {
  return (b) => a + b;
}

const add5 = add(5);
console.log(add5(3)); // 8
console.log(add5(10)); // 15
```

Useful when you want to partially apply some arguments now and the rest later. Libraries like Ramda are built on this pattern.

## Memoization

Cache the result of expensive computations so you do not repeat them.

```javascript
function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn(...args);
    cache.set(key, result);
    return result;
  };
}

const slowSquare = (n) => {
  console.log("computing...");
  return n * n;
};

const fastSquare = memoize(slowSquare);
console.log(fastSquare(5)); // "computing..." then 25
console.log(fastSquare(5)); // 25 (no log — cached)
```

React's `useMemo` and `React.memo` are forms of memoization.

## Debounce (preview)

You built a debounce in the utilities module. Here is the shape:

```javascript
function debounce(fn, delayMs) {
  let timer = null;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delayMs);
  };
}
```

The returned function delays execution until the caller stops calling. Used for search-as-you-type, window resize handlers, and form validation.

## What you learned

- **IIFE** creates a one-off private scope. Mostly replaced by modules.
- **Currying** splits a multi-argument function into a chain of single-argument functions.
- **Memoization** caches results to avoid redundant computation. React's `useMemo` uses this.
