## Why this matters

Functions are the single most important idea in programming. Every program you will ever read — small or large — is built out of functions calling other functions. Learning to write good functions is learning to write good programs.

## What a function does

A function takes input, does some work, and optionally returns output.

```javascript
function add(a, b) {
  return a + b;
}

console.log(add(3, 4)); // 7
```

`add` is the name. `a` and `b` are **parameters** — placeholders for the values you will pass in. `3` and `4` are the **arguments** — the actual values at call time. `return` sends a value back to the caller.

## Declaration vs expression

Two ways to create a function:

```javascript
// Function declaration — hoisted (available before its line)
function greet(name) {
  return `Hello, ${name}!`;
}

// Function expression — not hoisted
const greet2 = function (name) {
  return `Hello, ${name}!`;
};
```

Declarations are hoisted — you can call them before the line where they appear. Expressions are not. In practice both work; declarations are more common for top-level functions.

## Arrow functions

The modern shorthand:

```javascript
const double = (n) => n * 2;
console.log(double(5)); // 10
```

If the body is a single expression, the braces and `return` are optional. Parentheses around a single parameter are optional too: `n => n * 2`.

Arrow functions are the default for callbacks and short helpers. Regular `function` declarations are conventional for named top-level functions.

## Functions are values

In JavaScript, a function is a value like any other. You can assign it to a variable, pass it as an argument, or return it from another function.

```javascript
const ops = [Math.floor, Math.ceil, Math.round];
console.log(ops.map((fn) => fn(3.7))); // [3, 4, 4]
```

This is what makes JavaScript a "functions are first-class citizens" language. You will use this property constantly once you learn callbacks and higher-order functions.

## What you learned

- A function takes input (parameters), does work, and returns output.
- Declarations are hoisted; expressions and arrows are not.
- Functions are values — you can assign, pass, and return them.
