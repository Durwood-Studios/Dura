## Why this matters

A quiz app combines everything from Phase 1: arrays of objects for questions, control flow for scoring, functions for logic, and state tracking for progress. It is the capstone project that proves you can combine concepts.

## The data

```javascript
const questions = [
  {
    question: "What is 2 + 2?",
    options: ["3", "4", "5", "22"],
    correct: 1,
  },
  {
    question: "What does typeof null return?",
    options: ["'null'", "'undefined'", "'object'", "'boolean'"],
    correct: 2,
  },
  {
    question: "Which keyword declares a constant?",
    options: ["var", "let", "const", "define"],
    correct: 2,
  },
];
```

## The state

```javascript
let currentIndex = 0;
let score = 0;
```

## Core functions

```javascript
function getCurrentQuestion() {
  return questions[currentIndex] ?? null;
}

function submitAnswer(selectedIndex) {
  const q = getCurrentQuestion();
  if (!q) return { correct: false, done: true };
  const correct = selectedIndex === q.correct;
  if (correct) score++;
  currentIndex++;
  return {
    correct,
    done: currentIndex >= questions.length,
    score,
    total: questions.length,
  };
}

function getResults() {
  return {
    score,
    total: questions.length,
    percent: Math.round((score / questions.length) * 100),
  };
}
```

## Running the quiz

```javascript
console.log(submitAnswer(1)); // { correct: true, done: false, ... }
console.log(submitAnswer(0)); // { correct: false, done: false, ... }
console.log(submitAnswer(2)); // { correct: true, done: true, ... }
console.log(getResults()); // { score: 2, total: 3, percent: 67 }
```

## What you learned

- A quiz app is data (questions array) + state (index, score) + logic (submit, score, results).
- Separating data from logic lets you reuse both independently.
- Fisher-Yates shuffle randomizes an array without bias — a real algorithm you will use again.
