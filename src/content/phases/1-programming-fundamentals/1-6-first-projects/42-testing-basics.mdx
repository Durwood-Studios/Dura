## Why this matters

Tests are code that checks other code automatically. Without tests, every change is a gamble — did you break something? With tests, you run them after every change and know instantly. Every professional codebase has them. DURA's CI runs tests on every push.

## The simplest test

```javascript
function add(a, b) {
  return a + b;
}

// Test
function testAdd() {
  console.assert(add(2, 3) === 5, "2 + 3 should be 5");
  console.assert(add(-1, 1) === 0, "-1 + 1 should be 0");
  console.assert(add(0, 0) === 0, "0 + 0 should be 0");
  console.log("add: all tests passed");
}

testAdd();
```

`console.assert` logs the message only when the condition is false. No output means all passed. This is a crude but real test.

## What to test

1. **The happy path.** Does it work with normal input?
2. **Edge cases.** Empty arrays, zero, negative numbers, null.
3. **Error cases.** Does it throw or return an error when it should?

```javascript
function testDivide() {
  console.assert(divide(10, 2) === 5, "normal division");
  console.assert(divide(0, 5) === 0, "zero numerator");
  console.assert(divide(10, 0) === null, "division by zero returns null");
}
```

## The pattern: Arrange, Act, Assert

```javascript
function testToggleTodo() {
  // Arrange
  const todo = { id: 1, text: "test", done: false };

  // Act
  const toggled = { ...todo, done: !todo.done };

  // Assert
  console.assert(toggled.done === true, "should be done");
  console.assert(todo.done === false, "original unchanged");
}
```

Most tests follow this shape: set up the inputs, run the function, check the outputs.

## Real test frameworks

In real projects you use a framework like Vitest or Jest:

```javascript
import { test, expect } from "vitest";

test("add returns the sum", () => {
  expect(add(2, 3)).toBe(5);
});
```

The idea is the same: arrange, act, assert. The framework adds better error messages, automatic test discovery, and CI integration.

## What you learned

- Tests are code that checks other code. `console.assert` is the simplest form.
- The **Arrange, Act, Assert** pattern structures every test.
- Test the happy path, edge cases, and error cases. Real projects use Vitest or Jest.
