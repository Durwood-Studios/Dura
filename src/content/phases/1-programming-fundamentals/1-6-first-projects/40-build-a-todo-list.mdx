## Why this matters

A todo list is the "Hello World" of real applications. It exercises CRUD — Create, Read, Update, Delete — the four operations behind every app from Instagram to Jira. Build this once and you have the mental model for building anything.

## The data model

```javascript
let todos = [];
let nextId = 1;
```

Each todo is an object: `{ id, text, done }`. The array is the state. The `nextId` counter ensures unique IDs.

## Create

```javascript
function addTodo(text) {
  const todo = { id: nextId++, text, done: false };
  todos = [...todos, todo];
  return todo;
}
```

Spread to avoid mutating the original array (pure-function habit from lesson 20).

## Read

```javascript
function listTodos() {
  return todos;
}

function getTodo(id) {
  return todos.find((t) => t.id === id) ?? null;
}
```

## Update

```javascript
function toggleTodo(id) {
  todos = todos.map((t) => (t.id === id ? { ...t, done: !t.done } : t));
}
```

Map returns a new array. The spread on the matching todo creates a new object with the flipped `done`.

## Delete

```javascript
function removeTodo(id) {
  todos = todos.filter((t) => t.id !== id);
}
```

## Putting it together

```javascript
addTodo("Learn JavaScript");
addTodo("Build a todo list");
addTodo("Celebrate");
toggleTodo(2);
removeTodo(1);

for (const t of listTodos()) {
  const check = t.done ? "✓" : " ";
  console.log(`[${check}] ${t.id}: ${t.text}`);
}
```

## What you learned

- CRUD (Create, Read, Update, Delete) is the pattern behind every data-driven app.
- Use `map` for updates and `filter` for deletes — always return new arrays instead of mutating.
- A simple `{ id, text, done }` object is enough to model a todo. Start minimal.
