## Why this matters

A calculator is small enough to finish in one sitting and complex enough to exercise every concept from Phase 1: variables, functions, conditionals, error handling, and user input. This lesson walks through building one, step by step.

## Step 1: The core functions

```javascript
function add(a, b) {
  return a + b;
}
function subtract(a, b) {
  return a - b;
}
function multiply(a, b) {
  return a * b;
}
function divide(a, b) {
  if (b === 0) return "Error: division by zero";
  return a / b;
}
```

Each function is pure, testable, and handles its own edge case.

## Step 2: The dispatcher

```javascript
function calculate(a, op, b) {
  switch (op) {
    case "+":
      return add(a, b);
    case "-":
      return subtract(a, b);
    case "*":
      return multiply(a, b);
    case "/":
      return divide(a, b);
    default:
      return "Error: unknown operator";
  }
}
```

One function to rule them all. The caller provides the operator as a string.

## Step 3: Input validation

```javascript
function parseInput(input) {
  const num = Number(input);
  if (Number.isNaN(num)) return null;
  return num;
}
```

Never trust user input. Parse it, validate it, then use it.

## Step 4: Putting it together

```javascript
function run(aStr, opStr, bStr) {
  const a = parseInput(aStr);
  const b = parseInput(bStr);
  if (a === null || b === null) return "Error: invalid number";
  return calculate(a, opStr, b);
}

console.log(run("10", "+", "5")); // 15
console.log(run("10", "/", "0")); // "Error: division by zero"
console.log(run("abc", "+", "5")); // "Error: invalid number"
```

## What you learned

- Build small, pure functions for each operation. Test them individually.
- A dispatcher (switch/object map) routes user input to the right function.
- Validate input before using it. Handle edge cases (division by zero, bad input).
