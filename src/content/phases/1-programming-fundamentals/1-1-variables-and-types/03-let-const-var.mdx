## Why this matters

JavaScript has three keywords for declaring variables: `const`, `let`, and `var`. Modern code uses `const` by default and `let` when reassignment is needed. The third, `var`, is a historical leftover with sharp edges. Knowing why lets you read old code and write new code correctly.

## const — the default

`const` declares a variable that cannot be reassigned after its initial value.

```javascript
const maxScore = 100;
maxScore = 200; // TypeError: Assignment to constant variable
```

Use `const` unless you know you need reassignment. About 90% of the variables in real codebases can be `const`. Making this the default prevents an entire category of bugs where a variable changes when you did not mean it to.

## let — when you need to reassign

`let` declares a variable that can be reassigned.

```javascript
let count = 0;
count = count + 1;
count = count + 1;
console.log(count); // 2
```

Counters, accumulators, loop variables, and values that genuinely change over time are the right use for `let`.

## var — don't use it

`var` is the original JavaScript declaration keyword. It still works, but it has two behaviors that cause bugs:

1. **Function-scoped, not block-scoped.** A `var` declared inside an `if` or `for` is visible to the entire surrounding function — not just the block.
2. **Hoisted with `undefined`.** A `var` name exists from the start of its function, even before the `var` line. Reading it early gives `undefined` instead of an error.

```javascript
function demo() {
  console.log(x); // undefined  ← no error!
  var x = 10;
}
```

Compare with `let`:

```javascript
function demo() {
  console.log(x); // ReferenceError: Cannot access 'x' before initialization
  let x = 10;
}
```

The `let` version is loud. That is what you want. Read `var` in old code, write `let` or `const` in new code.

## Scope in one sentence

`let` and `const` are **block-scoped** — they exist only inside the `{}` block where they were declared. `var` is **function-scoped** — it exists everywhere in the enclosing function.

```javascript
{
  const inside = 1;
}
console.log(inside); // ReferenceError
```

Block scoping is nearly always what you want. It keeps variables close to where they are used and prevents accidents.

## What you learned

- `const` by default. `let` when you need reassignment. Avoid `var`.
- `let` and `const` are block-scoped; `var` is function-scoped and hoisted as `undefined`.
- Block scoping keeps variables close to where they are used and makes mistakes loud.
