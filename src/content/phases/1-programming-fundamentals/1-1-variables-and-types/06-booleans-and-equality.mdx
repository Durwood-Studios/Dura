## Why this matters

Every `if` statement, every loop condition, every filter callback — all of them come down to a single boolean: `true` or `false`. If you do not understand how JavaScript decides which values are true and which are false, your conditions will surprise you.

## The boolean type

There are exactly two boolean values: `true` and `false`.

```javascript
console.log(typeof true); // "boolean"
console.log(typeof false); // "boolean"
```

## Comparison operators

Comparisons produce booleans.

```javascript
console.log(5 > 3); // true
console.log(5 < 3); // false
console.log(5 >= 5); // true
console.log(5 <= 4); // false
```

## Strict equality: === and !==

```javascript
console.log(5 === 5); // true
console.log(5 === "5"); // false — different types
console.log(5 !== "5"); // true
```

`===` checks value **and** type. This is the operator you should use. Always.

## Loose equality: == and != (avoid)

```javascript
console.log(5 == "5"); // true — JavaScript coerces the string to a number
console.log(null == undefined); // true
console.log(0 == false); // true
```

`==` applies type coercion before comparing. The rules are complex, surprising, and the source of real bugs. Do not use `==` unless you have a very specific reason (and you will not have one in this phase).

**Rule: use `===` and `!==` everywhere.**

## Truthy and falsy

JavaScript coerces values to boolean in contexts that expect one (like `if`). There are exactly six **falsy** values. Everything else is **truthy**.

Falsy: `false`, `0`, `""` (empty string), `null`, `undefined`, `NaN`.

```javascript
if (0) {
  console.log("nope"); // never runs — 0 is falsy
}

if ("hello") {
  console.log("yes"); // runs — non-empty string is truthy
}
```

You can explicitly convert to boolean with `Boolean(value)` or the double-bang `!!value`.

## Logical operators

```javascript
console.log(true && false); // false — AND
console.log(true || false); // true  — OR
console.log(!true); // false — NOT
```

`&&` returns the first falsy operand or the last value. `||` returns the first truthy operand or the last value. This short-circuit behavior powers common patterns like `name || "Anonymous"`.

## What you learned

- Booleans are `true` or `false`. Comparisons produce booleans.
- `===` checks type and value — use it everywhere. Avoid `==`.
- Six falsy values: `false`, `0`, `""`, `null`, `undefined`, `NaN`. Everything else is truthy.
