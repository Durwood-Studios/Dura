## Why this matters

Every program you will ever write manipulates **values**. The number `42`. The text `"hello"`. The boolean `true`. These are the atoms. Everything else — variables, functions, whole applications — is just ways of combining, moving, and transforming values.

Before you can use them, you need to know what kinds of values exist and what kind you are holding. That is what a **type** tells you.

## What a value is

A value is a piece of data the computer can hold. In JavaScript you can drop one into the console and it will print itself back.

```javascript
42;
("hello");
true;
null;
```

Each of those is a value. They are not instructions yet. They do not do anything on their own. But you can ask about them, combine them, and save them.

## What a type is

A type is the category a value belongs to. Same shape, same rules.

JavaScript has seven built-in types you will use constantly:

- **number** — `42`, `-3.14`, `0`
- **string** — `"hello"`, `'world'`
- **boolean** — `true`, `false`
- **null** — the deliberate absence of a value
- **undefined** — a value that was never set
- **object** — a collection of named values
- **function** — a value you can call

You can ask any value what type it is with `typeof`:

```javascript
console.log(typeof 42); // "number"
console.log(typeof "hello"); // "string"
console.log(typeof true); // "boolean"
```

## Why types matter

Types decide what you can do with a value. You can add two numbers. You can join two strings. You cannot add a function to a boolean in any way that makes sense.

```javascript
console.log(2 + 3); // 5
console.log("2" + "3"); // "23"  ← string concatenation, not math
console.log(2 + "3"); // "23"  ← JavaScript coerces the number to a string
```

That last line is **type coercion** — JavaScript silently converting one type to another to make an operation work. It is a source of real bugs. Phase 1 Module 5 (Debugging) is partly about spotting when coercion has bitten you.

## What you learned

- A **value** is a piece of data; a **type** is the category that value belongs to.
- JavaScript has seven built-in types — number, string, boolean, null, undefined, object, function.
- **Type coercion** is JavaScript converting types behind your back to make an expression work. Watch for it.
