## Why this matters

A program that cannot remember anything is not much of a program. **Variables** are how a program remembers: you give a value a name, and later you ask for that name and get the value back. That is the whole idea. Every program you will ever write is built on it.

## The basic shape

```javascript
const greeting = "Hello, world.";
console.log(greeting);
```

Line 1 creates a variable named `greeting` and stores the string `"Hello, world."` in it. Line 2 looks the name up and prints what is there.

The `=` is not a math equals sign — it is **assignment**. It means "take the value on the right and put it in the name on the left."

## Declare before you use

In JavaScript you have to declare a variable before you can use it. Declaration is the part with `const`, `let`, or `var`.

```javascript
const name = "Dustin";
let age = 37;
```

Try to use an undeclared name and JavaScript throws an error.

```javascript
console.log(x); // ReferenceError: x is not defined
```

Good languages make this mistake loud. JavaScript is one of them — as long as you use strict mode, which modern tooling does by default.

## Reassignment

A `let` variable can be given a new value later.

```javascript
let score = 0;
score = score + 10;
console.log(score); // 10
```

Read that second line carefully: "take the current value of `score`, add 10, store the result back in `score`." The right side is evaluated first, then the assignment happens. This is how counters, totals, and accumulators work.

A `const` variable cannot be reassigned. More on that next lesson.

## Naming rules

Names in JavaScript:

- Start with a letter, `_`, or `$`.
- Contain letters, digits, `_`, `$`.
- Are case-sensitive (`age` and `Age` are different).
- Cannot be reserved words (`if`, `function`, `class`, etc.).

Naming convention: **camelCase** for variables and functions. `userName`, `totalPrice`, `isActive`. Not `user_name` or `UserName`.

A good name describes **what** the value means, not how it is stored. `count` is better than `i`. `totalPrice` is better than `x`.

## What you learned

- A **variable** is a name bound to a value. Declare with `const` or `let`, assign with `=`.
- Reading an undeclared name throws a `ReferenceError`.
- Use **camelCase** and pick names that describe what the value means.
