## Why this matters

Every counter, price, score, timer, pixel coordinate, and loop index in a program is a number. JavaScript makes numbers look simple — there is only one number type. That simplicity hides a few sharp edges you will hit eventually.

## One type for everything

Unlike languages that separate integers from decimals, JavaScript has a single `number` type. Under the hood it is a 64-bit floating-point value (IEEE 754).

```javascript
console.log(typeof 42); // "number"
console.log(typeof 3.14); // "number"
console.log(typeof -7); // "number"
```

This means you can mix integers and decimals freely. It also means floating-point math applies to everything.

## Arithmetic

The basic operators work like a calculator:

```javascript
console.log(10 + 3); // 13
console.log(10 - 3); // 7
console.log(10 * 3); // 30
console.log(10 / 3); // 3.3333...
console.log(10 % 3); // 1   (remainder)
console.log(10 ** 3); // 1000 (exponent)
```

`%` is the **remainder** (modulo) operator. It tells you what is left over after division. Useful for checking even/odd, cycling through a list, or formatting clocks.

## The floating-point trap

```javascript
console.log(0.1 + 0.2); // 0.30000000000000004
```

This is not a JavaScript bug. It is how every language that uses IEEE 754 floating-point works. `0.1` cannot be represented exactly in binary, just like `1/3` cannot be represented exactly in decimal.

**Rule of thumb:** never compare floating-point results with `===`. Instead compare within a tolerance: `Math.abs(a - b) < 0.0001`. For money, use integers (cents, not dollars) or a dedicated library.

## Special number values

```javascript
console.log(1 / 0); // Infinity
console.log(-1 / 0); // -Infinity
console.log("abc" * 2); // NaN  — "Not a Number"
console.log(typeof NaN); // "number"  — yes, really
```

`NaN` is the result of math that does not make sense. The joke is that `typeof NaN` is `"number"`. Check for it with `Number.isNaN(value)`.

## Parsing strings to numbers

```javascript
console.log(Number("42")); // 42
console.log(parseInt("42px")); // 42 — stops at the first non-digit
console.log(parseFloat("3.14 degrees")); // 3.14
console.log(Number("hello")); // NaN
```

## What you learned

- JavaScript has one `number` type for integers and decimals alike (IEEE 754 float).
- `%` gives the remainder; `**` gives the exponent.
- `0.1 + 0.2 !== 0.3` — floating-point rounding is real. Compare within a tolerance or use integers for money.
