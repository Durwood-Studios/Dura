## Why this matters

Strings are everywhere: usernames, error messages, HTML content, file paths, API responses, log lines. After numbers, strings are the type you will manipulate most.

## Creating strings

Three syntaxes, one type.

```javascript
const single = "hello";
const double = "hello";
const template = `hello`;
```

Single and double quotes are interchangeable. Template literals (backticks) are special — they allow **interpolation** and multi-line strings.

```javascript
const name = "Dustin";
console.log(`Hello, ${name}!`); // "Hello, Dustin!"
```

The `${...}` syntax inside backticks evaluates the expression and inserts the result. This replaces clunky concatenation (`"Hello, " + name + "!"`).

## Strings are immutable

Once a string is created, you cannot change its characters in place. Every string operation returns a **new** string.

```javascript
const original = "hello";
const upper = original.toUpperCase();
console.log(original); // "hello"  — unchanged
console.log(upper); // "HELLO"  — new string
```

This matters for performance with very large strings, but for most work it is simply how strings behave.

## Length and access

```javascript
const word = "DURA";
console.log(word.length); // 4
console.log(word[0]); // "D"
console.log(word[3]); // "A"
console.log(word.at(-1)); // "A" — .at() supports negative indexes
```

Indexes are zero-based. The first character is at index 0, not 1. This convention appears everywhere in programming.

## Common methods

```javascript
const msg = "  Hello, World!  ";
console.log(msg.trim()); // "Hello, World!"
console.log(msg.includes("World")); // true
console.log(msg.indexOf("World")); // 9
console.log(msg.slice(2, 7)); // "Hello"
console.log(msg.replace("World", "DURA")); // "  Hello, DURA!  "
console.log("a-b-c".split("-")); // ["a", "b", "c"]
```

These return new values. They never modify the original string.

## What you learned

- Strings are created with quotes or backticks; template literals allow `${expression}` interpolation.
- Strings are **immutable** — every method returns a new string.
- Indexes are zero-based. Common methods: `length`, `trim`, `includes`, `indexOf`, `slice`, `split`, `replace`.
