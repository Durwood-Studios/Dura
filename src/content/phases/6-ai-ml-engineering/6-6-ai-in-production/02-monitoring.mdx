## Why this matters

An AI service that works today can silently degrade tomorrow. Model providers change behavior without warning. Prompts that cost $0.02 start costing $0.20 after a context window grows. Latency spikes at 3 AM go unnoticed until users complain on Monday. Without monitoring, you are flying blind over expensive, unpredictable infrastructure. Good observability turns "something feels slow" into "p95 latency increased 40% after the prompt change at 2:14 PM."

## What to measure

AI services need the same metrics as any backend service — plus a few unique ones. Four categories cover the essentials.

```javascript
// Core metrics to track for every AI request
const AI_METRICS = {
  // Latency — how long did inference take?

  // Cost — how much did this request cost?

  volume: {
    requestsPerMinute: "Total throughput",
    activeUsers: "Unique users making AI requests",
    queueDepth: "Jobs waiting to be processed",
  },
};
```

## Latency percentiles

Averages hide problems. If your average latency is 800ms but 5% of requests take 8 seconds, your users experience is far worse than the average suggests. Track percentiles.

```javascript
class LatencyTracker {
  constructor() {
    this.samples = [];
  }

  record(durationMs) {
    this.samples.push(durationMs);
    // Keep a rolling window to avoid unbounded memory
    if (this.samples.length > 10000) {
      this.samples = this.samples.slice(-10000);
    }
  }

  percentile(p) {
    if (this.samples.length === 0) return 0;
    const sorted = [...this.samples].sort((a, b) => a - b);
    const index = Math.ceil((p / 100) * sorted.length) - 1;
    return sorted[Math.max(0, index)];
  }

  report() {
    return {
      p50: this.percentile(50), // Median — the "typical" experience
      p95: this.percentile(95), // Bad but not rare
      p99: this.percentile(99), // The worst 1% of users see this
      count: this.samples.length,
    };
  }
}

// Usage
const tracker = new LatencyTracker();

async function timedInference(prompt, modelClient) {
  const start = Date.now();
  try {
    const result = await modelClient.complete(prompt);
    return result;
  } finally {
    tracker.record(Date.now() - start);
  }
}
```

Set alerts on p95 and p99, not on averages. A p95 alert at 5 seconds catches problems long before the average moves.

## Token usage and cost tracking

Every token costs money. Track spending per user, per model, and per feature so you know where the budget goes.

```javascript
class CostTracker {
  constructor(pricing) {
    // Pricing per 1K tokens
    this.pricing = pricing;
    this.usage = [];
  }

  record(entry) {
    const inputCost = (entry.inputTokens / 1000) * this.pricing.input;
    const outputCost = (entry.outputTokens / 1000) * this.pricing.output;
    this.usage.push({
      timestamp: Date.now(),
      userId: entry.userId,
      feature: entry.feature,
      model: entry.model,
      inputTokens: entry.inputTokens,
      outputTokens: entry.outputTokens,
      cost: inputCost + outputCost,
    });
  }

  costByFeature(since) {
    const recent = this.usage.filter((u) => u.timestamp >= since);
    const byFeature = {};
    for (const entry of recent) {
      byFeature[entry.feature] = (byFeature[entry.feature] || 0) + entry.cost;
    }
    return byFeature;
  }

  dailyCost() {
    const oneDayAgo = Date.now() - 86400000;
    return this.usage.filter((u) => u.timestamp >= oneDayAgo).reduce((sum, u) => sum + u.cost, 0);
  }
}

const costs = new CostTracker({ input: 0.0015, output: 0.002 });

// After each model call
costs.record({
  userId: "user_123",
  feature: "code-review",
  model: "gpt-4o-mini",
  inputTokens: 1200,
  outputTokens: 450,
});
```

## Logging without PII

AI request logs are invaluable for debugging but dangerous if they contain personal information. Strip PII before logging.

```javascript
function sanitizeForLogging(request, response) {
  return {
    requestId: request.id,
    timestamp: new Date().toISOString(),
    model: request.model,
    // Log prompt length and hash, NOT the actual prompt
    promptLength: request.prompt.length,
    promptHash: hashString(request.prompt),
    // Log metadata, not content
    responseLength: response.text.length,
    inputTokens: response.usage.inputTokens,
    outputTokens: response.usage.outputTokens,
    latencyMs: response.latencyMs,
    status: response.error ? "error" : "success",
    // Never log: user email, prompt text, response text, IP addresses
  };
}

function hashString(str) {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    const char = str.charCodeAt(i);
    hash = ((hash << 5) - hash + char) | 0;
  }
  return hash.toString(16);
}
```

If you must store full prompts for debugging, encrypt them at rest and set automatic deletion after 7-30 days. Never log them to plaintext files on shared infrastructure.

## Alerting rules

Alerts should be actionable. If an alert fires and the on-call engineer cannot act on it, remove it.

```javascript
const ALERT_RULES = [
  {
    name: "high-p95-latency",
    condition: (metrics) => metrics.latency.p95 > 5000,
    message: "AI inference p95 latency exceeded 5 seconds",
    severity: "warning",
  },
  {
    name: "error-rate-spike",
    condition: (metrics) => metrics.errorRate > 0.05,
    message: "AI error rate above 5%",
    severity: "critical",
  },
  {
    name: "daily-cost-budget",
    condition: (metrics) => metrics.dailyCost > 100,
    message: "Daily AI spend exceeded $100 budget",
    severity: "warning",
  },
  {
    name: "queue-depth",
    condition: (metrics) => metrics.queueDepth > 500,
    message: "AI job queue depth above 500 — workers may be stuck",
    severity: "critical",
  },
];

function checkAlerts(metrics) {
  const fired = [];
  for (const rule of ALERT_RULES) {
    if (rule.condition(metrics)) {
      fired.push({ name: rule.name, message: rule.message, severity: rule.severity });
    }
  }
  return fired;
}
```

## What you learned

- Production AI monitoring requires tracking latency percentiles (p50/p95/p99), token costs, error rates, and queue depth — not just averages
- Log AI request metadata (token counts, latency, hashes) instead of raw prompt/response content to avoid PII exposure
- Alert rules should be actionable and tied to specific thresholds that indicate real user impact or budget risk
