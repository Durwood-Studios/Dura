## Why this matters

Every AI request costs money. A single user running a script in a loop can burn through your monthly budget in hours. A viral moment can send traffic from 100 requests per minute to 10,000. Without rate limiting, you have no control over costs, no protection against abuse, and no way to guarantee fair access. Rate limiting is not optional — it is the seatbelt between your AI service and financial disaster.

## Token budgets

API rate limits typically count requests, but AI services should also count tokens. A user sending 10 requests of 50 tokens each is very different from 10 requests of 4,000 tokens each.

```javascript
class TokenBudget {
  constructor(limits) {
    this.limits = limits; // { dailyTokens, monthlyTokens }
    this.usage = new Map(); // userId → { daily, monthly, lastReset }
  }

  getUsage(userId) {
    if (!this.usage.has(userId)) {
      this.usage.set(userId, {
        daily: 0,
        monthly: 0,
        dayStart: this.startOfDay(),
        monthStart: this.startOfMonth(),
      });
    }
    const record = this.usage.get(userId);

    // Reset counters if the window has rolled over
    const now = Date.now();
    if (now >= record.dayStart + 86400000) {
      record.daily = 0;
      record.dayStart = this.startOfDay();
    }
    if (now >= record.monthStart + 30 * 86400000) {
      record.monthly = 0;
      record.monthStart = this.startOfMonth();
    }
    return record;
  }

  canSpend(userId, estimatedTokens) {
    const usage = this.getUsage(userId);
    return (
      usage.daily + estimatedTokens <= this.limits.dailyTokens &&
      usage.monthly + estimatedTokens <= this.limits.monthlyTokens
    );
  }

  spend(userId, actualTokens) {
    const usage = this.getUsage(userId);
    usage.daily += actualTokens;
    usage.monthly += actualTokens;
  }

  remaining(userId) {
    const usage = this.getUsage(userId);
    return {
      daily: Math.max(0, this.limits.dailyTokens - usage.daily),
      monthly: Math.max(0, this.limits.monthlyTokens - usage.monthly),
    };
  }

  startOfDay() {
    const d = new Date();
    d.setHours(0, 0, 0, 0);
    return d.getTime();
  }

  startOfMonth() {
    const d = new Date();
    d.setDate(1);
    d.setHours(0, 0, 0, 0);
    return d.getTime();
  }
}
```

Estimate tokens before the request (prompt length / 4 is a rough approximation for English text), then update with the actual count from the model response.

## Leaky bucket vs token bucket

Two classic algorithms power most rate limiters. They solve the same problem differently.

**Token bucket** allows bursts. Tokens accumulate at a steady rate up to a maximum. Each request consumes tokens. If the bucket is empty, the request is rejected.

```javascript
class TokenBucketLimiter {
  constructor(capacity, refillRate) {
    this.capacity = capacity; // Max tokens in the bucket
    this.refillRate = refillRate; // Tokens added per second
    this.buckets = new Map(); // userId → { tokens, lastRefill }
  }

  allow(userId, cost = 1) {
    const now = Date.now();
    let bucket = this.buckets.get(userId);

    if (!bucket) {
      bucket = { tokens: this.capacity, lastRefill: now };
      this.buckets.set(userId, bucket);
    }

    // Refill based on elapsed time
    const elapsed = (now - bucket.lastRefill) / 1000;
    bucket.tokens = Math.min(this.capacity, bucket.tokens + elapsed * this.refillRate);
    bucket.lastRefill = now;

    // Check if enough tokens are available
    if (bucket.tokens >= cost) {
      bucket.tokens -= cost;
      return { allowed: true, remaining: Math.floor(bucket.tokens) };
    }

    const retryAfter = Math.ceil((cost - bucket.tokens) / this.refillRate);
    return { allowed: false, retryAfter };
  }
}
```

**Leaky bucket** enforces a steady rate. Requests enter a queue that drains at a fixed rate. If the queue is full, new requests are rejected. No bursts.

```javascript
class LeakyBucketLimiter {
  constructor(capacity, drainRate) {
    this.capacity = capacity; // Max queue size
    this.drainRate = drainRate; // Requests drained per second
    this.buckets = new Map();
  }

  allow(userId) {
    const now = Date.now();
    let bucket = this.buckets.get(userId);

    if (!bucket) {
      bucket = { water: 0, lastDrain: now };
      this.buckets.set(userId, bucket);
    }

    // Drain based on elapsed time
    const elapsed = (now - bucket.lastDrain) / 1000;
    bucket.water = Math.max(0, bucket.water - elapsed * this.drainRate);
    bucket.lastDrain = now;

    // Try to add one unit
    if (bucket.water < this.capacity) {
      bucket.water += 1;
      return { allowed: true, queueLevel: bucket.water };
    }

    return { allowed: false, retryAfter: Math.ceil(1 / this.drainRate) };
  }
}
```

Use token bucket when bursts are acceptable (interactive chat). Use leaky bucket when you need smooth, predictable throughput (batch processing pipelines).

## Graceful degradation

When a user hits their limit, do not just return a 429 and walk away. Give them useful information and, when possible, a degraded experience rather than nothing.

```javascript
async function handleWithDegradation(req, limits, models) {
  const userId = req.user.id;
  const prompt = req.body.prompt;
  const estimatedTokens = Math.ceil(prompt.length / 4);

  // Tier 1: Full access
  if (limits.canSpend(userId, estimatedTokens)) {
    const result = await models.primary.complete(prompt);
    limits.spend(userId, result.usage.totalTokens);
    return { status: 200, body: result, tier: "full" };
  }

  // Tier 2: Cheaper model with reduced output
  if (limits.remaining(userId).daily > 500) {
    const result = await models.cheap.complete(prompt, { maxTokens: 256 });
    limits.spend(userId, result.usage.totalTokens);
    return {
      status: 200,
      body: result,
      tier: "reduced",
      warning: "Using a smaller model due to usage limits",
    };
  }

  // Tier 3: Cached/static response only
  const cached = await getCachedResponse(prompt);
  if (cached) {
    return { status: 200, body: cached, tier: "cached" };
  }

  // Tier 4: Reject with helpful details
  const remaining = limits.remaining(userId);
  return {
    status: 429,
    body: {
      error: "Token budget exhausted",
      daily: remaining.daily,
      monthly: remaining.monthly,
      resetsAt: new Date(limits.getUsage(userId).dayStart + 86400000).toISOString(),
    },
  };
}
```

The best rate-limited APIs tell you exactly when you can retry and how much budget remains.

## What you learned

- Token budgets track AI spending per user at daily and monthly granularity, providing accurate cost control beyond simple request counting
- Token bucket and leaky bucket algorithms offer different tradeoffs: bursts vs steady throughput
- Graceful degradation (cheaper models, cached responses, informative 429s) preserves user experience when limits are reached
