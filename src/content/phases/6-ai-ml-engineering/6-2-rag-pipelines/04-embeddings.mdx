## Why this matters

You cannot ctrl-F your way through 50,000 document chunks. Keyword search fails when the user says "refund" but the document says "return policy." Embeddings solve this by converting text into numerical vectors where **similar meanings land near each other** in vector space. "How do I get my money back?" and "What is the return policy?" produce vectors that are close together — even though they share zero keywords.

## What is an embedding?

An embedding is a list of numbers (a vector) that represents the meaning of a piece of text. A modern embedding model like OpenAI's `text-embedding-3-small` produces a 1536-dimensional vector. Each dimension captures some aspect of meaning — though no single dimension has an interpretable label.

```javascript
// Conceptual: what embeddings look like
const embedding1 = [0.023, -0.041, 0.118 /* ...1533 more numbers */];
const embedding2 = [0.019, -0.038, 0.121 /* ...1533 more numbers */];

// These two vectors are "close" if their texts have similar meaning
```

You do not train embedding models yourself. You call an API (OpenAI, Cohere, Voyage) or run an open-source model (Sentence Transformers, Nomic). The model was pre-trained on massive text corpora to learn that "dog" is closer to "puppy" than to "database."

## Cosine similarity

How do you measure "closeness" between two vectors? The standard metric is **cosine similarity**: the cosine of the angle between two vectors. It ranges from -1 (opposite) to 1 (identical), with 0 meaning unrelated.

```javascript
function cosineSimilarity(a, b) {
  let dotProduct = 0;
  let normA = 0;
  let normB = 0;

  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }

  return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}

// Example with small vectors for illustration
const refund = [0.9, 0.1, 0.3];
const returnPolicy = [0.85, 0.15, 0.28];
const shipping = [0.1, 0.8, 0.2];

console.log(cosineSimilarity(refund, returnPolicy)); // ~0.99 (very similar)
console.log(cosineSimilarity(refund, shipping)); // ~0.45 (different topics)
```

Cosine similarity ignores vector magnitude and focuses on direction. This is important because it means a short sentence and a long paragraph about the same topic will still be close — their directions align even if their magnitudes differ.

**Dimensionality** is the length of the vector. Higher dimensions capture more nuance but cost more storage and compute. For most RAG systems, 768-1536 dimensions is sufficient.

## Calling an embedding API

In production, you batch your chunks and send them to an embedding API:

```javascript
async function embedTexts(texts, apiKey) {
  const response = await fetch("https://api.openai.com/v1/embeddings", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${apiKey}`,
    },
    body: JSON.stringify({
      model: "text-embedding-3-small",
      input: texts, // Array of strings — batch embedding
    }),
  });

  if (!response.ok) {
    throw new Error(`Embedding API error: ${response.status}`);
  }

  const data = await response.json();
  return data.data.map((item) => item.embedding);
}

// Batch embedding — always batch, never embed one at a time
const chunks = ["How to reset password", "Refund policy details", "Shipping info"];
// const embeddings = await embedTexts(chunks, process.env.OPENAI_API_KEY);
```

### Batch embedding matters

Embedding one chunk at a time means one HTTP request per chunk. For 10,000 chunks, that is 10,000 round trips. Batch embedding sends hundreds of chunks per request, reducing latency by 100x. Most APIs accept batches of 100-2000 texts.

```javascript
async function batchEmbed(chunks, batchSize = 100) {
  const allEmbeddings = [];

  for (let i = 0; i < chunks.length; i += batchSize) {
    const batch = chunks.slice(i, i + batchSize);
    const embeddings = await embedTexts(batch);
    allEmbeddings.push(...embeddings);

    // Respect rate limits
    if (i + batchSize < chunks.length) {
      await new Promise((resolve) => setTimeout(resolve, 200));
    }
  }

  return allEmbeddings;
}
```

### Key engineering decisions

1. **Same model for indexing and querying.** If you embed documents with `text-embedding-3-small`, you must embed queries with the same model. Vectors from different models are incompatible.
2. **Normalize or not.** Some models return normalized vectors (magnitude = 1). If yours does not, normalize before storing — it makes cosine similarity equivalent to dot product, which is faster.
3. **Dimensionality reduction.** OpenAI's v3 models support a `dimensions` parameter to reduce output size. Dropping from 1536 to 512 dimensions saves 66% storage with modest quality loss.

## What you learned

- Embeddings convert text into numerical vectors where similar meanings produce nearby vectors, enabling semantic search beyond keyword matching.
- Cosine similarity measures vector closeness by direction (not magnitude), ranging from -1 to 1.
- Always batch embed (hundreds per request), use the same model for indexing and querying, and choose dimensionality based on your quality-vs-cost tradeoff.
