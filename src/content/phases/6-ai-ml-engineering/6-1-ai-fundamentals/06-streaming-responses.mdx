## Why this matters

A non-streaming AI response makes the user stare at a spinner for 5-15 seconds. A streaming response shows the first word in under a second. Same total time, completely different experience. Time-to-first-token is the UX metric that matters most.

## How streaming works

Instead of waiting for the full response, the API sends **Server-Sent Events** (SSE) — small chunks of JSON, each containing a few tokens.

```text
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"A "}}
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"closure "}}
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"is "}}
data: {"type":"message_stop"}
```

Each `data:` line is one event. You read them as they arrive and append text to the UI.

## Consuming a stream in JavaScript

```javascript
async function streamResponse(url, body, apiKey) {
  const response = await fetch(url, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": apiKey,
      "anthropic-version": "2023-06-01",
    },
    body: JSON.stringify({ ...body, stream: true }),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let fullText = "";

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value, { stream: true });
    // Parse SSE events from chunk
    const lines = chunk.split("\n");
    for (const line of lines) {
      if (!line.startsWith("data: ")) continue;
      const json = line.slice(6);
      if (json === "[DONE]") break;
      try {
        const event = JSON.parse(json);
        if (event.delta?.text) {
          fullText += event.delta.text;
          // Update UI here — append to a DOM element
        }
      } catch {
        /* skip non-JSON lines */
      }
    }
  }

  return fullText;
}
```

The key objects: `ReadableStream`, `getReader()`, `TextDecoder`. The stream delivers raw bytes; you decode them, split on newlines, parse the SSE events, and extract the text deltas.

## Building a streaming UI

The pattern for any streaming UI:

1. Show a typing indicator or cursor immediately.
2. As each delta arrives, append it to the displayed text.
3. On completion, finalize the message and enable actions (copy, share).
4. On error mid-stream, show what you have + an error message.

## Error handling mid-stream

Streams can break partway through — network errors, API timeouts, context overflow. Always:

- Wrap the reader loop in try/catch.
- Save partial output so the user sees what was generated.
- Show a clear "generation interrupted" message.

## What you learned

- Streaming shows tokens as they arrive, reducing perceived latency. Time-to-first-token is the key metric.
- SSE delivers `data:` lines with JSON deltas. Parse with ReadableStream + TextDecoder.
- Always handle mid-stream errors: save partial output and show a clear interruption message.
