## Why this matters

"Return JSON" in a prompt works most of the time. Most of the time is not good enough for production. Structured outputs — via tool use, JSON mode, and schema validation — turn "usually works" into "always works or fails cleanly."

## The problem with "just ask for JSON"

```text
Prompt: "Extract the name and age. Return JSON."
Response: "Here's the JSON: {name: "Ada", age: 30}"
```

Three problems: the model added preamble text, used unquoted keys, and you have no guarantee the shape matches your code's expectations.

## Tool use / function calling

The reliable solution. You define a **tool** with a JSON Schema, and the model is constrained to call it:

```javascript
const tools = [
  {
    name: "extract_person",
    description: "Extract person info from text",
    input_schema: {
      type: "object",
      properties: {
        name: { type: "string", description: "Full name" },
        age: { type: "integer", description: "Age in years" },
        email: { type: "string", description: "Email if mentioned" },
      },
      required: ["name"],
    },
  },
];
```

The model responds with a `tool_use` content block containing structured JSON that matches your schema. No preamble, no markdown wrapping.

## Anthropic tool_use response

```javascript
// Response content block:
{
  type: "tool_use",
  id: "toolu_abc123",
  name: "extract_person",
  input: {
    name: "Ada Lovelace",
    age: 36,
    email: null
  }
}
```

## Always validate

Even with tool use, validate the output. Models can produce valid JSON that violates business rules (negative ages, empty required fields, wrong enum values).

```javascript
function validatePerson(input) {
  if (!input.name || input.name.trim().length === 0) {
    return { valid: false, error: "Name is required" };
  }
  if (input.age !== undefined && (input.age < 0 || input.age > 150)) {
    return { valid: false, error: "Age out of range" };
  }
  return { valid: true, data: input };
}
```

In production, use Zod:

```javascript
import { z } from "zod";

const PersonSchema = z.object({
  name: z.string().min(1),
  age: z.number().int().min(0).max(150).optional(),
  email: z.string().email().nullable(),
});

const result = PersonSchema.safeParse(toolInput);
if (!result.success) {
  console.error("Validation failed:", result.error);
}
```

## The full pattern

1. Define a tool with a JSON Schema matching your expected output.
2. Send the request with the tool definition.
3. Parse the `tool_use` block from the response.
4. Validate with Zod (or equivalent).
5. Use the validated data in your application.

If validation fails, you can retry with a clarified prompt or fall back to a default.

## What you learned

- **Tool use** (function calling) gives the model a schema to fill — far more reliable than "return JSON."
- **Always validate** the output. Zod is the standard validation library for TypeScript.
- The pattern: define schema → call with tool → parse tool_use block → validate → use.
