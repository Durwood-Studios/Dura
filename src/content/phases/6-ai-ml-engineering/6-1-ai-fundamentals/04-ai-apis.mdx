## Why this matters

Every AI feature you build talks to a model through an HTTP API. Understanding the request/response structure, authentication, error codes, and rate limits is the difference between a demo that works sometimes and a production service that works always.

## The Anthropic Messages API

```javascript
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": process.env.ANTHROPIC_API_KEY,
    "anthropic-version": "2023-06-01",
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1024,
    system: "You are a helpful assistant.",
    messages: [{ role: "user", content: "What is a closure?" }],
  }),
});

const data = await response.json();
console.log(data.content[0].text);
```

Key fields: `model`, `max_tokens` (required), `system` (optional string), `messages` (array of user/assistant turns).

## The OpenAI Chat Completions API

```javascript
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${process.env.OPENAI_API_KEY}`,
  },
  body: JSON.stringify({
    model: "gpt-4o",
    messages: [
      { role: "system", content: "You are a helpful assistant." },
      { role: "user", content: "What is a closure?" },
    ],
  }),
});

const data = await response.json();
console.log(data.choices[0].message.content);
```

The shapes differ: Anthropic uses `content[0].text`, OpenAI uses `choices[0].message.content`. Both return usage stats (input/output tokens).

## Authentication

Both APIs use API keys in headers. **Never expose these in client-side code.** Always proxy through your own backend:

```text
Client → Your Server → AI API
```

Your server holds the API key. The client never sees it.

## Rate limits and retries

APIs return `429 Too Many Requests` when you exceed rate limits. The response includes `retry-after` headers.

```javascript
async function callWithRetry(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (err) {
      if (err.status === 429 && i < maxRetries - 1) {
        const delay = Math.pow(2, i) * 1000;
        await new Promise((r) => setTimeout(r, delay));
        continue;
      }
      throw err;
    }
  }
}
```

Exponential backoff: wait 1s, 2s, 4s. Never retry 500s aggressively — the server is already struggling.

## Error handling

Common status codes:

- `400` — bad request (malformed JSON, missing required field)
- `401` — invalid API key
- `429` — rate limited
- `500` — server error (retry with backoff)
- `529` — overloaded (Anthropic-specific, retry with longer backoff)

## What you learned

- Anthropic and OpenAI APIs have similar patterns but different response shapes. Know both.
- API keys live on the server, proxied from the client. Never expose them.
- Handle 429s with exponential backoff. Handle 500s with cautious retries.
