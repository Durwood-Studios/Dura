## Why this matters

You cannot debug what you do not understand. When an LLM hallucinates, truncates, or gives inconsistent answers, understanding the mechanics — tokens, attention, context windows, sampling — tells you why and what to do about it.

## Tokens, not words

LLMs do not process words. They process **tokens** — sub-word chunks produced by a tokenizer (usually Byte Pair Encoding). The word "understanding" might become `["under", "standing"]` or `["understand", "ing"]` depending on the tokenizer.

A rough rule: **1 token ≈ 4 characters** in English. The word "hello" is 1 token. A JSON blob with lots of punctuation might be 2-3x more tokens than you expect.

```javascript
// Rough token estimation
function estimateTokens(text) {
  return Math.ceil(text.length / 4);
}

console.log(estimateTokens("Hello, how are you?")); // ~5
```

Why this matters: you pay per token, context windows are measured in tokens, and the model literally cannot see past its token limit.

## The transformer architecture

The transformer (2017, "Attention Is All You Need") is the architecture behind every major LLM. At a conceptual level:

1. **Tokenize** the input into a sequence of token IDs.
2. **Embed** each token as a high-dimensional vector (a list of numbers).
3. **Self-attention** — each token "looks at" every other token to understand context. The word "bank" looks at "river" vs "money" to decide its meaning.
4. **Feed-forward layers** transform the attention output.
5. **Repeat** through many layers (Claude has dozens).
6. **Predict** the next token by computing probabilities over the entire vocabulary.

The model generates one token at a time, appends it to the input, and repeats. This is why output is slower than input — each new token requires a full forward pass.

## Context windows

The context window is the total number of tokens the model can process at once — input + output combined. Claude's context window is up to 200K tokens. GPT-4o is 128K.

When you exceed the window, the model literally cannot see the overflow. It is not "forgetting" — the tokens simply are not there.

## Temperature and sampling

After computing token probabilities, the model samples from the distribution:

- **Temperature 0** — always pick the most probable token. Deterministic but repetitive.
- **Temperature 0.7** — moderate randomness. Good for most tasks.
- **Temperature 1.0+** — high randomness. Creative but less reliable.

**top_p** (nucleus sampling) is an alternative: only sample from tokens whose cumulative probability exceeds a threshold.

## Why LLMs hallucinate

LLMs predict the most likely next token. "Likely" is not "true." If the training data associated a pattern, the model reproduces it — even if the specific claim is fabricated. The model has no internal fact-checker. It is a very sophisticated autocomplete.

## What you learned

- LLMs process **tokens** (≈4 chars each), not words. You pay per token and context windows are measured in tokens.
- The transformer generates one token at a time using self-attention. Output is sequential; input is parallel.
- Hallucination happens because "statistically likely" ≠ "factually true." The model has no internal fact-checker.
