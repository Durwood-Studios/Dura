## Why this matters

The difference between a mediocre AI feature and a great one is usually the prompt, not the model. Prompt engineering is the first tool you reach for — before RAG, before fine-tuning, before agents. Get good at it and you solve 80% of problems without any infrastructure.

## The message structure

Modern LLM APIs use a messages array with roles:

```javascript
const messages = [
  { role: "system", content: "You are a helpful coding assistant." },
  { role: "user", content: "Explain closures in JavaScript." },
  { role: "assistant", content: "A closure is..." },
];
```

- **System** — sets behavior, personality, constraints. The model treats this as persistent instructions.
- **User** — the human's input.
- **Assistant** — the model's previous responses (for multi-turn conversations).

The system prompt is your primary control surface. Be specific: "You are a senior JavaScript developer who gives concise answers with code examples" beats "You are helpful."

## Few-shot prompting

Show the model what you want by example:

```text
Classify the sentiment of each review.

Review: "This product is amazing!"
Sentiment: positive

Review: "Terrible quality, broke after one day."
Sentiment: negative

Review: "It's okay, nothing special."
Sentiment:
```

The model sees the pattern and continues it. Few-shot works because LLMs are pattern-completion machines — show the pattern, get the pattern.

## Chain-of-thought

Ask the model to think step by step before answering:

```text
Q: If a store has 3 shelves with 8 books each, and
   2 shelves with 12 books each, how many books total?

Let me think step by step:
1. 3 shelves × 8 books = 24 books
2. 2 shelves × 12 books = 24 books
3. Total = 24 + 24 = 48 books

Answer: 48
```

Chain-of-thought dramatically improves accuracy on reasoning tasks. The model is more likely to get the right answer when it "shows its work."

## Structured output via prompting

When you need JSON, tell the model explicitly:

```text
Extract the following fields from the text below.
Return ONLY valid JSON with these keys: name, email, role.

Text: "Jane Smith (jane@example.com) is the engineering lead."
```

For critical paths, combine this with tool use (lesson 07) for guaranteed schema compliance.

## Common patterns

- **Classification:** "Classify this text as one of: [categories]. Return only the category name."
- **Extraction:** "Extract all dates mentioned in this text. Return as a JSON array."
- **Summarization:** "Summarize this in 3 bullet points. Each bullet should be one sentence."
- **Code generation:** "Write a JavaScript function that [spec]. Include error handling."

## The engineering part

Prompt engineering is engineering because it requires:

- **Version control** — track what changed and why.
- **Testing** — run the same prompt against test cases.
- **Measurement** — accuracy, latency, cost per call.
- **Iteration** — small changes, measure, repeat.

## What you learned

- The system prompt is your main control surface. Be specific about role, format, and constraints.
- **Few-shot** provides examples; **chain-of-thought** asks for step-by-step reasoning. Both dramatically improve quality.
- Prompt engineering is real engineering: version control, testing, measurement, iteration.
