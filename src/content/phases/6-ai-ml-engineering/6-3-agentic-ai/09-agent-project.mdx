## Why this matters

You have learned individual concepts — tool calling, planning, memory, multi-agent patterns, human-in-the-loop, evaluation. This lesson puts them all together. Building a multi-agent research assistant from scratch is the fastest way to internalize how these pieces connect. The system you build here is a simplified version of what powers real production tools: coding assistants, competitive intelligence platforms, and automated report generators.

## Architecture overview

The research assistant has three agents:

1. **SearchAgent** — takes a query, calls search tools, returns raw findings
2. **SummaryAgent** — takes raw findings, produces a concise summary
3. **Orchestrator** — receives the user's question, delegates to search and summary agents, manages the overall workflow

```javascript
// Agent definitions
const searchAgent = {
  name: "SearchAgent",
  systemPrompt: `You are a research specialist. Given a query:
1. Search for relevant information using your tools.
2. Return raw findings as a structured JSON array of { source, content, relevance }.
3. If initial results are insufficient, refine your query and search again.
Return only the findings — do not summarize.`,
  tools: ["web_search", "search_papers"],
};

const summaryAgent = {
  name: "SummaryAgent",
  systemPrompt: `You are a summarization specialist. Given raw research findings:
1. Identify the key themes and facts.
2. Resolve any contradictions between sources.
3. Produce a clear, structured summary with citations.
Return a markdown summary.`,
  tools: [],
};

const orchestrator = {
  name: "Orchestrator",
  systemPrompt: `You coordinate research tasks. Given a user question:
1. Break it into specific search queries.
2. Delegate each query to the SearchAgent.
3. Once all searches are complete, send findings to the SummaryAgent.
4. Review the summary and return it to the user.
If the summary is missing key information, run additional searches.`,
  tools: ["delegate_search", "delegate_summary"],
};
```

## Building the search agent

The search agent wraps tool calls in a focused loop. It has a single job: find information.

```javascript
async function runSearchAgent(query, searchTools, llm) {
  const messages = [
    { role: "system", content: searchAgent.systemPrompt },
    { role: "user", content: query },
  ];

  const findings = [];

  for (let step = 0; step < 5; step++) {
    const response = await llm.chat(messages, { tools: searchTools });

    if (!response.toolCalls?.length) {
      // Agent is done searching — parse findings from response
      try {
        const parsed = JSON.parse(response.content);
        findings.push(...parsed);
      } catch {
        findings.push({ source: "agent", content: response.content, relevance: "high" });
      }
      break;
    }

    for (const tc of response.toolCalls) {
      try {
        const result = await searchTools[tc.name](tc.arguments);
        messages.push({ role: "tool", toolCallId: tc.id, content: JSON.stringify(result) });
        findings.push(...(Array.isArray(result) ? result : [result]));
      } catch (error) {
        messages.push({ role: "tool", toolCallId: tc.id, content: `Error: ${error.message}` });
      }
    }

    messages.push({ role: "assistant", content: response.content, toolCalls: response.toolCalls });
  }

  return findings;
}
```

## Building the summary agent

The summary agent does not use tools. It takes raw findings and synthesizes them.

```javascript
async function runSummaryAgent(findings, llm) {
  const findingsText = findings.map((f, i) => `[${i + 1}] (${f.source}) ${f.content}`).join("\n\n");

  const messages = [
    { role: "system", content: summaryAgent.systemPrompt },
    {
      role: "user",
      content: `Summarize these research findings:\n\n${findingsText}`,
    },
  ];

  const response = await llm.chat(messages);
  return response.content;
}
```

## Building the orchestrator

The orchestrator ties everything together. It decomposes the user's question, delegates work, and assembles the result.

```javascript
async function runOrchestrator(userQuestion, searchTools, llm) {
  // Step 1: Decompose the question into search queries
  const decomposition = await llm.chat([
    {
      role: "system",
      content:
        "Break this research question into 2-4 specific search queries. Return a JSON array of strings.",
    },
    { role: "user", content: userQuestion },
  ]);

  let queries;
  try {
    queries = JSON.parse(decomposition.content);
  } catch {
    queries = [userQuestion]; // Fallback: use the original question
  }

  // Step 2: Run search agent for each query (parallel)
  const allFindings = [];
  const searchPromises = queries.map((query) => runSearchAgent(query, searchTools, llm));
  const searchResults = await Promise.all(searchPromises);

  for (const findings of searchResults) {
    allFindings.push(...findings);
  }

  // Step 3: Deduplicate findings
  const uniqueFindings = deduplicateFindings(allFindings);

  // Step 4: Summarize
  const summary = await runSummaryAgent(uniqueFindings, llm);

  // Step 5: Quality check — does the summary address the question?
  const qualityCheck = await llm.chat([
    {
      role: "system",
      content:
        "Does this summary adequately answer the original question? Reply YES or NO with a brief explanation.",
    },
    {
      role: "user",
      content: `Question: ${userQuestion}\n\nSummary: ${summary}`,
    },
  ]);

  if (qualityCheck.content.startsWith("NO") && queries.length < 4) {
    // Run additional search with refined query
    const refinedSearch = await runSearchAgent(
      `${userQuestion} — additional details needed: ${qualityCheck.content}`,
      searchTools,
      llm
    );
    uniqueFindings.push(...refinedSearch);
    return await runSummaryAgent(uniqueFindings, llm);
  }

  return summary;
}

function deduplicateFindings(findings) {
  const seen = new Set();
  return findings.filter((f) => {
    const key = f.content.slice(0, 100);
    if (seen.has(key)) return false;
    seen.add(key);
    return true;
  });
}
```

## Adding memory and evaluation

Wrap the orchestrator with episodic memory and trajectory logging:

```javascript
class ResearchAssistant {
  constructor(searchTools, llm) {
    this.searchTools = searchTools;
    this.llm = llm;
    this.episodicMemory = [];
  }

  async research(question) {
    const startTime = Date.now();
    const trace = { question, steps: [], toolCalls: [] };

    // Check episodic memory for similar past research
    const pastResearch = this.episodicMemory.find((ep) =>
      ep.question
        .split(" ")
        .filter((w) => w.length > 3)
        .some((w) => question.includes(w))
    );

    let context = "";
    if (pastResearch) {
      context = `\n\nRelevant past research: ${pastResearch.summary.slice(0, 500)}`;
    }

    const summary = await runOrchestrator(question + context, this.searchTools, this.llm);

    // Record this episode
    this.episodicMemory.push({
      question,
      summary: summary.slice(0, 1000),
      timestamp: Date.now(),
      durationMs: Date.now() - startTime,
    });

    trace.finalOutput = summary;
    return { summary, trace };
  }
}
```

## Putting it all together

The complete system flow:

1. User asks: "What are the main challenges of deploying LLMs in healthcare?"
2. Orchestrator decomposes into: ["LLM deployment challenges healthcare", "medical AI regulation 2025", "healthcare AI privacy concerns"]
3. SearchAgent runs three parallel searches, returning 15 findings
4. Orchestrator deduplicates to 11 unique findings
5. SummaryAgent synthesizes into a structured markdown report with citations
6. Orchestrator quality-checks — summary is adequate
7. Result returned to user; episode recorded in memory

Each agent stays focused. The search agent never tries to summarize. The summary agent never searches. The orchestrator never does domain work. This separation makes the system debuggable, testable, and improvable one piece at a time.

## What you learned

- A **multi-agent research assistant** decomposes questions into search queries, runs parallel searches, deduplicates findings, and synthesizes a summary — all coordinated by an orchestrator.
- **Separation of concerns** across agents makes each one independently testable and improvable without affecting the others.
- **Quality checking** the final output and triggering additional research when needed makes the system self-correcting and robust.
