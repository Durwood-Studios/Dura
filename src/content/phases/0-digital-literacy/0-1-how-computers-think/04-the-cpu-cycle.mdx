## Why this matters

Every program — every game, every browser tab, every calculator — eventually becomes a stream of tiny instructions running on a CPU. Understanding the loop the CPU runs is the difference between thinking of "the computer" as magic and thinking of it as a very fast, very simple machine doing the same three steps a billion times a second.

## The loop

A CPU does three things in a tight loop:

1. **Fetch** — grab the next instruction from memory.
2. **Decode** — figure out what the instruction means.
3. **Execute** — do the thing.

Then it does it again. And again. And again. A modern CPU does this billions of times per second.

```text
fetch  → decode → execute
  ↑                  ↓
  └──────────────────┘
```

That's it. That is the entire job of a CPU. Everything else — the operating system, your apps, this lesson — is built out of that loop.

## Clock speed isn't everything

A CPU's "clock speed" is how many cycles per second it can run. 3 GHz means 3 billion cycles per second.

But more GHz doesn't always mean faster overall. A modern 3 GHz CPU is wildly faster than a 3 GHz CPU from 2005, because each cycle now does more work, and the CPU runs many instructions in parallel.

When somebody compares CPUs by clock speed alone, they're missing most of the picture.

## What you learned

- A CPU runs the same three-step loop billions of times per second: fetch, decode, execute.
- Everything a computer does is built out of that loop.
- Clock speed alone doesn't tell you which CPU is faster — modern CPUs do much more work per cycle.
