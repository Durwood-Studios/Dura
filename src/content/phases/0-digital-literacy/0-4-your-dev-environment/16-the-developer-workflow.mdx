## Why this matters

You now know enough to do real work. This lesson ties everything together into the loop a working developer runs every day. Once this loop is muscle memory, you stop thinking about the tools and start thinking about the problem.

## The five steps

1. **Write** — open your editor, change some code.
2. **Test** — run the code (manually or with automated tests) to confirm it works.
3. **Commit** — stage and snapshot your changes locally with a clear message.
4. **Push** — upload to GitHub so it's safe and shareable.
5. **Deploy** — get the change in front of real users (in your case for now, that just means visiting it on the internet).

```text
write → test → commit → push → deploy
  ↑                                ↓
  └────────────────────────────────┘
```

You'll go around this loop dozens of times per day. The whole loop, for a small change, might take five minutes.

## Why this order matters

Each step is a checkpoint. If you skip one, you create a category of problems:

- **Skip testing** → bugs land in production.
- **Skip committing** → if your laptop crashes, you lose work.
- **Skip pushing** → only you have the code.
- **Skip deploying** → users never see the change.

The discipline isn't bureaucracy. It's a series of small safety nets, and skipping any of them costs more later than it saves now.

## Continuous Integration

Most modern projects automate steps 2 and 5 with **continuous integration** (CI). When you push to GitHub, a robot runs your tests. If they pass, another robot deploys the change. You see only the final result: green check or red X.

You don't need to set this up yet. But know it exists, because it's how real teams move fast without breaking everything.

for (let i = 0; i < stages.length; i++) {
if (!stages[i]()) return i;
}
return -1;
}

const stages = [
() => true,
() => true,
() => false,
];

console.log(runPipeline(stages));`}

## What you learned

- Every working developer runs the same loop: write → test → commit → push → deploy.
- Each step is a safety net; skipping one creates a predictable category of bugs.
- Continuous integration automates the test and deploy steps so teams can move fast safely.
