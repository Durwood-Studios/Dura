## Why this matters

Without version control, software development is impossible past about 50 lines. With it, you can rewind any change, branch into experiments, collaborate with strangers, and recover from mistakes that would otherwise destroy a day. Git is the version control system every modern developer uses, and the basics take an afternoon to learn.

## What version control is

Imagine if every time you saved a document, your computer kept a labeled snapshot of every previous version. You could rewind to any earlier state, see exactly what changed between any two snapshots, and split into parallel "what if" branches that you could merge later.

That's version control. Git is the most popular implementation, by a wide margin.

## Init and status

`git init` turns a directory into a Git repository.

```text
$ cd my-project
$ git init
Initialized empty Git repository in /Users/dustin/my-project/.git/
```

`git status` shows you what's changed since the last snapshot.

```text
$ git status
On branch main

No commits yet

Untracked files:
  README.md
```

You'll run `git status` _constantly_. It's the cheapest way to check where you stand.

## Add and commit

A snapshot in Git is called a **commit**. Making one is a two-step process: stage the files you want included, then commit them with a message.

```text
$ git add README.md
$ git commit -m "Add a README"
[main (root-commit) f3a2b1c] Add a README
 1 file changed, 12 insertions(+)
```

The message is for your future self — and your teammates. "Fixed bug" is a bad message. "Fix off-by-one in lesson scroll tracker" is a good message.

## Why two steps?

Staging exists so you can group related changes into one commit. Maybe you fixed a bug AND added a feature in the same session — staging lets you commit them separately, with separate messages, even though both sets of changes are sitting in your working directory.

## What you learned

- Version control lets you snapshot, rewind, and branch your work.
- The basic flow is `git init` → `git add` → `git commit`.
- `git status` is the safest, cheapest command — run it whenever you're not sure where you stand.
