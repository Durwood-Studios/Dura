## Why this matters

You will spend more hours with your code editor than with any other tool, full stop. The difference between a good setup and a default one is the difference between sliding through a workday and grinding through it. The good news: a sensible setup takes about 30 minutes and serves you for years.

## Just use VS Code

There is real, lively debate about which editor is "best." Vim, Neovim, JetBrains, Sublime, Zed — each has passionate fans. As a beginner, ignore that debate. Use **VS Code**.

It's free, runs everywhere, has the largest ecosystem of extensions, and is what most companies you'll join already use. You can switch later if you want.

Download it from `code.visualstudio.com`. The default install is fine.

## Day-one extensions

A handful of extensions transform VS Code from a generic text editor into a coding environment.

- **Prettier** — formats your code automatically every time you save. Eliminates an entire category of arguments.
- **ESLint** — flags suspicious JavaScript before you run it.
- **GitLens** — shows who wrote each line and when, right next to the code.
- **Material Icon Theme** — file icons that make it easy to scan a sidebar full of files.

Install via the Extensions panel (`Ctrl+Shift+X` / `Cmd+Shift+X`). Type the name, click install.

## The integrated terminal

VS Code has a built-in terminal. Open it with `Ctrl+\`` (backtick). It's the same shell as a standalone terminal, but it lives next to your code. You'll switch back and forth constantly — running a command, scrolling back to the code, then back to the terminal. Having both in one window saves an enormous amount of context-switching.

## What you learned

- VS Code is the right default for beginners — free, popular, huge ecosystem.
- A small set of extensions (Prettier, ESLint, GitLens, an icon theme) covers most needs.
- The integrated terminal keeps your code and shell in one window — use it.
