## Why this matters

Knowing `ls` and `cd` makes you functional. Knowing pipes makes you _fast_. The Unix philosophy is built on the idea that small tools, combined cleverly, are more powerful than big tools used alone. This lesson is the first taste of that idea.

## Pipes: chain commands together

The pipe character `|` takes the output of one command and sends it as input to the next.

```text
$ ls | wc -l
17
```

`ls` listed the files. `wc -l` counted the lines. Together, they answered "how many files are in this directory?" without any new tool.

You can chain as many as you want.

```text
$ ls -la | grep "^d" | wc -l
```

That's "list everything → keep only lines starting with d (directories) → count them."

## grep: search through text

`grep` finds lines matching a pattern. Add `-r` to recurse into directories.

```text
$ grep -r "TODO" src/
src/lib/auth.ts:// TODO: handle expired tokens
src/components/Header.tsx:// TODO: mobile menu
```

In a single command you've found every TODO in your project.

## history: what did I just type?

`history` shows your recent commands. Use the up arrow to scroll through them. `Ctrl+R` lets you search backwards through history — type a few letters and the matching command appears.

This is the single biggest productivity multiplier in the terminal. You almost never need to retype a command from scratch.

## Aliases: shortcuts you create

If you keep typing the same long command, make an alias.

```text
$ alias gs="git status"
$ gs
On branch main
nothing to commit, working tree clean
```

Real developers fill `.bashrc` or `.zshrc` with aliases. You'll start collecting your own.

## What you learned

- Pipes (`|`) chain small commands into powerful one-liners.
- `grep` finds matching lines in files; combine it with pipes to search anything.
- `history` and `Ctrl+R` save you from retyping. Aliases save you from typing at all.
