## Why this matters

If you've only ever used a computer through windows, icons, and clicks, the terminal looks like a step backward. It isn't. It's a step toward _control_.

Every senior developer you'll meet uses one daily. They aren't doing it to look cool. They're doing it because typing one line is faster than clicking through five menus, and because terminal commands compose in ways graphical apps never can.

## What it actually is

A terminal is a text-based interface to your operating system. You type a command, it runs, you see the result. That's it.

Behind the scenes, the terminal is talking to a **shell** — a program whose job is to read what you typed, figure out what to do, and run it. On macOS and Linux, the most common shell is `bash` or `zsh`. On Windows, it's PowerShell or `cmd.exe` (or `bash` if you install WSL).

```text
$ echo "hello world"
hello world
```

The `$` is called the **prompt**. Anything before it tells you where you are. Anything after it is what you typed.

## Why type instead of click?

Three reasons.

**Speed.** Once you know the commands, you'll move through your filesystem faster than any file explorer can keep up.

**Repeatability.** You can save commands in a file and run them again. You cannot save a sequence of mouse clicks.

**Composability.** The output of one command can become the input of the next. We'll see how in lesson 8 — it's the most powerful idea in this whole module.

## How to open one

- **macOS:** Spotlight (`Cmd+Space`), type "Terminal", hit enter.
- **Windows:** Search for "Windows Terminal" or "PowerShell" in the Start menu.
- **Linux:** `Ctrl+Alt+T` on most desktops, or look in your application menu for "Terminal".

You'll see a blinking cursor and a prompt. That's home base.

## What you learned

- A terminal is a text interface to your OS; a shell is the program that interprets your commands.
- Developers use terminals for speed, repeatability, and composability.
- Every operating system has a built-in terminal — open it once and you're already past the hardest part.
