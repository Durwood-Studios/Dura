## Why this matters

Navigating is half of using a terminal. The other half is creating, reading, and modifying files. Four small commands cover the basics — and once you know them, you can scaffold an entire project without ever touching the mouse.

## mkdir: make a directory

```text
$ mkdir notes
$ ls
notes
```

Add `-p` to create nested directories in one shot:

```text
$ mkdir -p notes/2026/april
```

The `-p` means "make parents as needed." Without it, `mkdir` errors out if any parent directory is missing.

## touch: create an empty file

```text
$ touch ideas.md
$ ls
ideas.md  notes
```

`touch` was originally designed to update file timestamps; the side effect of creating a missing file is just how everyone uses it now.

## cat: read a file

```text
$ cat README.md
# My Project
A small project to learn the terminal.
```

`cat` ("concatenate") prints the contents of one or more files to your terminal. For very long files, use `less` instead — it lets you scroll page by page and quit with `q`.

## echo and redirection: write a file

`echo` prints text. The `>` operator redirects that text into a file.

```text
$ echo "Hello, world." > hello.txt
$ cat hello.txt
Hello, world.
```

A single `>` overwrites. A double `>>` appends.

```text
$ echo "Second line." >> hello.txt
$ cat hello.txt
Hello, world.
Second line.
```

You just wrote a two-line file without ever opening an editor.

## What you learned

- `mkdir` makes directories; add `-p` for nested ones.
- `touch` creates empty files; `cat` reads them; `less` is better for long files.
- `echo "text" > file` writes; `>>` appends.
