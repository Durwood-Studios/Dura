## Why this matters

Every modern app talks to other apps. Your weather app talks to a forecasting service. Your fitness tracker talks to a cloud sync service. Your favorite game talks to a leaderboard server. The thing they're all using is an **API** — and once you understand what one is, you can connect almost anything to almost anything else.

## What an API actually is

API stands for **Application Programming Interface**. The plain-English version: it's a contract that says "if you ask me for X in this exact way, I'll give you Y in this exact format."

Think of it like a restaurant menu. The menu doesn't show you the kitchen — it just tells you what you can order and what you'll get back. An API works the same way: it hides the implementation and exposes only the operations you're allowed to perform.

## REST: the most common style

Most web APIs follow a style called **REST**. The idea: you ask for "resources" using HTTP methods.

| Method | Means   | Example            |
| ------ | ------- | ------------------ |
| GET    | Read    | GET `/users/42`    |
| POST   | Create  | POST `/users`      |
| PUT    | Replace | PUT `/users/42`    |
| DELETE | Remove  | DELETE `/users/42` |

A REST API URL is just a noun. The HTTP method is the verb. Together they describe the action.

## JSON: how the data looks

The format almost every modern API uses for sending data back and forth is **JSON** (JavaScript Object Notation). It's a small, strict subset of JavaScript object syntax.

```json
{
  "id": 42,
  "name": "Dustin",
  "active": true,
  "skills": ["typescript", "rust", "design"]
}
```

That's the entire format: objects, arrays, strings, numbers, booleans, and `null`. Every modern language has built-in JSON support. You'll use it daily as a developer.

## What you learned

- An API is a contract: "ask me this way, get this back."
- REST APIs use HTTP methods (GET/POST/PUT/DELETE) on resource URLs.
- JSON is the format almost every modern API uses to send data — small, strict, universally supported.
