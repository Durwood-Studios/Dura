## Why this matters

You type `dura.dev` into your browser and a page appears. Half a dozen distinct technologies cooperate to make that happen, and every one of them can fail. Knowing what each piece does turns "the internet is broken" into "DNS is slow today."

## Anatomy of a URL

A URL has more parts than people realize.

```text
https://dura.dev/dictionary/algorithm?from=lesson#definition
└─┬─┘   └───┬──┘└────────┬───────────┘└────┬────┘└────┬────┘
scheme   host           path             query     fragment
```

- **Scheme** — `https` says "use the secure HTTP protocol."
- **Host** — `dura.dev` is the human name of the server.
- **Path** — `/dictionary/algorithm` selects a specific resource on that server.
- **Query** — `?from=lesson` carries extra parameters.
- **Fragment** — `#definition` jumps to a section after the page loads (the server never sees it).

## DNS: name → number

Computers don't talk to each other by name. They talk by IP address — `142.250.65.78` style. DNS (Domain Name System) is a giant distributed phone book that translates names into addresses.

When you type `dura.dev`, your computer asks a DNS server "what's the IP for dura.dev?" The DNS server replies with a number, and the browser uses that number to actually connect.

DNS results are cached aggressively, which is why you sometimes see "DNS still propagating" after a domain change — the old cached answer hasn't expired everywhere.

## HTTP: how the conversation happens

Once your browser knows the IP, it opens a TCP connection and sends an HTTP **request**:

```text
GET /dictionary/algorithm HTTP/1.1
Host: dura.dev
Accept: text/html
```

The server replies with an HTTP **response**:

```text
HTTP/1.1 200 OK
Content-Type: text/html

<!DOCTYPE html>...
```

That's the whole protocol. A request, then a response. Status codes (`200`, `404`, `500`) tell you what happened. Headers carry metadata. The body carries the actual content.

## What you learned

- A URL has a scheme, host, path, query, and fragment — each piece does a different job.
- DNS translates names into IP addresses; results are cached, which can cause stale lookups.
- HTTP is just request and response, with status codes telling you how things went.
