## Why this matters

SQL is the language every database speaks. Whether you're using PostgreSQL, MySQL, SQLite, or a cloud warehouse like BigQuery, the same four operations — SELECT, INSERT, UPDATE, DELETE — form the foundation. Learning them well means you can query any relational database, debug performance issues, and write application code that doesn't accidentally corrupt data.

## SELECT: reading data

`SELECT` retrieves rows from a table. At minimum you specify which columns you want and which table to query:

```sql
SELECT id, username, email
FROM   users
WHERE  created_at > '2024-01-01'
ORDER  BY username ASC
LIMIT  10
OFFSET 20;
```

- `WHERE` filters which rows are returned
- `ORDER BY` sorts the result (ASC = smallest first, DESC = largest first)
- `LIMIT` caps the number of rows — essential for pagination
- `OFFSET` skips rows — used with LIMIT to implement pages

`SELECT *` returns every column. Avoid it in production code — it's fragile (breaks when you add/remove columns), wasteful (transfers data you don't need), and hides intent. Name your columns explicitly.

### Filtering operators

```sql
-- Equality and inequality
WHERE status = 'active'
WHERE age != 0

-- Comparison
WHERE price > 100
WHERE price BETWEEN 10 AND 100

-- Lists
WHERE country IN ('US', 'CA', 'MX')

-- Pattern matching (% is wildcard)
WHERE email LIKE '%@gmail.com'

-- NULL checks (never use = NULL — it doesn't work)
WHERE deleted_at IS NULL
WHERE deleted_at IS NOT NULL

-- Combine conditions
WHERE status = 'active' AND age >= 18
WHERE role = 'admin' OR role = 'moderator'
WHERE NOT (status = 'banned')
```

### Column aliases

```sql
SELECT
  u.username                          AS author,
  COUNT(p.id)                         AS post_count,
  MAX(p.published_at)                 AS last_published
FROM users u
JOIN posts p ON p.user_id = u.id
GROUP BY u.id, u.username;
```

`AS` renames a column in the result. Table aliases (`u`, `p`) shorten repeated references.

## INSERT: creating rows

```sql
-- Insert one row
INSERT INTO users (email, username)
VALUES ('alice@example.com', 'alice');

-- Insert multiple rows at once
INSERT INTO posts (user_id, title, status)
VALUES
  (1, 'First post',  'draft'),
  (1, 'Second post', 'published');

-- Return the auto-generated id (PostgreSQL-specific)
INSERT INTO users (email, username)
VALUES ('bob@example.com', 'bob')
RETURNING id, created_at;
```

`RETURNING` is a PostgreSQL feature that returns columns from the newly inserted row — critical for getting the `id` your application needs without a separate SELECT.

## UPDATE: modifying rows

```sql
-- Always include WHERE — otherwise you update every row
UPDATE users
SET    email = 'newemail@example.com',
       updated_at = NOW()
WHERE  id = 42;

-- Update based on another table's data
UPDATE posts
SET    status = 'archived'
WHERE  user_id IN (
  SELECT id FROM users WHERE banned = true
);
```

**The single most common SQL mistake**: writing `UPDATE table SET col = val` without a `WHERE` clause. It updates every row. In production, that's a data disaster. In PostgreSQL you can protect against this with `default_transaction_read_only` in staging, or by wrapping in a transaction you can inspect before committing.

## DELETE: removing rows

```sql
-- Always include WHERE
DELETE FROM sessions
WHERE  expires_at < NOW();

-- Delete and return what was deleted
DELETE FROM invitations
WHERE  token = 'abc123'
RETURNING user_id, email;
```

The same `WHERE`-or-everything warning applies to DELETE. Soft deletes (setting `deleted_at = NOW()` instead of removing the row) are often safer and preserve history.

## Aggregate functions

Aggregates compute a value across multiple rows:

```sql
SELECT
  COUNT(*)          AS total_users,
  COUNT(avatar_url) AS users_with_avatar,  -- NULLs are excluded
  AVG(age)          AS average_age,
  MIN(created_at)   AS first_signup,
  MAX(created_at)   AS last_signup,
  SUM(credits)      AS total_credits
FROM users
WHERE status = 'active';
```

### GROUP BY and HAVING

```sql
-- Count posts per user
SELECT   user_id, COUNT(*) AS post_count
FROM     posts
WHERE    status = 'published'
GROUP BY user_id
HAVING   COUNT(*) >= 5          -- filter on the aggregated result
ORDER BY post_count DESC;
```

- `WHERE` runs _before_ grouping — it filters individual rows
- `HAVING` runs _after_ grouping — it filters the aggregated groups
- You can only SELECT columns that appear in GROUP BY or inside an aggregate function

{`// Implement an in-memory SQL-like query engine
// Parses a subset of SELECT statements and executes them

function createQueryEngine(tables) {
// tables: { tableName: [{ ...row }] }

function query(sql) {
const normalized = sql.replace(/\\s+/g, ' ').trim();

    // Parse: SELECT cols FROM table [WHERE field = value] [ORDER BY field [DESC]] [LIMIT n]
    const selectMatch = normalized.match(
      /^SELECT\\s+(.+?)\\s+FROM\\s+(\\w+)(?:\\s+WHERE\\s+(\\w+)\\s*=\\s*'?([^'\\s]+)'?)?(?:\\s+ORDER\\s+BY\\s+(\\w+)(?:\\s+(ASC|DESC))?)?(?:\\s+LIMIT\\s+(\\d+))?/i
    );

    if (!selectMatch) {
      throw new Error(\`Cannot parse: \${sql}\`);
    }

    const [, colsPart, tableName, whereField, whereValue, orderField, orderDir, limitStr] = selectMatch;

    const table = tables[tableName];
    if (!table) throw new Error(\`Table "\${tableName}" does not exist\`);

    // Parse column list (supports "col AS alias")
    const colDefs = colsPart === '*'
      ? null
      : colsPart.split(',').map(c => {
          const m = c.trim().match(/^(\\w+)(?:\\s+AS\\s+(\\w+))?$/i);
          return m ? { field: m[1], alias: m[2] || m[1] } : null;
        }).filter(Boolean);

    // Filter
    let rows = [...table];
    if (whereField && whereValue !== undefined) {
      const coerced = isNaN(whereValue) ? whereValue : Number(whereValue);
      rows = rows.filter(r => String(r[whereField]) === String(coerced) || r[whereField] === coerced);
    }

    // Project columns
    if (colDefs) {
      rows = rows.map(row => {
        const out = {};
        for (const { field, alias } of colDefs) {
          out[alias] = row[field];
        }
        return out;
      });
    }

    // Order
    if (orderField) {
      rows.sort((a, b) => {
        const aVal = a[orderField], bVal = b[orderField];
        const cmp = aVal < bVal ? -1 : aVal > bVal ? 1 : 0;
        return orderDir === 'DESC' ? -cmp : cmp;
      });
    }

    // Limit
    if (limitStr) {
      rows = rows.slice(0, parseInt(limitStr, 10));
    }

    return rows;

}

return { query };
}

## What you learned

- SELECT retrieves data; specify columns explicitly and always include ORDER BY + LIMIT for pagination
- Filtering operators include =, !=, BETWEEN, IN, LIKE, IS NULL, IS NOT NULL, AND, OR, NOT
- INSERT can insert multiple rows at once; RETURNING fetches the auto-generated id
- UPDATE and DELETE without WHERE affect every row — always include a WHERE clause
- Aggregate functions (COUNT, SUM, AVG, MIN, MAX) compute across multiple rows
- GROUP BY groups rows for aggregation; HAVING filters the resulting groups
