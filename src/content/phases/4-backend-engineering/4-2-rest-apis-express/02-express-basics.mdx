## Why this matters

Node.js ships with an `http` module that can handle requests — but using it directly means writing your own routing logic, your own body parser, your own error handling, and your own everything else. Express solves that. It is the most widely used Node.js web framework: minimal, unopinionated, and so ubiquitous that understanding Express means understanding the pattern that dozens of other frameworks (Fastify, Koa, Hapi) react to or extend. Every Express concept you learn — middleware pipelines, route parameters, the req/res objects — transfers directly to every serious Node.js backend you will encounter.

## What Express adds

Express wraps Node's `http.createServer()` and adds three fundamental capabilities:

**Routing** — matching incoming requests to handler functions based on HTTP method and URL pattern.

**Middleware pipeline** — a composable chain of functions that process every request in sequence, each getting a chance to inspect, modify, or respond before passing control to the next.

**Request/response helpers** — convenience methods on the req and res objects that make common operations much less verbose.

A minimal Express app looks like this:

```javascript
import express from "express";

const app = express();

// Middleware: parse incoming JSON bodies
app.use(express.json());

// Route: GET /hello
app.get("/hello", (req, res) => {
  res.json({ message: "Hello, world!" });
});

app.listen(3000, () => console.log("Server running on port 3000"));
```

## The middleware pipeline

This is the mental model that unlocks all of Express. Every incoming request passes through a **pipeline** of middleware functions. Each middleware has the same signature:

```javascript
function myMiddleware(req, res, next) {
  // Do something with the request
  // Then either:
  next();          // Pass control to the next middleware
  // OR:
  res.json({...}); // Respond and end the chain
}
```

`app.use()` registers middleware that runs for every request, regardless of method or path. `app.get()`, `app.post()`, etc., register route handlers that only run when the method and path match.

Middleware runs in the **order it is registered**. This matters enormously:

```javascript
// Logging — runs first for every request
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next(); // MUST call next() or the request hangs
});

// Auth check — runs second
app.use((req, res, next) => {
  const token = req.headers.authorization;
  if (!token) {
    return res.status(401).json({ error: "Unauthorized" });
    // Returning here ends the chain — no next() needed
  }
  next();
});

// Route handler — runs third, only if auth passed
app.get("/dashboard", (req, res) => {
  res.json({ data: "..." });
});
```

If logging middleware is registered after the route, it never runs for that route. If auth middleware is registered after the route, unauthenticated users can access it. Order is everything.

## The req object

`req` represents the incoming HTTP request. The properties you will use constantly:

```javascript
app.get("/users/:id", (req, res) => {
  req.params.id; // Route parameter — "42" from /users/42
  req.query.page; // Query string — "2" from ?page=2
  req.body; // Parsed request body (requires express.json())
  req.headers["authorization"]; // Raw header value
  req.method; // "GET", "POST", etc.
  req.path; // "/users/42"
  req.ip; // Client IP address
});
```

**Route parameters** (`:id`) match any value in that position and are available on `req.params`. **Query strings** (`?page=2&sort=name`) are parsed into `req.query`. The **request body** (`req.body`) is available after JSON or URL-encoded body parsing middleware runs — it is `undefined` without that middleware.

## The res object

`res` represents the outgoing HTTP response. The most useful methods:

```javascript
// Send JSON (sets Content-Type: application/json automatically)
res.json({ message: "ok" });

// Set status then send
res.status(201).json({ id: newUser.id });

// Send plain text
res.send("Not found");

// Set a header manually
res.set("X-Request-Id", requestId);

// Redirect
res.redirect(301, "/new-location");

// No body (e.g. successful DELETE)
res.status(204).send();
```

Chain `status()` before `json()` to set both at once. After calling `res.json()` or `res.send()`, the response is sent — calling them again throws an error.

## Router modules

As an app grows, putting every route in one file becomes unmanageable. Express's `Router` class creates mini-applications that handle a subset of routes:

```javascript
// routes/users.js
import { Router } from "express";
const router = Router();

router.get("/", (req, res) => res.json(getAllUsers()));
router.post("/", (req, res) => res.status(201).json(createUser(req.body)));
router.get("/:id", (req, res) => res.json(getUser(req.params.id)));

export default router;

// app.js
import usersRouter from "./routes/users.js";
app.use("/users", usersRouter);
// Now GET /users, POST /users, GET /users/:id all work
```

The router's paths are relative to the prefix given to `app.use()`. `/` in the router becomes `/users`, `/:id` becomes `/users/:id`.

## Error-handling middleware

Express uses a special four-argument middleware signature for error handling. The extra first argument receives any error passed to `next(error)`:

```javascript
// Regular middleware calls next(error) to trigger this
app.use((err, req, res, next) => {
  console.error(err);
  res.status(err.statusCode || 500).json({
    error: err.message || "Internal Server Error",
  });
});
```

This must be registered **last** — after all routes and normal middleware. When any route or middleware calls `next(new Error('...'))`, Express skips all remaining normal middleware and jumps directly to this handler.

## Try it

<SandboxExercise language="javascript">
{`// Build a minimal Express-like App class.
// Implement: use(fn), get(path, fn), post(path, fn), handle(method, path, body)
// Middleware runs in registration order, each receiving (req, res, next).
// Route handlers match method + path (support :param placeholders).
// handle() returns { status: number, body: any }.

class App {
constructor() {
this.middlewares = [];
this.routes = [];
}

use(fn) {
this.middlewares.push({ type: "middleware", fn });
return this;
}

get(path, fn) {
this.routes.push({ method: "GET", path, fn });
return this;
}

post(path, fn) {
this.routes.push({ method: "POST", path, fn });
return this;
}

_matchPath(pattern, path) {
const paramNames = [];
const regexStr = pattern
.replace(/:([\\w]+)/g, (_, name) => { paramNames.push(name); return "([^/]+)"; })
.replace(/\\//g, "\\\\/");
const match = path.match(new RegExp(\`^\${regexStr}$\`));
if (!match) return null;
const params = {};
paramNames.forEach((name, i) => { params[name] = match[i + 1]; });
return params;
}

handle(method, path, body = null) {
let responseResult = null;

    const res = {
      _status: 200,
      status(code) { this._status = code; return this; },
      json(data) { responseResult = { status: this._status, body: data }; }
    };

    const req = { method, path, body, params: {}, query: {} };

    // Build pipeline: all middleware + matching route handler
    const pipeline = [...this.middlewares];

    // Find matching route
    let matchedRoute = null;
    for (const route of this.routes) {
      if (route.method !== method) continue;
      const params = this._matchPath(route.path, path);
      if (params !== null) {
        matchedRoute = { fn: route.fn, params };
        break;
      }
    }

    if (matchedRoute) {
      req.params = matchedRoute.params;
      pipeline.push({ type: "route", fn: matchedRoute.fn });
    }

    // Run pipeline
    let i = 0;
    function next() {
      if (i >= pipeline.length) return;
      const current = pipeline[i++];
      current.fn(req, res, next);
    }
    next();

    return responseResult || { status: 404, body: { error: "Not Found" } };

}
}

// Demo: logging → auth → route handler
const app = new App();

// Logging middleware
app.use((req, res, next) => {
console.log(\`[LOG] \${req.method} \${req.path}\`);
next();
});

// Auth middleware
app.use((req, res, next) => {
if (req.body && req.body.token === "secret") {
req.userId = "user-42";
next();
} else {
res.status(401).json({ error: "Unauthorized" });
}
});

app.get("/users/:id", (req, res, next) => {
res.status(200).json({ id: req.params.id, requestedBy: req.userId });
});

app.post("/users", (req, res, next) => {
res.status(201).json({ created: true, data: req.body });
});

console.log("--- Authenticated GET ---");
console.log(app.handle("GET", "/users/42", { token: "secret" }));

console.log("\\n--- Unauthenticated GET ---");
console.log(app.handle("GET", "/users/42", {}));

console.log("\\n--- Authenticated POST ---");
console.log(app.handle("POST", "/users", { token: "secret", name: "Alice" }));

console.log("\\n--- Unknown route ---");
console.log(app.handle("GET", "/products", { token: "secret" }));`}

## What you learned

- Express adds routing, a middleware pipeline, and request/response helpers on top of Node's built-in http module.
- Middleware runs in registration order; each function must call next() or send a response.
- `req.params` holds route parameters, `req.query` holds query strings, `req.body` holds the parsed request body.
- `Router` modules split routes across files, with paths relative to the prefix given to `app.use()`.
- Error-handling middleware uses a four-argument signature `(err, req, res, next)` and must be registered last.
