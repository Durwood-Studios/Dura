## Why this matters

Writing the same function for every type is tedious and fragile. Generics let you write code that works with many types while traits let you constrain those types to ones that support the operations you need. Together, they give you polymorphism — the ability to write code that works with different types — without the runtime overhead of virtual dispatch (by default). This is how Rust achieves its "zero-cost abstractions" promise for generic code, and understanding traits is essential because the entire standard library is built on them.

## What traits are

A **trait** defines a set of methods that a type must implement. Think of it as an interface in TypeScript or Java, but with the ability to provide default implementations.

```rust
trait Summary {
    fn summarize(&self) -> String;

    // Default implementation — types can override or use as-is
    fn preview(&self) -> String {
        format!("{}...", &self.summarize()[..20])
    }
}
```

## Implementing traits

You implement a trait for a specific type with an `impl Trait for Type` block:

```rust
struct Article {
    title: String,
    author: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{} by {}", self.title, self.author)
    }
    // preview() uses the default implementation
}

struct Tweet {
    username: String,
    text: String,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("@{}: {}", self.username, self.text)
    }
}
```

Now both `Article` and `Tweet` can be used anywhere a `Summary` is expected.

## Generic functions with trait bounds

Generics let you write functions that work with any type. **Trait bounds** constrain which types are accepted:

```rust
// This function accepts any type that implements Summary
fn print_summary<T: Summary>(item: &T) {
    println!("{}", item.summarize());
}

// Equivalent syntax using where clause (cleaner for multiple bounds)
fn print_detailed<T>(item: &T)
where
    T: Summary + std::fmt::Display,
{
    println!("Summary: {}", item.summarize());
    println!("Full: {}", item);
}
```

At compile time, Rust **monomorphizes** generic functions — it generates specialized versions for each concrete type used. `print_summary::<Article>` and `print_summary::<Tweet>` become two separate functions in the binary. There is no runtime dispatch, no vtable lookup. The generic code runs as fast as hand-written specialized code.

## Common traits in the standard library

Rust's standard library defines traits you will use constantly:

**Display** — human-readable formatting (used by `println!("{}", x)`):

```rust
use std::fmt;

impl fmt::Display for Article {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "'{}' by {}", self.title, self.author)
    }
}
```

**Debug** — developer-readable formatting (used by `println!("{:?}", x)`). Usually derived automatically:

```rust
#[derive(Debug)]
struct Point { x: f64, y: f64 }
```

**Clone** — explicit deep copy. `#[derive(Clone)]` generates it automatically for types whose fields are all Clone.

**Iterator** — the backbone of Rust's iteration:

```rust
struct Counter { current: u32, max: u32 }

impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<u32> {
        if self.current < self.max {
            self.current += 1;
            Some(self.current)
        } else {
            None
        }
    }
}
```

Once you implement `Iterator`, you get `.map()`, `.filter()`, `.fold()`, `.collect()`, and dozens of other methods for free — they are all provided as default methods on the Iterator trait.

## Traits vs interfaces

If you know TypeScript interfaces, Rust traits will feel familiar — but there are key differences:

- Traits can have **default method implementations**, not just signatures.
- Traits can be implemented for types you did not define (with restrictions).
- Trait methods are **monomorphized** by default — no virtual dispatch overhead.
- You can use `dyn Trait` for dynamic dispatch when you need it, but it is opt-in.

## What you learned

- **Traits** define shared behavior (method signatures with optional defaults) that types implement — similar to interfaces but more powerful.
- **Generics with trait bounds** let you write type-safe, reusable code; Rust **monomorphizes** generics at compile time for zero-cost polymorphism.
- Standard library traits like **Display**, **Debug**, **Clone**, and **Iterator** provide the foundation for formatting, copying, and iteration across all Rust types.
