## Why this matters

Every program models data. In JavaScript, you use plain objects and hope the shape stays consistent. In Rust, structs and enums define data shapes precisely, and the compiler rejects code that does not match. Enums in Rust are far more powerful than in most languages — each variant can carry different data, and the `match` expression forces you to handle every case. This is the foundation for `Option<T>` (Rust's replacement for null) and `Result<T, E>` (Rust's replacement for exceptions), two types you will use in virtually every Rust program.

## Structs: named fields

A **struct** groups related data under a single type. Unlike JavaScript objects, every field has a declared type, and you cannot add or remove fields at runtime.

```rust
struct User {
    name: String,
    email: String,
    active: bool,
    login_count: u64,
}

fn main() {
    let user = User {
        name: String::from("Dustin"),
        email: String::from("dustin@example.com"),
        active: true,
        login_count: 1,
    };
    println!("{} has logged in {} times", user.name, user.login_count);
}
```

## Methods with impl blocks

You add methods to a struct using an **impl block**. The first parameter is `self` (takes ownership), `&self` (borrows immutably), or `&mut self` (borrows mutably).

```rust
impl User {
    // Associated function (like a static method) — no self parameter
    fn new(name: String, email: String) -> User {
        User { name, email, active: true, login_count: 0 }
    }

    // Method — borrows self immutably
    fn summary(&self) -> String {
        format!("{} ({})", self.name, self.email)
    }

    // Method — borrows self mutably
    fn log_in(&mut self) {
        self.login_count += 1;
    }
}
```

## Enums: variants with data

Rust enums are **algebraic data types**. Each variant can carry different types of data — no data, a tuple, or named fields.

```rust
enum Shape {
    Circle(f64),                        // radius
    Rectangle { width: f64, height: f64 }, // named fields
    Point,                              // no data
}
```

This is fundamentally different from enums in C or Java, where variants are just named integers. Rust enums can represent "one of these different shapes, each with its own associated data."

## Pattern matching with match

The `match` expression is how you work with enums. It is like a `switch` statement that the compiler forces to be exhaustive — you must handle every variant.

```rust
fn area(shape: &Shape) -> f64 {
    match shape {
        Shape::Circle(radius) => std::f64::consts::PI * radius * radius,
        Shape::Rectangle { width, height } => width * height,
        Shape::Point => 0.0,
    }
}
```

If you add a new variant to the enum and forget to handle it in a `match`, the code will not compile. This is a major safety advantage over if-else chains or switch statements that silently fall through.

## Option: Rust's replacement for null

Rust has no null. Instead, the standard library provides `Option<T>`, an enum with two variants:

```rust
enum Option<T> {
    Some(T),  // A value exists
    None,     // No value
}
```

Any time a value might not exist, you use `Option`. The compiler forces you to handle the `None` case — you cannot accidentally dereference null because null does not exist.

```rust
fn find_user(id: u64) -> Option<User> {
    if id == 1 {
        Some(User::new(String::from("Dustin"), String::from("d@ex.com")))
    } else {
        None
    }
}

fn main() {
    match find_user(1) {
        Some(user) => println!("Found: {}", user.summary()),
        None => println!("User not found"),
    }
}
```

## Result: success or failure

`Result<T, E>` is an enum for operations that can fail:

```rust
enum Result<T, E> {
    Ok(T),   // Success with value T
    Err(E),  // Failure with error E
}
```

You will learn much more about Result in the next lesson on error handling. For now, know that it works exactly like Option — the compiler forces you to handle both cases.

## What you learned

- **Structs** define data with named, typed fields; methods are added via `impl` blocks with `&self`, `&mut self`, or `self` parameters.
- **Enums** in Rust are algebraic data types where each variant can carry different data; `match` forces exhaustive handling of all variants.
- **Option\<T\>** replaces null (Some/None) and **Result\<T, E\>** represents success/failure (Ok/Err) — the compiler ensures you handle every case.
