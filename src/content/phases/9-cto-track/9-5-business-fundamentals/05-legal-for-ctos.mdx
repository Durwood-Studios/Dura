## Why this matters

You are not a lawyer. You should not try to be one. But you are the person in the room who makes decisions that create legal exposure — every day. Which open-source library you include in the product, how you handle user data, what your engineers' employment agreements say, how you license your API. Each of these is a legal decision wearing an engineering costume.

The CTO who ignores legal fundamentals eventually has an expensive conversation with outside counsel that begins with "We have a problem." The CTO who understands the landscape catches issues early, asks the right questions, and brings problems to legal counsel while they are still cheap to fix.

## Open-Source Licensing

Open-source licensing is the legal minefield most CTOs walk through daily without realizing it. Every `npm install` adds license obligations to your product.

```javascript
const LICENSE_CATEGORIES = {
  permissive: {
    licenses: ["MIT", "BSD-2-Clause", "BSD-3-Clause", "Apache-2.0", "ISC"],
    obligations: "Include the license text and copyright notice in your distribution",
    commercialUse: "YES — no restrictions on commercial use or proprietary products",
    riskLevel: "LOW",
    example:
      "You use a MIT-licensed library in your SaaS product. " +
      "Include the MIT license text somewhere accessible. That's it.",
  },

  weakCopyleft: {

    obligations:

    commercialUse: "YES — with conditions on modifications to the library itself",
    riskLevel: "MEDIUM",
    example:
      "You use an LGPL library via its public API. Your code stays proprietary. " +
      "If you modify the LGPL library itself, those modifications must be shared.",
  },

  strongCopyleft: {
    licenses: ["GPL-2.0", "GPL-3.0", "AGPL-3.0"],
    obligations:
      "Any software that includes or links to GPL code must also be " +
      "distributed under GPL. AGPL extends this to network use (SaaS).",
    commercialUse: "YES — but the entire combined work must be open-sourced under GPL",
    riskLevel: "HIGH",
    example:
      "You include a GPL library in your proprietary SaaS backend. " +
      "Under AGPL, your entire backend may need to be open-sourced. " +
      "This is the 'copyleft contamination' that legal teams fear.",
  },
};

// The CTO's obligation: know what is in your dependency tree.
function auditLicenses(dependencies) {
  const results = { safe: [], review: [], blocked: [] };

  const permissive = new Set(["MIT", "BSD-2-Clause", "BSD-3-Clause", "Apache-2.0", "ISC"]);
  const copyleft = new Set(["GPL-2.0", "GPL-3.0", "AGPL-3.0"]);

  for (const dep of dependencies) {
    if (permissive.has(dep.license)) {
      results.safe.push(dep);
    } else if (copyleft.has(dep.license)) {
      results.blocked.push(dep);
    } else {
      results.review.push(dep);
    }
  }

  return {
    safe: results.safe.length,
    needsReview: results.review.length,
    blocked: results.blocked.length,
    blockedPackages: results.blocked.map((d) => `${d.name} (${d.license})`),
    action:
      results.blocked.length > 0
        ? "STOP — copyleft licenses detected. Review with legal before shipping."
        : results.review.length > 0
          ? "Review non-standard licenses with legal"
          : "All clear — permissive licenses only",
  };
}
```

## Data Privacy

GDPR, CCPA, and their successors are not legal abstractions — they are engineering requirements that affect database design, logging, API design, and system architecture.

// The deletion problem is harder than it looks
function assessDeletionCapability(systems) {
  const assessment = systems.map((system) => ({
    name: system.name,
    canDeleteUser: system.hasDeletionAPI,
    dataTypes: system.personalDataTypes,
    retentionPolicy: system.retentionDays,
    risk: !system.hasDeletionAPI ? "HIGH — manual deletion required" : "LOW",
  }));

  const incomplete = assessment.filter((s) => !s.canDeleteUser);

  return {
    systems: assessment,
    compliant: incomplete.length === 0,
    gaps: incomplete.map((s) => s.name),
    recommendation:
      incomplete.length > 0

## What you learned

- IP ownership depends on employment agreements and jurisdiction — contractors own their work by default unless a written assignment exists.
- Open-source license compliance is a daily engineering responsibility — strong copyleft (GPL, AGPL) in your dependency tree can force you to open-source proprietary code.
- Data privacy regulations like GDPR are architecture requirements that affect database design, deletion capabilities, and every third-party vendor that touches user data.
