## Why this matters

You can build the most elegant distributed system on the planet. If each customer costs $500 to acquire and generates $200 in revenue before churning, the company is dead. It's just a matter of when.

Unit economics is the language your CEO, CFO, and investors speak. When the board asks "Is the business model working?", they are not asking about your system architecture. They are asking whether the math works at the individual customer level. A CTO who cannot engage in this conversation is a CTO who gets surprised by layoffs.

The uncomfortable truth: many technical decisions that feel like engineering problems are actually unit economics problems. That expensive third-party API you integrated? It changed gross margin. That free tier you built? It shifted CAC. That reliability investment you made? It moved churn. You are already making unit economics decisions — the question is whether you are making them intentionally.

## Customer Acquisition Cost (CAC)

CAC answers a simple question: how much does it cost to get one paying customer?

```javascript
// CAC = Total Sales & Marketing Spend / New Customers Acquired

function calculateCAC(salesMarketingSpend, newCustomers) {
  if (newCustomers === 0) return Infinity; // spending money, getting nobody
  return salesMarketingSpend / newCustomers;
}

// Q1 example: $300K on marketing and sales, 150 new customers
const cac = calculateCAC(300_000, 150);
console.log(`CAC: $${cac}`); // CAC: $2000

// But CAC is more nuanced than one formula.
// Blended CAC includes all customers (organic + paid).
// Paid CAC isolates only paid acquisition channels.

function segmentedCAC(channels) {
  return channels.map((channel) => ({
    name: channel.name,
    cac: channel.spend / channel.customers,
    efficiency: channel.customers / channel.spend, // customers per dollar
  }));
}

const channels = [
  { name: "Google Ads", spend: 150_000, customers: 60 },
  { name: "Content Marketing", spend: 80_000, customers: 50 },
  { name: "Sales Team", spend: 70_000, customers: 40 },
];

console.table(segmentedCAC(channels));
// Google Ads: $2,500 CAC — expensive but scalable
// Content: $1,600 CAC — cheaper but slower
// Sales: $1,750 CAC — mid-range, relationship-driven
```

As CTO, your technical decisions directly affect CAC. A self-serve onboarding flow that actually works reduces the need for sales-assisted demos. A product that delivers value in the first session improves trial-to-paid conversion. A public API with great docs creates an organic acquisition channel. These are engineering investments with measurable CAC impact.

## Lifetime Value (LTV)

LTV answers the complementary question: how much revenue does one customer generate over their entire relationship?

```javascript
// Simple LTV = ARPU * Average Customer Lifetime

function simpleLTV(monthlyRevenue, avgLifetimeMonths) {
  return monthlyRevenue * avgLifetimeMonths;
}

// If a customer pays $200/month and stays 24 months on average:
console.log(`LTV: $${simpleLTV(200, 24)}`); // LTV: $4,800

// More precise: factor in gross margin and churn rate
function ltv(monthlyARPU, grossMarginPercent, monthlyChurnRate) {
  if (monthlyChurnRate === 0) return Infinity; // no one ever leaves (unlikely)
  const grossMarginMultiplier = grossMarginPercent / 100;
  return (monthlyARPU * grossMarginMultiplier) / monthlyChurnRate;
}

// $200/month, 75% gross margin, 4% monthly churn
const customerLTV = ltv(200, 75, 0.04);
console.log(`Gross-margin-adjusted LTV: $${customerLTV}`); // $3,750

// Cohort-based LTV is even better — track actual revenue per cohort
function cohortLTV(cohort) {
  const totalRevenue = cohort.months.reduce((sum, month) => sum + month.revenue, 0);
  const avgPerCustomer = totalRevenue / cohort.startingCustomers;
  return {
    cohortName: cohort.name,
    observedLTV: avgPerCustomer,
    monthsObserved: cohort.months.length,
    isComplete: cohort.activeCustomers === 0,
  };
}
```

## The LTV:CAC Ratio

This is the single ratio investors obsess over. It tells you whether each customer is profitable and by how much.

```javascript
function evaluateUnitEconomics(ltv, cac) {
  const ratio = ltv / cac;

  if (ratio < 1) {
    return {
      ratio,
      verdict: "UNSUSTAINABLE",
      meaning: "You lose money on every customer. Fix immediately.",
    };
  }
  if (ratio < 3) {
    return {
      ratio,
      verdict: "MARGINAL",
      meaning: "Profitable but thin. Vulnerable to cost increases or churn spikes.",
    };
  }
  if (ratio <= 5) {
    return {
      ratio,
      verdict: "HEALTHY",
      meaning: "Strong unit economics. The 3:1 to 5:1 range is the sweet spot.",
    };
  }
  return {
    ratio,
    verdict: "UNDERINVESTING",
    meaning: "You could grow faster. You are leaving acquisition on the table.",
  };
}

console.log(evaluateUnitEconomics(4800, 2000)); // 2.4x — MARGINAL
console.log(evaluateUnitEconomics(4800, 1200)); // 4.0x — HEALTHY
console.log(evaluateUnitEconomics(4800, 500)); // 9.6x — UNDERINVESTING
```

A ratio below 3:1 means the company is spending too much to acquire customers relative to what they generate. Above 5:1 often means the company is underinvesting in growth — it could afford to spend more on acquisition. The "perfect" ratio depends on capital efficiency and growth stage, but 3:1 is the benchmark VCs cite most often.

## Payback Period

Even if LTV:CAC is healthy, timing matters. Payback period is how many months until a customer's revenue covers their acquisition cost.

```javascript
function paybackMonths(cac, monthlyARPU, grossMarginPercent) {
  const monthlyGrossProfit = monthlyARPU * (grossMarginPercent / 100);
  if (monthlyGrossProfit <= 0) return Infinity;
  return Math.ceil(cac / monthlyGrossProfit);
}

// CAC $2,000, $200/month revenue, 75% gross margin
const months = paybackMonths(2000, 200, 75);
console.log(`Payback: ${months} months`); // 14 months

// Why this matters: if your payback period is 18 months but your
// average customer churns at 12 months, you never recover CAC.
// You are subsidizing every customer who leaves.

function isPaybackViable(cac, monthlyARPU, grossMargin, avgLifetimeMonths) {
  const payback = paybackMonths(cac, monthlyARPU, grossMargin);
  const headroom = avgLifetimeMonths - payback;
  return {
    paybackMonths: payback,
    customerLifetime: avgLifetimeMonths,
    profitableMonths: Math.max(0, headroom),
    viable: headroom > 0,
    assessment:
      headroom > 12
        ? "Strong — plenty of profitable months after payback"
        : headroom > 0
          ? "Tight — profitable but vulnerable to churn increases"
          : "Broken — customers churn before you recover acquisition cost",
  };
}

console.log(isPaybackViable(2000, 200, 75, 24));
// payback: 14, lifetime: 24, profitable months: 10 — Tight
```

## Gross Margin

Gross margin is what remains after you subtract the direct costs of delivering the product. For a SaaS company, this includes hosting, third-party APIs, support, and infrastructure — but not R&D, sales, or G&A.

```javascript
function grossMargin(revenue, costOfGoodsSold) {
  if (revenue === 0) return 0;
  return ((revenue - costOfGoodsSold) / revenue) * 100;
}

// Monthly revenue: $400K
// COGS: hosting $30K, third-party APIs $25K, support $20K, infrastructure $15K
const margin = grossMargin(400_000, 90_000);
console.log(`Gross margin: ${margin.toFixed(1)}%`); // 77.5%

// SaaS benchmarks:
// > 80% — excellent (software-like margins)
// 70-80% — good (typical SaaS)
// 60-70% — concerning (high infrastructure or API costs)
// < 60% — red flag (looks more like a services business)

// CTO lever: every dollar you save in COGS goes straight to gross margin.
function marginImpact(currentRevenue, currentCOGS, cogsReduction) {
  const currentMargin = grossMargin(currentRevenue, currentCOGS);
  const newMargin = grossMargin(currentRevenue, currentCOGS - cogsReduction);
  return {
    before: `${currentMargin.toFixed(1)}%`,
    after: `${newMargin.toFixed(1)}%`,
    improvement: `${(newMargin - currentMargin).toFixed(1)} percentage points`,
    annualImpact: `$${(cogsReduction * 12).toLocaleString()} saved annually`,
  };
}

// Migrating from a $25K/month third-party API to an in-house solution
console.log(marginImpact(400_000, 90_000, 25_000));
// Before: 77.5%, After: 83.8%, +6.3 points, $300K saved annually
```

This is where the CTO's infrastructure decisions become board-level talking points. A migration that saves $25K/month in API costs is not just an engineering project — it is a 6.3-point gross margin improvement that changes the company's valuation multiple.

## What you learned

- CAC, LTV, and their ratio determine whether each customer is a profitable investment or a subsidy the company cannot sustain.
- Payback period reveals timing risk — a healthy LTV:CAC ratio means nothing if customers churn before you recover acquisition cost.
- Gross margin is the CTO's direct lever on unit economics — every dollar saved in infrastructure costs flows straight to the bottom line and changes how the business is valued.
