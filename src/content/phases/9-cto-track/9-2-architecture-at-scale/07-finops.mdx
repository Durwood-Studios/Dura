## Why this matters

Cloud bills are the silent startup killer. The convenience of "spin up anything instantly" becomes a liability when nobody tracks what is running. Companies routinely discover they are spending 30-40% of their cloud budget on resources that are idle, oversized, or forgotten. FinOps — financial operations for cloud — is the discipline of making cloud spending visible, accountable, and optimized. As a technical leader, cloud cost is as much your responsibility as uptime.

## Understanding your cloud bill

Cloud pricing is deliberately complex. You pay for compute by the second, storage by the GB-month, data transfer by the GB (but only egress, never ingress), and every managed service has its own unit. The first step is decomposing your bill into categories you can act on.

```javascript
// Cloud bill analyzer
function analyzeCloudBill(lineItems) {
  const categories = {};

  lineItems.forEach((item) => {
    if (!categories[item.category]) {
      categories[item.category] = { total: 0, items: [] };
    }
    categories[item.category].total += item.monthlyCost;
    categories[item.category].items.push(item);
  });

  // Sort categories by cost
  const sorted = Object.entries(categories)
    .sort(([, a], [, b]) => b.total - a.total)
    .map(([name, data]) => ({
      category: name,
      monthlyCost: `$${Math.round(data.total)}`,
      topItems: data.items
        .sort((a, b) => b.monthlyCost - a.monthlyCost)
        .slice(0, 3)
        .map((i) => `${i.name}: $${Math.round(i.monthlyCost)}`),
    }));

  const totalMonthly = lineItems.reduce((sum, i) => sum + i.monthlyCost, 0);

  // The top 3 categories are usually 80%+ of spend
  const topThreePct =
    sorted.slice(0, 3).reduce((sum, c) => sum + parseInt(c.monthlyCost.replace("$", "")), 0) /
    totalMonthly;

  return {
    totalMonthly: `$${Math.round(totalMonthly).toLocaleString()}`,
    annualized: `$${Math.round(totalMonthly * 12).toLocaleString()}`,
    categories: sorted,
    topThreeAccountFor: `${Math.round(topThreePct * 100)}% of total spend`,
  };
}
```

## Reserved instances and savings plans

On-demand pricing is the default and most expensive option. If you know you will run a workload for 1-3 years, reserved instances or savings plans cut costs 30-60%. The tradeoff is commitment — you pay whether you use the capacity or not.

```javascript
// Reserved instance ROI calculator
function reservedInstanceROI(workload) {
  const {
    instanceType,
    onDemandHourly,
    reservedMonthly1Year,
    reservedMonthly3Year,
    currentUtilizationPercent,
    expectedGrowthPercent,
    instanceCount,
  } = workload;

  const onDemandMonthly = onDemandHourly * 730 * instanceCount; // 730 hours/month avg
  const onDemandYearly = onDemandMonthly * 12;

  const reserved1YearYearly = reservedMonthly1Year * 12 * instanceCount;
  const reserved3YearYearly = reservedMonthly3Year * 12 * instanceCount;

  // Risk: if you over-commit, you waste money
  // If utilization is below 70%, reserved instances may not pay off
  const utilizationRisk =
    currentUtilizationPercent < 70
      ? "WARNING: Current utilization is low. Rightsize before committing."
      : currentUtilizationPercent < 50
        ? "DANGER: Under 50% utilization. Reserved instances will waste money."
        : "Utilization supports commitment.";

  // Growth consideration: will you need more or fewer instances?
  const instancesInOneYear = Math.ceil(instanceCount * (1 + expectedGrowthPercent / 100));
  const instancesInThreeYears = Math.ceil(
    instanceCount * Math.pow(1 + expectedGrowthPercent / 100, 3)
  );

  return {
    instanceType,
    currentInstances: instanceCount,
    onDemandYearly: `$${Math.round(onDemandYearly).toLocaleString()}`,
    reserved1Year: {
      yearlyCost: `$${Math.round(reserved1YearYearly).toLocaleString()}`,
      savings: `$${Math.round(onDemandYearly - reserved1YearYearly).toLocaleString()}`,
      savingsPercent: `${Math.round((1 - reserved1YearYearly / onDemandYearly) * 100)}%`,
    },
    reserved3Year: {
      yearlyCost: `$${Math.round(reserved3YearYearly).toLocaleString()}`,
      savings: `$${Math.round(onDemandYearly - reserved3YearYearly).toLocaleString()}`,
      savingsPercent: `${Math.round((1 - reserved3YearYearly / onDemandYearly) * 100)}%`,
    },
    utilizationRisk,
    growthForecast: {
      instancesIn1Year: instancesInOneYear,
      instancesIn3Years: instancesInThreeYears,
      recommendation:
        instancesInThreeYears > instanceCount * 2
          ? "Fast growth — use 1-year terms and re-evaluate"
          : "Stable growth — 3-year terms are safe",
    },
  };
}

console.log(
  JSON.stringify(
    reservedInstanceROI({
      instanceType: "c6g.xlarge",
      onDemandHourly: 0.136,
      reservedMonthly1Year: 63,
      reservedMonthly3Year: 42,
      currentUtilizationPercent: 72,
      expectedGrowthPercent: 25,
      instanceCount: 8,
    }),
    null,
    2
  )
);
```

## Rightsizing

Rightsizing means matching instance sizes to actual usage. Most teams provision for peak and run at 20-30% utilization average. Dropping one instance size saves 40-50% on that resource.

```javascript
// Rightsizing analyzer
function rightsizeAnalysis(instances) {
  return instances.map((instance) => {
    const cpuHeadroom = 100 - instance.avgCPUPercent;
    const memHeadroom = 100 - instance.avgMemoryPercent;

    let recommendation;
    let savingsEstimate;

    if (instance.avgCPUPercent < 20 && instance.avgMemoryPercent < 20) {
      recommendation = "DOWNSIZE 2 TIERS";
      savingsEstimate = 0.75;
    } else if (instance.avgCPUPercent < 40 && instance.avgMemoryPercent < 40) {
      recommendation = "DOWNSIZE 1 TIER";
      savingsEstimate = 0.45;
    } else if (instance.peakCPUPercent > 90 || instance.peakMemoryPercent > 90) {
      recommendation = "UPSIZE or add autoscaling";
      savingsEstimate = -0.5; // Costs more
    } else {
      recommendation = "RIGHT-SIZED";
      savingsEstimate = 0;
    }

    return {
      name: instance.name,
      currentCost: `$${instance.monthlyCost}`,
      avgCPU: `${instance.avgCPUPercent}%`,
      peakCPU: `${instance.peakCPUPercent}%`,
      avgMemory: `${instance.avgMemoryPercent}%`,
      peakMemory: `${instance.peakMemoryPercent}%`,
      recommendation,
      estimatedNewCost: `$${Math.round(instance.monthlyCost * (1 - savingsEstimate))}`,
      monthlySavings: `$${Math.round(instance.monthlyCost * savingsEstimate)}`,
    };
  });
}
```

## Unit economics

The most important FinOps metric is cost per unit of value — cost per user, cost per transaction, cost per API call. Absolute cloud spend going up is fine if it goes up slower than revenue. Cloud spend per customer going up is a problem.

```javascript
// Unit economics tracker
function unitEconomics(months) {
  return months.map((month) => {
    const costPerUser = month.cloudSpend / month.activeUsers;
    const costPerTransaction = month.cloudSpend / month.transactions;
    const cloudAsPercentOfRevenue = (month.cloudSpend / month.revenue) * 100;

    return {
      month: month.name,
      activeUsers: month.activeUsers.toLocaleString(),
      cloudSpend: `$${month.cloudSpend.toLocaleString()}`,
      revenue: `$${month.revenue.toLocaleString()}`,
      costPerUser: `$${costPerUser.toFixed(3)}`,
      costPerTransaction: `$${costPerTransaction.toFixed(4)}`,
      cloudPercentOfRevenue: `${cloudAsPercentOfRevenue.toFixed(1)}%`,
      trend:
        cloudAsPercentOfRevenue > 30
          ? "CRITICAL — cloud costs eating margins"
          : cloudAsPercentOfRevenue > 15
            ? "WARNING — watch closely"
            : "HEALTHY",
    };
  });
}

console.log(
  unitEconomics([
    { name: "Jan", activeUsers: 50000, transactions: 200000, cloudSpend: 4200, revenue: 25000 },
    { name: "Feb", activeUsers: 65000, transactions: 280000, cloudSpend: 5100, revenue: 32000 },
    { name: "Mar", activeUsers: 80000, transactions: 350000, cloudSpend: 7800, revenue: 38000 },
    { name: "Apr", activeUsers: 95000, transactions: 420000, cloudSpend: 11500, revenue: 44000 },
  ])
);
```

## What you learned

- Cloud bills decompose into compute, storage, networking, and managed services — the top 3 categories typically account for 80%+ of spend and are where optimization efforts should focus
- Rightsizing (matching instance size to actual utilization) should precede reserved instance commitments — committing to oversized instances locks in waste for 1-3 years
- Unit economics (cost per user, cost per transaction) is the metric that matters — absolute cloud spend increasing is fine if it grows slower than revenue
