## Why this matters

The build vs. buy decision is the most common source of wasted engineering time. Teams build custom solutions for problems that mature SaaS products solve because building feels productive. Teams buy enterprise platforms for problems that need a 200-line internal tool because buying feels safe. Both instincts are wrong roughly half the time. You need a framework that turns this emotional decision into a financial one.

## Total cost of ownership

The purchase price of a SaaS tool is 30-50% of its total cost. The rest is integration, training, workflow changes, data migration, and the ongoing tax of working around the vendor's assumptions. Similarly, the initial build cost of a custom solution is 20-30% of its total cost. The rest is maintenance, on-call, upgrades, documentation, and hiring people who understand it.

```javascript
// Total Cost of Ownership calculator
function calculateTCO(buildOption, buyOption, yearsToModel) {
  // BUILD costs
  const buildCosts = {
    initialDevelopment: buildOption.engineerMonths * buildOption.monthlyEngineerCost,
    yearlyMaintenance:
      buildOption.maintenanceEngineerPercent * buildOption.monthlyEngineerCost * 12,
    yearlyInfrastructure: buildOption.monthlyInfraCost * 12,
    yearlyOnCall: buildOption.onCallHoursPerMonth * 12 * (buildOption.monthlyEngineerCost / 160),
    opportunityCost: buildOption.engineerMonths * buildOption.monthlyEngineerCost * 0.5,
    // What else could those engineers have built?
  };

  const buildYearOne =
    buildCosts.initialDevelopment +
    buildCosts.yearlyMaintenance +
    buildCosts.yearlyInfrastructure +
    buildCosts.yearlyOnCall +
    buildCosts.opportunityCost;

  const buildSubsequentYear =
    buildCosts.yearlyMaintenance + buildCosts.yearlyInfrastructure + buildCosts.yearlyOnCall;

  // BUY costs
  const buyCosts = {
    yearlyLicense: buyOption.monthlyLicenseCost * 12,
    integrationOneTime: buyOption.integrationEngineerWeeks * (buyOption.monthlyEngineerCost / 4),
    yearlyIntegrationMaintenance:
      buyOption.integrationMaintenanceHoursPerMonth * 12 * (buyOption.monthlyEngineerCost / 160),
    training: buyOption.trainingCostOneTime,
    dataMigration: buyOption.dataMigrationCost,
  };

  const buyYearOne =
    buyCosts.yearlyLicense +
    buyCosts.integrationOneTime +
    buyCosts.yearlyIntegrationMaintenance +
    buyCosts.training +
    buyCosts.dataMigration;

  const buySubsequentYear = buyCosts.yearlyLicense + buyCosts.yearlyIntegrationMaintenance;

  // Multi-year projection
  const projection = [];
  let buildTotal = 0;
  let buyTotal = 0;

  for (let year = 1; year <= yearsToModel; year++) {
    buildTotal += year === 1 ? buildYearOne : buildSubsequentYear;
    buyTotal += year === 1 ? buyYearOne : buySubsequentYear;
    projection.push({
      year,
      buildCumulative: Math.round(buildTotal),
      buyCumulative: Math.round(buyTotal),
      cheaper: buildTotal < buyTotal ? "BUILD" : "BUY",
    });
  }

  return {
    buildCosts,
    buyCosts,
    projection,
    breakEvenYear:
      projection.find((p, i) => i > 0 && p.cheaper !== projection[i - 1].cheaper)?.year ||
      (buildTotal < buyTotal
        ? `Build is always cheaper over ${yearsToModel} years`
        : `Buy is always cheaper over ${yearsToModel} years`),
  };
}

console.log(
  JSON.stringify(
    calculateTCO(
      {
        engineerMonths: 4,
        monthlyEngineerCost: 15000,
        maintenanceEngineerPercent: 0.15,
        monthlyInfraCost: 200,
        onCallHoursPerMonth: 8,
      },
      {
        monthlyLicenseCost: 2000,
        monthlyEngineerCost: 15000,
        integrationEngineerWeeks: 2,
        integrationMaintenanceHoursPerMonth: 4,
        trainingCostOneTime: 1000,
        dataMigrationCost: 2000,
      },
      5
    ),
    null,
    2
  )
);
```

## Vendor lock-in assessment

Lock-in is not binary. It is a spectrum from "swap in a weekend" to "18-month migration project." The cost of switching determines your negotiating power and your risk exposure if the vendor raises prices, gets acquired, or shuts down.

```javascript
// Vendor lock-in risk scorer
function assessLockIn(vendor) {
  let riskScore = 0;
  const risks = [];

  // Data portability
  if (vendor.dataExportFormat === "proprietary") {
    riskScore += 30;
    risks.push("Proprietary data format — migration requires custom ETL");
  } else if (vendor.dataExportFormat === "standard") {
    riskScore += 5;
  }

  // API surface area
  if (vendor.apiCallsInCodebase > 100) {
    riskScore += 20;
    risks.push(`${vendor.apiCallsInCodebase} API integrations — high switching cost`);
  } else if (vendor.apiCallsInCodebase > 20) {
    riskScore += 10;
  }

  // Contract terms
  if (vendor.annualContractRequired) {
    riskScore += 10;
    risks.push("Annual contract — locked for 12 months minimum");
  }
  if (vendor.dataDeletionOnCancellation) {
    riskScore += 15;
    risks.push("Data deleted on cancellation — must migrate before leaving");
  }

  // Market alternatives
  if (vendor.competitorCount < 3) {
    riskScore += 15;
    risks.push("Few competitors — limited alternatives if vendor fails");
  }

  // Team knowledge
  if (vendor.teamMembersWithExpertise < 2) {
    riskScore += 10;
    risks.push("Single point of knowledge — bus factor of 1");
  }

  const estimatedSwitchingCostMonths =
    vendor.apiCallsInCodebase * 0.02 +
    (vendor.dataExportFormat === "proprietary" ? 2 : 0.5) +
    (vendor.teamMembersWithExpertise < 2 ? 1 : 0);

  return {
    riskScore,
    riskLevel: riskScore > 60 ? "HIGH" : riskScore > 30 ? "MEDIUM" : "LOW",
    risks,
    estimatedSwitchingTimeMonths: Math.round(estimatedSwitchingCostMonths * 10) / 10,
    mitigations: [
      "Wrap vendor API in an internal abstraction layer",
      "Schedule quarterly data exports to portable format",
      "Document vendor-specific patterns for migration guide",
      "Maintain at least 2 team members with vendor expertise",
    ].slice(0, Math.ceil(riskScore / 20)),
  };
}

console.log(
  assessLockIn({
    name: "Analytics Platform",
    dataExportFormat: "proprietary",
    apiCallsInCodebase: 85,
    annualContractRequired: true,
    dataDeletionOnCancellation: false,
    competitorCount: 5,
    teamMembersWithExpertise: 1,
  })
);
```

## The maintenance burden trap

Custom software does not age gracefully. Dependencies need updating. Security patches need applying. The engineer who built it leaves, and the next person inherits code they did not write. The maintenance burden compounds year over year.

```javascript
// Maintenance burden estimator
function estimateMaintenanceBurden(project) {
  const {
    linesOfCode,
    dependencyCount,
    ageMonths,
    originalBuildersStillOnTeam,
    totalTeamSize,
    hasTests,
    hasDocumentation,
  } = project;

  // Base maintenance hours per month
  let monthlyHours = 0;

  // Code size factor: ~1 hour per 1000 lines per month
  monthlyHours += linesOfCode / 1000;

  // Dependency updates: ~0.5 hours per dependency per month
  monthlyHours += dependencyCount * 0.5;

  // Knowledge decay: maintenance cost increases when builders leave
  const knowledgeFactor = originalBuildersStillOnTeam > 0 ? 1 : 2.5; // 2.5x harder without original builders
  monthlyHours *= knowledgeFactor;

  // Documentation reduces maintenance
  if (hasDocumentation) monthlyHours *= 0.7;
  if (hasTests) monthlyHours *= 0.6;

  // Age factor: older systems accumulate tech debt
  const ageFactor = 1 + (ageMonths / 24) * 0.3; // 30% more per 2 years
  monthlyHours *= ageFactor;

  const monthlyEngineerCost = 15000;
  const monthlyCost = (monthlyHours / 160) * monthlyEngineerCost;

  return {
    estimatedMonthlyHours: Math.round(monthlyHours),
    estimatedMonthlyCost: `$${Math.round(monthlyCost).toLocaleString()}`,
    percentOfOneEngineer: `${Math.round((monthlyHours / 160) * 100)}%`,
    biggestFactor: !originalBuildersStillOnTeam
      ? "No original builders — knowledge transfer gap is the main cost driver"
      : dependencyCount > 20
        ? "High dependency count — update fatigue is the main cost driver"
        : "Code size — consider extracting or replacing less-used modules",
    fiveYearProjection: `$${Math.round(monthlyCost * 60 * 1.5).toLocaleString()} (includes 50% growth in complexity)`,
  };
}

console.log(
  estimateMaintenanceBurden({
    linesOfCode: 15000,
    dependencyCount: 35,
    ageMonths: 18,
    originalBuildersStillOnTeam: 0,
    totalTeamSize: 6,
    hasTests: true,
    hasDocumentation: false,
  })
);
```

## The decision framework

Build when the feature is your competitive advantage. Buy when it is infrastructure. Nobody wins customers because their internal logging system is custom-built.

```javascript
// Build vs. buy decision scorer
function buildVsBuyDecision(feature) {
  let buildScore = 0;
  let buyScore = 0;

  // Is this your competitive advantage?
  if (feature.isCoreDifferentiator) buildScore += 30;
  else buyScore += 20;

  // Does a mature solution exist?
  if (feature.matureSaaSExists) buyScore += 25;
  else buildScore += 15;

  // Customization needs
  if (feature.customizationNeeded > 0.7) buildScore += 20;
  else if (feature.customizationNeeded < 0.3) buyScore += 15;

  // Team capacity
  if (feature.teamHasBandwidth) buildScore += 10;
  else buyScore += 15;

  // Data sensitivity
  if (feature.dataCanLeaveNetwork === false) buildScore += 20;

  // Timeline pressure
  if (feature.neededWithinWeeks < 4) buyScore += 20;

  const total = buildScore + buyScore;
  return {
    buildScore,
    buyScore,
    recommendation: buildScore > buyScore ? "BUILD" : "BUY",
    confidence: `${Math.round((Math.abs(buildScore - buyScore) / total) * 100)}%`,
    reasoning: [
      feature.isCoreDifferentiator && "Core differentiator — own the code",
      !feature.isCoreDifferentiator &&
        "Not a differentiator — do not build commodity infrastructure",
      feature.matureSaaSExists && "Mature SaaS exists — leverage it",
      feature.customizationNeeded > 0.7 && "High customization needs — vendor will constrain you",
      !feature.teamHasBandwidth && "Team is at capacity — building adds load",
      feature.neededWithinWeeks < 4 && "Tight timeline — buying is faster to value",
    ].filter(Boolean),
  };
}

console.log(
  buildVsBuyDecision({
    name: "Authentication system",
    isCoreDifferentiator: false,
    matureSaaSExists: true,
    customizationNeeded: 0.2,
    teamHasBandwidth: false,
    dataCanLeaveNetwork: true,
    neededWithinWeeks: 3,
  })
);

console.log(
  buildVsBuyDecision({
    name: "ML recommendation engine",
    isCoreDifferentiator: true,
    matureSaaSExists: false,
    customizationNeeded: 0.9,
    teamHasBandwidth: true,
    dataCanLeaveNetwork: false,
    neededWithinWeeks: 12,
  })
);
```

## What you learned

- Total cost of ownership for build includes maintenance, on-call, knowledge transfer, and opportunity cost — typically 3-5x the initial development estimate over 3 years
- Vendor lock-in risk is measurable through data portability, API surface area, contract terms, and competitor count — wrap vendor APIs in abstraction layers to reduce switching costs
- Build when the feature is your competitive differentiator; buy when it is commodity infrastructure — nobody wins customers because of a custom logging system
