## Why this matters

Knowing Big O theory is table stakes. Applying it to code you have never seen before, under pressure, is the skill. This lesson closes the gap between "I understand the concepts" and "I can classify any function in 10 seconds." Every snippet here represents a pattern you will encounter in real codebases and technical interviews.

## The 10 snippets

Work through each snippet, predict the complexity, then read the explanation. The goal is not to get every one right immediately — the goal is to understand the reasoning until it becomes automatic.

---

**Snippet 1 — O(1): Hash map lookup**

```javascript
function getUserById(users, id) {
  return users[id]; // object property access
}
```

No iteration. No recursion. One operation regardless of how many users exist. Property access on a JavaScript object (hash map) is O(1).

---

**Snippet 2 — O(n): Single scan**

```javascript
function countEvens(arr) {
  let count = 0;
  for (const n of arr) {
    if (n % 2 === 0) count++;
  }
  return count;
}
```

One loop, visits each element once, O(1) work per element. Total: O(n).

---

**Snippet 3 — O(n²): Nested loops**

```javascript
function hasDuplicate(arr) {
  for (let i = 0; i < arr.length; i++) {
    for (let j = i + 1; j < arr.length; j++) {
      if (arr[i] === arr[j]) return true;
    }
  }
  return false;
}
```

Outer loop: n iterations. Inner loop: up to n-i-1 iterations. Sum across all i: n(n-1)/2 = O(n²). (Note: a Set-based solution would be O(n).)

---

**Snippet 4 — O(n log n): Merge sort recursion**

```javascript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;
  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));
  return merge(left, right); // O(n) to merge
}
```

Recursion depth: log n levels (halving each time). Work per level: O(n) to merge all elements at that level. Total: O(n log n).

---

**Snippet 5 — O(log n): Binary search**

```javascript
function binarySearch(arr, target) {
  let lo = 0,
    hi = arr.length - 1;
  while (lo <= hi) {
    const mid = Math.floor((lo + hi) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}
```

Each iteration cuts the search space in half. After k iterations, the search space is n / 2^k. When 2^k = n, k = log₂(n). Total: O(log n).

---

**Snippet 6 — O(n): Two pointer**

```javascript
function isPalindrome(str) {
  let left = 0,
    right = str.length - 1;
  while (left < right) {
    if (str[left] !== str[right]) return false;
    left++;
    right--;
  }
  return true;
}
```

Left pointer moves right at most n/2 times. Right pointer moves left at most n/2 times. Total steps: n. Two pointers, O(n).

---

**Snippet 7 — O(2^n): Naive recursion (Fibonacci)**

```javascript
function fib(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2); // two branches
}
```

Each call spawns two more calls. Call tree has roughly 2^n nodes. Each node does O(1) work. Total: O(2^n). At n=50, this would run longer than the age of the universe.

---

**Snippet 8 — O(n): Sliding window**

```javascript
function longestSubarrayUnderK(arr, k) {
  let left = 0,
    sum = 0,
    maxLen = 0;
  for (let right = 0; right < arr.length; right++) {
    sum += arr[right]; // right pointer always moves right
    while (sum > k) {
      sum -= arr[left];
      left++;
    } // left catches up
    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}
```

Right pointer moves from 0 to n-1: n steps. Left pointer only ever moves right and never passes right: at most n steps. Total: 2n = O(n).

---

**Snippet 9 — O(n × m): Two-array nested loop**

```javascript
function commonElements(a, b) {
  const result = [];
  for (const x of a) {
    // |a| = n iterations
    for (const y of b) {
      // |b| = m iterations each
      if (x === y) result.push(x);
    }
  }
  return result;
}
```

n × m comparisons. Not O(n²) unless n = m. A Set-based solution would be O(n + m).

---

**Snippet 10 — O(n!): Permutation generation**

```javascript
function permutations(arr) {
  if (arr.length <= 1) return [arr];
  const result = [];
  for (let i = 0; i < arr.length; i++) {
    const rest = [...arr.slice(0, i), ...arr.slice(i + 1)];
    for (const perm of permutations(rest)) {
      result.push([arr[i], ...perm]);
    }
  }
  return result;
}
```

For n items, there are n! permutations. The algorithm generates all of them, so it must do at least O(n!) work. Each level of recursion has n, n-1, n-2 … 1 branches. Product = n!.

## How to build the reflex

The patterns above reduce to five structural clues you can spot in seconds:

1. **No loop, no recursion** → O(1)
2. **One loop or one set of pointers moving through n items once** → O(n)
3. **Loop that halves** → O(log n)
4. **Outer O(n) with inner O(log n) sort or search** → O(n log n)
5. **Nested loops both dependent on n** → O(n²) — unless the inner loop is bounded by something small

When you see a new function: trace the loops, find the dominant nest level, check if anything halves, and you have the answer.

<SandboxExercise language="javascript">
{`// analyzeAlgorithm calls fn(size) for each size and infers complexity
// from the ratio of consecutive operation counts.

function analyzeAlgorithm(fn, sizes) {
const opCounts = sizes.map(n => fn(n));

// Infer complexity from ratio between consecutive sizes
let likelyComplexity = "unknown";
if (sizes.length >= 2) {
const n1 = sizes[sizes.length - 2];
const n2 = sizes[sizes.length - 1];
const c1 = opCounts[opCounts.length - 2];
const c2 = opCounts[opCounts.length - 1];
const ratio = c2 / c1;
const sizeRatio = n2 / n1;

    if (ratio < 1.1) likelyComplexity = "O(1)";
    else if (Math.abs(ratio - Math.log2(sizeRatio)) < 0.5) likelyComplexity = "O(log n)";
    else if (Math.abs(ratio - sizeRatio) < sizeRatio * 0.2) likelyComplexity = "O(n)";
    else if (Math.abs(ratio - sizeRatio * Math.log2(sizeRatio)) < sizeRatio * Math.log2(sizeRatio) * 0.2) likelyComplexity = "O(n log n)";
    else if (Math.abs(ratio - sizeRatio ** 2) < sizeRatio ** 2 * 0.2) likelyComplexity = "O(n²)";
    else likelyComplexity = "superlinear (possibly O(2^n))";

}

return { sizes, opCounts, likelyComplexity };
}

const sizes = [10, 100, 1000];

// O(1) — constant
const r1 = analyzeAlgorithm(n => 1, sizes);
console.log("Hash lookup:", r1.likelyComplexity, r1.opCounts);

// O(n) — single scan
const r2 = analyzeAlgorithm(n => n, sizes);
console.log("Single scan:", r2.likelyComplexity, r2.opCounts);

// O(n²) — nested loops
const r3 = analyzeAlgorithm(n => n \* n, sizes);
console.log("Nested loops:", r3.likelyComplexity, r3.opCounts);

// O(log n) — binary search
const r4 = analyzeAlgorithm(n => Math.floor(Math.log2(n)), sizes);
console.log("Binary search:", r4.likelyComplexity, r4.opCounts);

// O(n log n) — merge sort
const r5 = analyzeAlgorithm(n => n \* Math.floor(Math.log2(n)), sizes);
console.log("Merge sort:", r5.likelyComplexity, r5.opCounts);`}

## What you learned

- O(1) comes from no iteration — hash lookup, array index access.
- O(n) comes from a single pass through n items — linear scan, two pointers, sliding window.
- O(log n) comes from halving the problem space each step — binary search, tree traversal.
- O(n log n) is divide-and-conquer with O(n) merge work — merge sort, heap sort.
- O(n²) is nested loops both dependent on n — duplicate detection, naive all-pairs.
- O(2^n) is two recursive branches on the same input — naive Fibonacci, all subsets.
- O(n!) is generating all orderings — permutation algorithms.
