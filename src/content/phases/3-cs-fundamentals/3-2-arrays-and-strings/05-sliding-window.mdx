## Why this matters

Problems asking for the "best subarray" or "longest substring satisfying condition X" almost always have an O(n²) brute-force solution: try every start, expand to every end. The sliding window cuts that to O(n) by reusing work. Rather than recomputing from scratch at each start position, you maintain a window of state that is updated incrementally as it slides forward.

## Fixed-size window

The simplest variant: the window has a fixed size `k`. Slide it from left to right, maintaining a running value.

**Naive approach — O(n × k):**

```javascript
function maxSubarraySumNaive(arr, k) {
  let maxSum = -Infinity;
  for (let i = 0; i <= arr.length - k; i++) {
    let windowSum = 0;
    for (let j = i; j < i + k; j++) {
      windowSum += arr[j]; // recompute sum from scratch each window
    }
    maxSum = Math.max(maxSum, windowSum);
  }
  return maxSum;
}
```

For `n = 10,000` and `k = 1,000`, that is 10 million additions.

**Sliding window — O(n):**

```javascript
function maxSubarraySumWindow(arr, k) {
  // Build the first window
  let windowSum = 0;
  for (let i = 0; i < k; i++) windowSum += arr[i];
  let maxSum = windowSum;

  // Slide: subtract the element leaving, add the element entering
  for (let i = k; i < arr.length; i++) {
    windowSum += arr[i]; // add the new right element
    windowSum -= arr[i - k]; // remove the old left element
    maxSum = Math.max(maxSum, windowSum);
  }
  return maxSum;
}
```

Each element is added once and removed once — exactly n additions and n subtractions total. **O(n)** regardless of `k`.

## The key insight

When you slide the window one position right:

```
Before: [arr[0], arr[1], ..., arr[k-1]]  sum = S
After:  [arr[1], arr[2], ..., arr[k]]    sum = S - arr[0] + arr[k]
```

One subtraction, one addition. No inner loop. The "state" (the current sum) is updated incrementally rather than recomputed.

## Variable-size window

When the window size is not fixed, one pointer expands the window (right) and another shrinks it (left). The window grows as long as the constraint is satisfied and shrinks when violated.

**Longest substring without repeating characters:**

```javascript
function longestUniqueSubstring(str) {
  const seen = new Map(); // char → most recent index
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < str.length; right++) {
    const ch = str[right];

    // If we've seen this char inside the current window, shrink from the left
    if (seen.has(ch) && seen.get(ch) >= left) {
      left = seen.get(ch) + 1; // jump left past the duplicate
    }

    seen.set(ch, right); // record/update position
    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}

longestUniqueSubstring("abcabcbb"); // 3 ("abc")
longestUniqueSubstring("bbbbb"); // 1 ("b")
longestUniqueSubstring("pwwkew"); // 3 ("wke")
```

`right` moves forward on every iteration. `left` only moves forward — never backward. Each character is entered into the map once and causes at most one left-pointer jump. **O(n)** total.

## Tracking window state

For more complex constraints, the window maintains richer state. Common structures:

```javascript
// Frequency map — track character counts in the current window
const freq = new Map();

// When the window expands (right moves in):
const ch = str[right];
freq.set(ch, (freq.get(ch) ?? 0) + 1);

// When the window shrinks (left moves out):
const outgoing = str[left];
freq.set(outgoing, freq.get(outgoing) - 1);
if (freq.get(outgoing) === 0) freq.delete(outgoing);
left++;
```

This pattern supports problems like "longest substring with at most k distinct characters":

```javascript
function longestKDistinct(str, k) {
  const freq = new Map();
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < str.length; right++) {
    const ch = str[right];
    freq.set(ch, (freq.get(ch) ?? 0) + 1);

    // Shrink until we have at most k distinct characters
    while (freq.size > k) {
      const outgoing = str[left];
      freq.set(outgoing, freq.get(outgoing) - 1);
      if (freq.get(outgoing) === 0) freq.delete(outgoing);
      left++;
    }

    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}

longestKDistinct("eceba", 2); // 3 ("ece")
longestKDistinct("aa", 1); // 2 ("aa")
```

## When to use sliding window

Reach for sliding window when the problem involves:

- A **contiguous subarray or substring** (not arbitrary elements).
- A **condition** the subarray must satisfy (sum ≤ target, at most k distinct chars, no repeats).
- Finding the **longest, shortest, or best** such subarray.

If the problem asks about non-contiguous elements or arbitrary subsets, sliding window will not work — you need dynamic programming or backtracking instead.

## Why O(n) — both pointers move forward only

The efficiency proof is simple: `right` advances from 0 to n-1 — n steps. `left` starts at 0 and can only increase, never exceeding `right` — at most n steps. Together, both pointers make at most 2n moves. **O(n)**.

## What you learned

- The **fixed-size sliding window** updates state in O(1) per step by subtracting the element leaving and adding the element entering — total **O(n)** vs O(n×k) naive.
- The **variable-size window** expands (right moves) while the constraint holds and shrinks (left moves) when the constraint is violated.
- **Window state** — sum, count, frequency map — is updated incrementally, never recomputed from scratch.
- Both pointers move **forward only**, guaranteeing O(n) total regardless of how many shrink steps occur.
- Sliding window applies when you need the best **contiguous** subarray or substring satisfying a condition; use other techniques for non-contiguous selections.
