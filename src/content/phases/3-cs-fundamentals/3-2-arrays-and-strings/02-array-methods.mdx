## Why this matters

Every JavaScript codebase is full of `map`, `filter`, and `reduce`. They make intent clearer than for loops, compose naturally, and eliminate a category of off-by-one bugs. But they carry hidden complexity costs that matter when your dataset grows. Knowing when to use them — and when not to — is the difference between clean code and slow code.

## map — transform every element

`map` applies a function to every element and returns a **new array of the same length**. The original array is not modified.

```javascript
const prices = [10, 25, 7, 42];
const withTax = prices.map((p) => p * 1.08);
// [10.8, 27, 7.56, 45.36]
// prices is unchanged
```

Complexity: **O(n)** — one callback per element, one new array allocation.

The callback receives `(element, index, array)`. You rarely need index and array, but they are available:

```javascript
const labeled = prices.map((p, i) => `Item ${i}: $${p.toFixed(2)}`);
```

## filter — keep matching elements

`filter` returns a **new array** containing only elements for which the callback returns a truthy value. The output can be shorter than the input.

```javascript
const scores = [45, 82, 91, 33, 78, 60];
const passing = scores.filter((s) => s >= 60);
// [82, 91, 78, 60]
```

Complexity: **O(n)** — every element is tested once, the new array has at most n elements.

## reduce — collapse to a single value

`reduce` accumulates all elements into one result using a callback and an initial value. It is the most powerful and most often misunderstood method.

```javascript
const nums = [1, 2, 3, 4, 5];
const sum = nums.reduce((acc, n) => acc + n, 0);
// acc starts at 0, then: 1, 3, 6, 10, 15
// result: 15
```

`reduce` can replicate `map` and `filter`, but that is usually a readability antipattern. Use `reduce` when you genuinely need to transform a list into a single value: sum, max, frequency map, or a grouped object.

```javascript
// Group items by category with reduce
const items = [
  { name: "apple", category: "fruit" },
  { name: "broccoli", category: "vegetable" },
  { name: "banana", category: "fruit" },
];
const grouped = items.reduce((acc, item) => {
  if (!acc[item.category]) acc[item.category] = [];
  acc[item.category].push(item.name);
  return acc;
}, {});
// { fruit: ["apple", "banana"], vegetable: ["broccoli"] }
```

Complexity: **O(n)** — one callback per element.

## find, findIndex, some, every

These are all **O(n) worst case** but **short-circuit** early when possible:

```javascript
const users = [
  { id: 1, active: false },
  { id: 2, active: true },
  { id: 3, active: true },
];

users.find((u) => u.active); // { id: 2, active: true } — stops at index 1
users.findIndex((u) => u.active); // 1 — stops at index 1
users.some((u) => u.active); // true — stops at index 1
users.every((u) => u.active); // false — stops at index 0
```

Short-circuiting means that in the best case these are O(1). They are O(n) only when no match is found (`find`, `some`) or all elements match (`every`).

## sort — the classic gotcha

`Array.sort()` sorts **in place** and returns the same array. Its complexity is **O(n log n)**.

```javascript
const nums = [10, 9, 2, 1, 100];
nums.sort(); // [1, 10, 100, 2, 9] ← WRONG
```

The default sort converts elements to **strings** and sorts lexicographically. `"100" < "2"` because `"1" < "2"`. Always provide a comparator for numbers:

```javascript
nums.sort((a, b) => a - b); // ascending: [1, 2, 9, 10, 100]
nums.sort((a, b) => b - a); // descending: [100, 10, 9, 2, 1]
```

A comparator returning a negative number puts `a` before `b`; positive puts `b` before `a`; zero means equal.

## Method chaining and performance

Methods chain naturally because each returns an array:

```javascript
const result = data
  .filter((x) => x.active) // O(n) → m results
  .map((x) => x.value * 2); // O(m) → m results
```

The total cost is O(n) + O(m) — **two separate passes** over the data. For large datasets you can combine into a single `reduce`:

```javascript
// Single pass: filter + map together
const result = data.reduce((acc, x) => {
  if (x.active) acc.push(x.value * 2);
  return acc;
}, []);
```

This matters when n is in the hundreds of thousands. For typical UI datasets (< 10,000 items), the chained version is fine and far more readable.

## When NOT to use functional methods

Inside a tight inner loop (called millions of times), functional methods have overhead: function call per element, closure allocation, potential garbage collection. In those cases, a `for` loop is appropriate:

```javascript
// Performance-critical: use a for loop
let sum = 0;
for (let i = 0; i < arr.length; i++) {
  sum += arr[i]; // no callback overhead
}
```

The rest of the time — which is almost always — prefer `map`, `filter`, and `reduce`.

## flat and flatMap

`flat(depth)` flattens nested arrays up to `depth` levels deep. `flatMap` applies a mapping function and flattens one level — equivalent to `.map(...).flat(1)` but in a single pass.

```javascript
[[1, 2], [3, 4], [5]].flat(); // [1, 2, 3, 4, 5]
["hello", "world"].flatMap((w) => w.split("")); // ["h","e","l","l","o","w","o","r","l","d"]
```

{`// Given an array of score objects, implement three pipeline stages
// using only map/filter/reduce — then compose them.

const items = [
{ name: "Alice", score: 88, category: "math" },
{ name: "Bob", score: 42, category: "science" },
{ name: "Carol", score: 95, category: "math" },
{ name: "Dave", score: 67, category: "science" },
{ name: "Eve", score: 100, category: "math" },
{ name: "Frank", score: 55, category: "science" },
];

// Stage 1: keep only items matching a given category
function filterByCategory(items, cat) {
return items.filter((item) => item.category === cat);
}

// Stage 2: normalize scores to 0.0–1.0 range
function normalizeScores(items, maxScore) {
return items.map((item) => ({ ...item, score: item.score / maxScore }));
}

// Stage 3: collapse to a summary object
function summarize(items) {
return items.reduce(
(acc, item) => {
acc.count++;
acc.totalScore += item.score;
acc.avgScore = acc.totalScore / acc.count;
if (!acc.topScorer || item.score > acc.topScore) {
acc.topScorer = item.name;
acc.topScore = item.score;
}
return acc;
},
{ count: 0, totalScore: 0, avgScore: 0, topScorer: null, topScore: -Infinity }
);
}

// Compose: filter math students, normalize to 100-point max, summarize
const mathSummary = summarize(normalizeScores(filterByCategory(items, "math"), 100));
console.log("Math summary:", mathSummary);

const scienceSummary = summarize(normalizeScores(filterByCategory(items, "science"), 100));
console.log("Science summary:", scienceSummary);
`}

## What you learned

- `map` transforms every element into a new array of the **same length** — O(n).
- `filter` returns a **subset** matching a predicate — O(n).
- `reduce` **collapses** a list to a single value — O(n) and the most flexible of the three.
- The default `sort` is **lexicographic** — always pass `(a, b) => a - b` for numeric sorting.
- Chained methods make **multiple passes**; a single `reduce` can combine filter + map in one O(n) pass.
- `find`, `some`, and `every` **short-circuit** — O(1) best case, O(n) worst case.
