## Why this matters

Pattern recognition is the core skill in algorithmic problem-solving. You read a problem, identify the structure (sorted? contiguous? two-element constraint?), select a pattern (two pointers? sliding window? binary search?), and implement. These five problems cover all the patterns from this module. Work through each one systematically: understand the problem, identify the pattern, implement, verify, analyze.

## Problem 1: Move zeros to end (two pointers)

**Problem:** Given an array, move all zeros to the end while preserving the relative order of non-zero elements. Do it **in-place** with O(1) extra space.

```javascript
// Example: [0, 1, 0, 3, 12] → [1, 3, 12, 0, 0]
```

**Identify the pattern:** We need to partition the array: non-zeros on the left, zeros on the right. This is a classic **fast/slow pointer** problem — the slow pointer marks where the next non-zero should go; the fast pointer scans for non-zeros.

**Implement:**

```javascript
function moveZeros(arr) {
  let slow = 0; // next write position for non-zero elements

  // Fast pointer finds non-zeros; slow pointer places them
  for (let fast = 0; fast < arr.length; fast++) {
    if (arr[fast] !== 0) {
      arr[slow] = arr[fast];
      slow++;
    }
  }

  // Fill the rest with zeros
  while (slow < arr.length) {
    arr[slow] = 0;
    slow++;
  }

  return arr;
}
```

**Verify:**

```javascript
moveZeros([0, 1, 0, 3, 12]); // [1, 3, 12, 0, 0]
moveZeros([0, 0, 0]); // [0, 0, 0]
moveZeros([1, 2, 3]); // [1, 2, 3]
moveZeros([]); // []
```

**Complexity:** O(n) time — one pass. O(1) space — in-place.

## Problem 2: All pairs with given sum in sorted array (two pointers)

**Problem:** Given a sorted array, find all unique pairs that sum to a target. Return an array of `[a, b]` pairs.

**Identify the pattern:** "Two elements in a sorted array with a constraint" → **inward two pointers**.

```javascript
function allPairs(arr, target) {
  const pairs = [];
  let left = 0;
  let right = arr.length - 1;

  while (left < right) {
    const sum = arr[left] + arr[right];
    if (sum === target) {
      pairs.push([arr[left], arr[right]]);
      // Skip duplicates on both sides
      while (left < right && arr[left] === arr[left + 1]) left++;
      while (left < right && arr[right] === arr[right - 1]) right--;
      left++;
      right--;
    } else if (sum < target) {
      left++;
    } else {
      right--;
    }
  }

  return pairs;
}

allPairs([1, 2, 3, 4, 5, 6], 7); // [[1,6],[2,5],[3,4]]
allPairs([1, 1, 2, 2, 3, 3], 4); // [[1,3],[2,2]]
allPairs([1, 2, 3], 10); // []
```

**Complexity:** O(n) time — each pointer moves at most n steps. O(k) output space for k pairs.

## Problem 3: Longest substring with at most k distinct characters (sliding window)

**Problem:** Find the length of the longest substring containing at most k distinct characters.

**Identify the pattern:** "Longest contiguous substring satisfying a character constraint" → **variable sliding window** with a frequency map.

```javascript
function longestKDistinct(str, k) {
  if (k === 0) return 0;
  const freq = new Map();
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < str.length; right++) {
    const ch = str[right];
    freq.set(ch, (freq.get(ch) ?? 0) + 1);

    // Shrink until at most k distinct characters remain
    while (freq.size > k) {
      const outgoing = str[left];
      freq.set(outgoing, freq.get(outgoing) - 1);
      if (freq.get(outgoing) === 0) freq.delete(outgoing);
      left++;
    }

    maxLen = Math.max(maxLen, right - left + 1);
  }

  return maxLen;
}

longestKDistinct("eceba", 2); // 3 ("ece")
longestKDistinct("aa", 1); // 2 ("aa")
longestKDistinct("aabbcc", 2); // 4 ("aabb" or "bbcc")
longestKDistinct("abc", 0); // 0
longestKDistinct("", 3); // 0
```

**Complexity:** O(n) time — right and left each move at most n steps. O(k) space for the frequency map.

## Problem 4: Search in rotated sorted array (modified binary search)

**Problem:** A sorted array was rotated at an unknown pivot. Find the index of a target value, or -1 if not found.

```javascript
// Example: [4, 5, 6, 7, 0, 1, 2], target = 0 → 4
```

**Identify the pattern:** Sorted array, find target → **binary search** — but adapted for rotation. Key insight: even in a rotated array, at least one half of any sub-range is still sorted. Use that to decide which half to search.

```javascript
function searchRotated(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);
    if (arr[mid] === target) return mid;

    // Determine which half is properly sorted
    if (arr[left] <= arr[mid]) {
      // Left half is sorted
      if (arr[left] <= target && target < arr[mid]) {
        right = mid - 1; // target is in the sorted left half
      } else {
        left = mid + 1; // target must be in the right half
      }
    } else {
      // Right half is sorted
      if (arr[mid] < target && target <= arr[right]) {
        left = mid + 1; // target is in the sorted right half
      } else {
        right = mid - 1; // target must be in the left half
      }
    }
  }

  return -1;
}

searchRotated([4, 5, 6, 7, 0, 1, 2], 0); // 4
searchRotated([4, 5, 6, 7, 0, 1, 2], 3); // -1
searchRotated([1], 0); // -1
searchRotated([1, 3], 3); // 1
```

**Complexity:** O(log n) — still halving the search space each step.

## Problem 5: Anagram check (frequency map)

**Problem:** Given two strings, determine if they are anagrams — same characters in the same frequencies, regardless of order.

**Identify the pattern:** Character frequency counting. Build a frequency map from one string, subtract counts using the second string.

```javascript
function anagramCheck(s1, s2) {
  if (s1.length !== s2.length) return false;

  const freq = {};
  // Count up for s1
  for (const ch of s1) {
    freq[ch] = (freq[ch] ?? 0) + 1;
  }
  // Count down for s2
  for (const ch of s2) {
    if (!freq[ch]) return false; // char not in s1, or exhausted
    freq[ch]--;
  }

  return true; // all counts balanced to zero
}

anagramCheck("listen", "silent"); // true
anagramCheck("triangle", "integral"); // true
anagramCheck("hello", "world"); // false
anagramCheck("rat", "car"); // false
anagramCheck("", ""); // true
anagramCheck("a", "ab"); // false — different lengths
```

**Complexity:** O(n) time — two passes over strings of length n. O(k) space where k is the alphabet size (26 for lowercase ASCII, constant).

## Pattern recognition guide

Use this table when you encounter a new problem:

| Signal in the problem                        | Pattern to consider        |
| -------------------------------------------- | -------------------------- |
| "Find two elements that..." in sorted array  | Two pointers (inward scan) |
| "Longest/shortest contiguous subarray..."    | Sliding window             |
| "Find in sorted array / minimum valid value" | Binary search              |
| "Check if two strings have same chars..."    | Frequency map              |
| "Move / partition / compact in-place"        | Fast/slow pointer          |
| "Combine all elements into one result"       | reduce                     |
| "Transform all elements"                     | map                        |

## What you learned

- **Move zeros** — fast/slow pointers: slow marks write position, fast finds non-zero values — O(n) time, O(1) space.
- **All pairs** — inward two pointers on a sorted array, with duplicate-skipping logic — O(n) time.
- **Longest k-distinct** — variable sliding window with a frequency map to track distinct characters — O(n) time.
- **Rotated array search** — modified binary search: identify the sorted half, eliminate the other — O(log n) time.
- **Anagram check** — build a frequency map from s1, decrement with s2, verify all counts are zero — O(n) time, O(k) space.
- The skill that transfers beyond these five problems: **read the problem, identify the signal, select the pattern, implement, verify edge cases**.
