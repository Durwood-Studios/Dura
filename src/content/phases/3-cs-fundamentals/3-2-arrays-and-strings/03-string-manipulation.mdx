## Why this matters

String problems appear in nearly every coding interview and show up constantly in real work — parsing user input, building URLs, processing text data. The hidden danger is that JavaScript strings are immutable: every "modification" creates a new string. Do it in a loop and you accidentally write an O(n²) algorithm. Understanding immutability lets you choose the O(n) path every time.

## Strings as character sequences

A string is a sequence of characters, each accessible by index — just like a read-only array.

```javascript
const word = "hello";
word[0]; // "h"
word[4]; // "o"
word.length; // 5
word[0] = "j"; // silently fails — strings are immutable
```

You can read any character in O(1) the same way you read `arr[i]`. What you cannot do is modify a character in place — you must build a new string.

## Immutability and the O(n²) trap

Every time you use `+` or `+=` on a string, JavaScript allocates a new string that is the concatenation of both sides. In a loop, the string grows with each iteration:

```javascript
// Naive character-by-character build — O(n²)
function reverseNaive(str) {
  let result = "";
  for (let i = str.length - 1; i >= 0; i--) {
    result += str[i]; // each += copies the ENTIRE current result
  }
  return result;
}
```

For a string of length n, iteration 1 copies 1 character, iteration 2 copies 2, ..., iteration n copies n. Total copies: 1 + 2 + 3 + ... + n = **n(n+1)/2 = O(n²)**.

## The string-builder fix: push and join

The O(n) alternative is the **string-builder pattern**: collect characters into an array, then join once at the end. `Array.push` is O(1) amortized; `Array.join` is a single O(n) pass.

```javascript
// O(n) string builder
function reverseOptimal(str) {
  const chars = [];
  for (let i = str.length - 1; i >= 0; i--) {
    chars.push(str[i]); // O(1) per push
  }
  return chars.join(""); // one O(n) join at the end
}
```

Total: n pushes × O(1) + one join × O(n) = **O(n)**.

## Common string algorithms

**Palindrome check** — a string reads the same forwards and backwards:

```javascript
function isPalindrome(str) {
  const clean = str.toLowerCase().replace(/[^a-z0-9]/g, "");
  let left = 0;
  let right = clean.length - 1;
  while (left < right) {
    if (clean[left] !== clean[right]) return false;
    left++;
    right--;
  }
  return true;
}
isPalindrome("A man a plan a canal Panama"); // true
```

**Anagram detection** — two strings are anagrams if they contain the same characters in the same frequencies:

```javascript
function isAnagram(s1, s2) {
  if (s1.length !== s2.length) return false;
  const freq = {};
  for (const ch of s1) freq[ch] = (freq[ch] ?? 0) + 1;
  for (const ch of s2) {
    if (!freq[ch]) return false;
    freq[ch]--;
  }
  return true;
}
isAnagram("listen", "silent"); // true
isAnagram("hello", "world"); // false
```

**Character frequency counting** — foundation for anagram, compression, and encoding problems:

```javascript
function charFrequency(str) {
  const freq = new Map();
  for (const ch of str) {
    freq.set(ch, (freq.get(ch) ?? 0) + 1);
  }
  return freq;
}
charFrequency("banana"); // Map { b→1, a→3, n→2 }
```

## Essential string methods

```javascript
const s = "  Hello, World!  ";

s.trim(); // "Hello, World!" — remove leading/trailing whitespace
s.toLowerCase(); // "  hello, world!  "
s.toUpperCase(); // "  HELLO, WORLD!  "
s.includes("World"); // true — O(n)
s.indexOf("o"); // 4 — first occurrence, O(n)
s.slice(2, 7); // "Hello" — non-mutating, O(k)
s.split(", "); // ["  Hello", "World!  "] — O(n)
["Hello", "World"].join("-"); // "Hello-World" — O(n)
`Hello ${"World"}`; // "Hello World" — template literal, cleaner than +
```

`split` and `join` are complementary: split breaks a string into an array; join collapses an array into a string. Together they are the foundation of the string-builder pattern.

## Unicode complications

JavaScript strings are internally UTF-16. Most characters are one **code unit** (one `str[i]` index). But emoji and rare characters are **surrogate pairs** — two code units that represent one character.

```javascript
const emoji = "😀";
emoji.length; // 2 — two code units, not one character
emoji[0]; // "\uD83D" — half of a surrogate pair, meaningless alone
[...emoji]; // ["😀"] — spread uses code points correctly
emoji.codePointAt(0); // 128512 — the correct Unicode scalar value
```

For most string problems in interviews and courses, assume ASCII (a–z, A–Z, 0–9). For production code, use `[...str]` or `for...of` instead of `for i` loops when Unicode correctness matters.

## What you learned

- Strings are **immutable** in JavaScript — every concatenation creates a new string.
- `str += char` in a loop is **O(n²)** because each iteration copies the entire accumulated string.
- The **string-builder pattern** (push to array, join once) reduces this to **O(n)**.
- Palindrome, anagram, and character-frequency algorithms are foundational — all run in O(n).
- JavaScript strings are **UTF-16**: emoji and rare characters are surrogate pairs with `length === 2`; use `for...of` or spread for Unicode-safe iteration.
