## Why this matters

The naive solution to many array problems is a nested loop: try every pair of elements, giving O(n²). Two pointers cut that to O(n) by exploiting structure — usually sortedness or symmetry. Once you see the pattern, you will recognize it in dozens of problems and reach for it instinctively.

## The core idea

Two pointers are two indices that move through an array — sometimes from opposite ends toward the middle, sometimes at different speeds through the same end. The key is that each pointer moves forward only, and the two together traverse the array in a single pass.

```javascript
// Inward scan: pointers start at opposite ends and move toward each other
let left = 0;
let right = arr.length - 1;
while (left < right) {
  // do something with arr[left] and arr[right]
  left++; // move left inward, or
  right--; // move right inward — based on some condition
}
```

Because `left` starts at 0 and can only increase, and `right` starts at `n-1` and can only decrease, they meet after at most n steps total. **O(n)** — not O(n²).

## Palindrome check with inward scan

A string is a palindrome if `str[0] === str[n-1]`, `str[1] === str[n-2]`, and so on. Two inward pointers check each pair without building a reversed copy.

```javascript
function isPalindrome(str) {
  let left = 0;
  let right = str.length - 1;
  while (left < right) {
    if (str[left] !== str[right]) return false;
    left++;
    right--;
  }
  return true;
}

isPalindrome("racecar"); // true  — 3 comparisons
isPalindrome("hello"); // false — fails at l vs o
isPalindrome("a"); // true  — single char, loop never runs
```

Compare to the naive approach: `str === str.split("").reverse().join("")`. That is O(n) space and allocates two new arrays. Two pointers are O(1) space.

## Pair sum in a sorted array

Given a sorted array, find two elements that sum to a target. Brute force tries every pair: O(n²). With two pointers, sorted order tells you which direction to move:

```javascript
function twoSumSorted(arr, target) {
  let left = 0;
  let right = arr.length - 1;
  while (left < right) {
    const sum = arr[left] + arr[right];
    if (sum === target) return { left, right };
    if (sum < target)
      left++; // too small — need a bigger value, move left up
    else right--; // too big  — need a smaller value, move right down
  }
  return null; // no pair found
}

const sorted = [1, 3, 5, 7, 9, 11, 15];
twoSumSorted(sorted, 16); // { left: 1, right: 5 } → 3 + 13... wait:
// arr[1]=3, arr[5]=11 → 3+11=14 < 16, so left moves to 2
// arr[2]=5, arr[5]=11 → 5+11=16 ✓ → { left: 2, right: 5 }
```

Why does this work? If the current sum is too small, we need a larger element — and since the array is sorted, the only larger elements are to the right of `left`. Moving `left` up is the only rational move. Same logic applies when the sum is too large.

## Remove duplicates from a sorted array (fast/slow pointers)

Two pointers moving in the same direction — one "fast" (the scanner) and one "slow" (the writer) — can compact an array in place.

```javascript
function removeDuplicatesSorted(arr) {
  if (arr.length === 0) return 0;
  let slow = 0; // points to the last unique element written
  for (let fast = 1; fast < arr.length; fast++) {
    if (arr[fast] !== arr[slow]) {
      slow++;
      arr[slow] = arr[fast]; // overwrite the next slot with the new unique value
    }
  }
  return slow + 1; // length of the deduplicated section
}

const arr = [1, 1, 2, 3, 3, 3, 4, 5, 5];
const len = removeDuplicatesSorted(arr);
console.log(arr.slice(0, len)); // [1, 2, 3, 4, 5]
```

`fast` visits every element once. `slow` only advances when a new unique value is found. Total: O(n).

## Container with most water

Classic problem: given an array where `heights[i]` is the height of a wall at position i, find the two walls that trap the most water.

```javascript
function maxWater(heights) {
  let left = 0;
  let right = heights.length - 1;
  let maxArea = 0;
  while (left < right) {
    const width = right - left;
    const height = Math.min(heights[left], heights[right]);
    maxArea = Math.max(maxArea, width * height);
    // Move the shorter wall — moving the taller one can only reduce height
    if (heights[left] < heights[right]) left++;
    else right--;
  }
  return maxArea;
}

maxWater([1, 8, 6, 2, 5, 4, 8, 3, 7]); // 49 (walls at index 1 and 8)
```

The insight: water volume is limited by the shorter wall. Moving the taller wall inward can only shorten the width without gaining height — no improvement possible. Moving the shorter wall _might_ find a taller one, potentially increasing area.

## When to reach for two pointers

Two pointers work when:

- The array is **sorted** (enables the directional logic of pair sum).
- The problem has **symmetry** (palindrome — left and right converge on a center).
- You need to **partition or compact** in place (remove duplicates, move zeros).
- The problem involves **two elements** and a constraint (pair sum, closest pair, container area).

If the array is unsorted and the problem requires exact pairs with no ordering property, you likely need a hash set or map instead.

## What you learned

- **Inward-scanning** two pointers start at opposite ends and converge in O(n) — each pointer moves at most n steps total.
- **Pair sum** on a sorted array: if sum is too small move left up, if too large move right down — O(n) vs O(n²) brute force.
- **Fast/slow pointers** compact arrays in-place: slow marks the write position, fast scans all elements — O(n) and O(1) space.
- Two pointers work best on **sorted arrays**, **symmetric** problems, and **two-element constraint** problems.
- When the array is unsorted and you need arbitrary pairs, a hash set or map is the right tool instead.
