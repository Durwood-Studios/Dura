## Why this matters

Frequency counting is a pattern, not a trick. Once you recognize it, you will see it everywhere: anagram detection, histogram problems, finding the most common element, checking if a string uses only allowed characters. It is one of the most compact transformations in all of computer science — you reduce an O(n²) comparison problem to an O(n) counting problem.

## The core pattern

Scan once, count everything, query the counts.

```javascript
function buildFrequencyMap(items) {
  const freq = new Map();
  for (const item of items) {
    freq.set(item, (freq.get(item) ?? 0) + 1);
  }
  return freq;
}
```

The `?? 0` (nullish coalescing) handles the first time an item appears — `freq.get(item)` returns `undefined`, which `?? 0` converts to 0 before adding 1. After this one pass, you have O(1) access to the count of any element.

## Anagram detection

Two strings are anagrams if they contain exactly the same characters in the same quantities. Frequency counting makes this trivial:

```javascript
function isAnagram(s, t) {
  if (s.length !== t.length) return false;
  const freq = buildFrequencyMap(s);
  for (const char of t) {
    const count = freq.get(char);
    // char in t but not in s — not an anagram
    if (!count) return false;
    freq.set(char, count - 1);
    if (count - 1 === 0) freq.delete(char);
  }
  // If freq is empty, all counts balanced out
  return freq.size === 0;
}
```

Time: O(n). Space: O(k) where k is the alphabet size (26 for lowercase English, effectively O(1)).

## Finding anagram positions in a string (sliding window)

A harder problem: find all starting indices in string `s` where an anagram of pattern `p` begins. The key insight is that a fixed-size window of characters can be compared to the pattern's frequency map in O(1) per step if you **update the window incrementally** instead of rebuilding the frequency map from scratch.

```javascript
function findAnagrams(s, p) {
  const result = [];
  if (s.length < p.length) return result;

  const pFreq = buildFrequencyMap(p);
  const windowFreq = new Map();
  const k = p.length;

  // Initialize the first window
  for (let i = 0; i < k; i++) {
    const c = s[i];
    windowFreq.set(c, (windowFreq.get(c) ?? 0) + 1);
  }

  // Check first window
  if (mapsEqual(pFreq, windowFreq)) result.push(0);

  // Slide the window
  for (let i = k; i < s.length; i++) {
    // Add the new character entering the window
    const incoming = s[i];
    windowFreq.set(incoming, (windowFreq.get(incoming) ?? 0) + 1);

    // Remove the character leaving the window
    const outgoing = s[i - k];
    windowFreq.set(outgoing, windowFreq.get(outgoing) - 1);
    if (windowFreq.get(outgoing) === 0) windowFreq.delete(outgoing);

    if (mapsEqual(pFreq, windowFreq)) result.push(i - k + 1);
  }
  return result;
}

function mapsEqual(a, b) {
  if (a.size !== b.size) return false;
  for (const [key, val] of a) {
    if (b.get(key) !== val) return false;
  }
  return true;
}
```

This runs in O(n × k) for the map comparison. A more optimized version tracks the number of "matches" between the two maps to avoid full comparison, achieving true O(n).

## Top-K most frequent elements

Given an array, return the k elements that appear most often. The naive approach sorts by frequency — O(n log n). You can do O(n) with bucket sort by frequency:

```javascript
function topKFrequent(nums, k) {
  // Step 1: count frequencies — O(n)
  const freq = new Map();
  for (const n of nums) {
    freq.set(n, (freq.get(n) ?? 0) + 1);
  }

  // Step 2: bucket sort by frequency — O(n)
  // bucket[i] holds all numbers that appear exactly i times
  const buckets = new Array(nums.length + 1).fill(null).map(() => []);
  for (const [num, count] of freq) {
    buckets[count].push(num);
  }

  // Step 3: collect from highest frequency bucket down — O(n)
  const result = [];
  for (let i = buckets.length - 1; i >= 0 && result.length < k; i--) {
    result.push(...buckets[i]);
  }
  return result.slice(0, k);
}
```

Total: O(n). The trick is that frequency is bounded by n, which makes bucket sort viable.

## Subarray sum equals k (prefix sums + frequency)

Count the number of contiguous subarrays that sum to exactly k. Brute force is O(n²). The hash map approach is O(n) using **prefix sums**.

Define `prefix[i]` = sum of `nums[0..i-1]`. A subarray `nums[j..i-1]` sums to k when `prefix[i] - prefix[j] = k`, or equivalently `prefix[j] = prefix[i] - k`.

So: as you compute prefix sums left to right, check how many previous prefix sums equal `currentPrefix - k`. Store prefix sums in a frequency map.

```javascript
function subarraySum(nums, k) {
  let count = 0;
  let prefixSum = 0;
  const seen = new Map([[0, 1]]); // prefix sum 0 has occurred once (before any elements)

  for (const num of nums) {
    prefixSum += num;
    // How many previous positions j have prefix[j] = prefixSum - k?
    count += seen.get(prefixSum - k) ?? 0;
    seen.set(prefixSum, (seen.get(prefixSum) ?? 0) + 1);
  }
  return count;
}
```

The `seen` map starts with `{0: 1}` to handle subarrays starting at index 0 (a prefix sum of 0 exists before any elements are added).

## What you learned

- Frequency counting — one pass to build a count map — converts O(n²) comparison problems to O(n)
- Sliding window maintains a frequency map incrementally (add incoming, remove outgoing) to avoid O(k) rebuilds per step
- Top-K most frequent elements can be solved in O(n) with bucket sort, using frequency as the bucket index
- Prefix sums combined with a frequency map solve subarray sum in O(n) by converting "subarray sum = k" into "prefix[i] - prefix[j] = k"
