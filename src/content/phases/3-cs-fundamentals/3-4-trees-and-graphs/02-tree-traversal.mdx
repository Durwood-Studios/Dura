## Why this matters

Trees are only useful if you can visit their nodes — to search, process, or collect values. But unlike an array where you always go left to right, trees can be traversed in multiple valid orders, each with its own use case. Knowing which traversal to reach for, and why, is a fundamental skill you'll use in search algorithms, compilers, and interview problems.

## The four traversal strategies

Every traversal visits all n nodes exactly once, so all are O(n) in time. The difference is the **order** in which nodes are visited.

### Depth-first traversals (use recursion)

Three traversals explore as deep as possible before backtracking. They differ only in when the current node is processed relative to its children.

**Inorder: left → current → right**

```javascript
function inorder(node) {
  if (node === null) return [];
  return [...inorder(node.left), node.val, ...inorder(node.right)];
}
```

Why it matters: on a Binary Search Tree, inorder traversal visits nodes in **sorted ascending order**. It's the go-to for "give me all values sorted."

**Preorder: current → left → right**

```javascript
function preorder(node) {
  if (node === null) return [];
  return [node.val, ...preorder(node.left), ...preorder(node.right)];
}
```

Why it matters: the root is always first. This makes preorder ideal for **cloning or serializing** a tree — you can reconstruct the original tree by replaying insertions in preorder sequence.

**Postorder: left → right → current**

```javascript
function postorder(node) {
  if (node === null) return [];
  return [...postorder(node.left), ...postorder(node.right), node.val];
}
```

Why it matters: children are always processed before their parent. This is what you want when a parent's result depends on its children — computing subtree sizes, freeing memory (delete children before the parent), or evaluating expression trees.

### Level-order traversal (use a queue)

**Level-order (BFS): level by level, left to right**

Instead of recursion, level-order uses a **queue** (first in, first out):

```javascript
function levelOrder(root) {
  if (root === null) return [];
  const result = [];
  const queue = [root];

  while (queue.length > 0) {
    const node = queue.shift(); // dequeue from front
    result.push(node.val);
    if (node.left !== null) queue.push(node.left);
    if (node.right !== null) queue.push(node.right);
  }

  return result;
}
```

Why it matters: level-order finds the **shortest path** from the root to any target node (in terms of number of edges). It's also the natural way to print a tree level by level or to find nodes at a specific depth.

## Choosing the right traversal

| Goal                                      | Use                       |
| ----------------------------------------- | ------------------------- |
| Get BST values in sorted order            | Inorder                   |
| Copy or serialize a tree                  | Preorder                  |
| Compute subtree results before the parent | Postorder                 |
| Find nearest node to root, level-by-level | Level-order               |
| Determine tree height                     | Postorder (or either DFS) |

## Memory usage

All three depth-first traversals use O(h) memory, where h is the tree height — that's the size of the call stack. For a balanced tree, h = O(log n). For a degenerate tree (linked list shape), h = O(n).

Level-order uses O(w) memory, where w is the maximum width of the tree. For a perfect binary tree, the last level has n/2 nodes, so worst-case memory is O(n).

## The iterative inorder (bonus)

Recursive depth-first traversals can always be rewritten iteratively using an explicit stack. The iterative inorder is a common interview question because it's trickier than it looks:

```javascript
function inorderIterative(root) {
  const result = [];
  const stack = [];
  let curr = root;

  while (curr !== null || stack.length > 0) {
    // Go as far left as possible
    while (curr !== null) {
      stack.push(curr);
      curr = curr.left;
    }
    // Process the node, then move right
    curr = stack.pop();
    result.push(curr.val);
    curr = curr.right;
  }

  return result;
}
```

The key insight: push nodes as you go left, then pop to process and pivot right. The stack simulates the call stack of the recursive version.

## What you learned

- **Inorder** (left → node → right): produces sorted output on BSTs
- **Preorder** (node → left → right): root first — ideal for cloning or serializing
- **Postorder** (left → right → node): children before parent — ideal for bottom-up computation
- **Level-order** (BFS): uses a queue, visits nodes level by level
- All traversals are O(n) time; DFS uses O(h) space, BFS uses O(w) space
