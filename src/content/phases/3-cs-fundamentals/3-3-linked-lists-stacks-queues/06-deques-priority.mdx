## Why this matters

Stack and queue are the most restrictive useful data structures — they expose only one end. Once you need to add or remove from both ends efficiently, you need a deque. Once you need "serve the most important item first" instead of "serve the oldest item first," you need a priority queue. These two generalizations appear constantly: deques underpin the sliding window maximum algorithm used in stock price analysis and video processing; priority queues are the backbone of Dijkstra's shortest path, A\* search, and operating system schedulers.

## The deque (double-ended queue)

A **deque** (pronounced "deck") supports four O(1) operations instead of two:

- **pushFront(val)**: add to the front
- **pushBack(val)**: add to the back
- **popFront()**: remove and return from the front
- **popBack()**: remove and return from the back

A stack is a deque that only uses `pushBack` and `popBack`. A queue is a deque that only uses `pushBack` and `popFront`. The deque generalizes both.

JavaScript has no built-in deque. You can simulate one with a doubly linked list (all four operations are O(1) with sentinel nodes), or approximate it with a circular buffer array for cache-friendly performance.

### Deque applications

**Palindrome checking**: add characters one at a time to the back, then compare by popping from the front and back simultaneously. If all pairs match, the string is a palindrome.

**Sliding window maximum** (covered more in lesson 07): maintain a deque of candidate indices. Elements enter from the back when discovered and are evicted from the front when they fall out of the window. The front always holds the current maximum's index.

**Undo/redo with branching history**: a deque lets you navigate backward (popBack) or forward (pushBack) and also insert an alternative at the current position.

## The priority queue

A **priority queue** is a different generalization: instead of ordering items by insertion time (FIFO) or access recency (LIFO), it always returns the item with the highest (or lowest) **priority value** next.

Core operations:

- **insert(val)**: add an item with a given priority — O(log n)
- **extractMin()** or **extractMax()**: remove and return the highest-priority item — O(log n)
- **peek()**: look at the highest-priority item — O(1)

Why O(log n) and not O(1)? Because after removing the top item, the structure must reorganize to determine the new highest-priority item. The best general-purpose data structure for this is the **heap**.

## The binary heap

A **min-heap** is a complete binary tree where every node's value is less than or equal to its children. The smallest value is always at the root — O(1) access.

Heaps are stored in arrays — no node objects, no pointers. For a node at index `i`:

- Left child: index `2i + 1`
- Right child: index `2i + 2`
- Parent: index `Math.floor((i - 1) / 2)`

```javascript
class MinHeap {
  constructor() {
    this._data = [];
  }

  peek() {
    return this._data[0]; // root is always the minimum
  }

  insert(val) {
    this._data.push(val); // add to end
    this._bubbleUp(); // restore heap property
  }

  extractMin() {
    if (!this._data.length) throw new Error("Heap is empty");
    const min = this._data[0];
    const last = this._data.pop();
    if (this._data.length > 0) {
      this._data[0] = last; // move last element to root
      this._sinkDown(0); // restore heap property downward
    }
    return min;
  }

  _bubbleUp() {
    let i = this._data.length - 1;
    while (i > 0) {
      const parent = Math.floor((i - 1) / 2);
      if (this._data[parent] <= this._data[i]) break; // heap property satisfied
      // swap with parent
      [this._data[i], this._data[parent]] = [this._data[parent], this._data[i]];
      i = parent;
    }
  }

  _sinkDown(i) {
    const n = this._data.length;
    while (true) {
      const left = 2 * i + 1;
      const right = 2 * i + 2;
      let smallest = i;

      if (left < n && this._data[left] < this._data[smallest]) smallest = left;
      if (right < n && this._data[right] < this._data[smallest]) smallest = right;

      if (smallest === i) break; // already in the right place
      [this._data[i], this._data[smallest]] = [this._data[smallest], this._data[i]];
      i = smallest;
    }
  }

  size() {
    return this._data.length;
  }
}
```

**Bubble up**: after inserting at the end, swap upward with the parent as long as the new element is smaller than its parent. This restores the min-heap property from bottom to top.

**Sink down**: after replacing the root with the last element (during extractMin), swap downward with the smaller child as long as the element is larger than a child. This restores the min-heap property from top to bottom.

Both operations touch at most `O(log n)` nodes — the height of the tree.

## Min-heap vs max-heap

A **min-heap** always extracts the minimum. A **max-heap** always extracts the maximum. To convert a min-heap implementation to a max-heap, flip all comparisons: `<=` becomes `>=`, `<` becomes `>`.

Or use a transformation: to get a max-heap from a min-heap implementation, insert negated values (`insert(-val)`) and negate on extract (`return -extractMin()`).

## Real-world priority queue applications

**OS process scheduling**: high-priority processes jump ahead of low-priority ones. The scheduler maintains a priority queue of ready processes and always runs the highest-priority one next.

**Dijkstra's shortest path**: a min-heap holds unvisited nodes keyed by current known shortest distance. Always expand the closest unvisited node — O((V + E) log V) total.

**A\* pathfinding**: same as Dijkstra but with a heuristic term. Used in every game engine for NPC movement.

**Event simulation**: events are prioritized by timestamp. The simulation always processes the next earliest event — a min-heap on time.

**Median maintenance**: use two heaps (one max-heap for the lower half, one min-heap for the upper half) to maintain the running median in O(log n) per insertion.

## What you learned

- A deque generalizes stack and queue by supporting O(1) add/remove at both front and back.
- A priority queue always removes the highest-priority item, regardless of insertion order.
- Binary heaps implement priority queues with O(log n) insert and extract, O(1) peek.
- Heap nodes are stored in a plain array using index arithmetic — no pointers needed.
- Priority queues power Dijkstra's algorithm, OS schedulers, event simulation, and more.
