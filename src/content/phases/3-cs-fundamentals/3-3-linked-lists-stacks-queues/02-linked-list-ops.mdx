## Why this matters

Every linked list interview question — and they are among the most common — boils down to a small set of pointer manipulation patterns. Once you internalize how to track a previous pointer, how to swap next references without losing your place, and how to use two pointers moving at different speeds, you can solve the vast majority of linked list problems on sight. These are also the same patterns that appear in real code: reversing a list, detecting infinite loops, finding the midpoint without knowing the length.

## Insert at a position

To insert at index `k`, traverse to index `k-1` (the node just before the target position), then rewire two pointers. The old `prev.next` becomes the new node's `.next`, and `prev.next` is updated to point to the new node.

```javascript
function insertAt(list, index, val) {
  const newNode = new Node(val);

  // Inserting at head is a special case
  if (index === 0) {
    newNode.next = list.head;
    list.head = newNode;
    return;
  }

  let curr = list.head;
  for (let i = 0; i < index - 1; i++) {
    if (!curr) throw new Error("Index out of bounds");
    curr = curr.next;
  }
  // curr is now the node at index - 1
  newNode.next = curr.next;
  curr.next = newNode;
}
```

**Always handle the empty list and the head insertion as special cases** before entering the loop — they are the most common sources of null-pointer errors.

## Delete by value

To delete a node containing a specific value, you need to track the **previous node** so you can bypass the target. The deleted node becomes unreachable and is garbage-collected.

```javascript
function deleteByValue(list, val) {
  if (!list.head) return;

  // If the head is the target, just advance the head
  if (list.head.val === val) {
    list.head = list.head.next;
    return;
  }

  let prev = null;
  let curr = list.head;
  while (curr !== null) {
    if (curr.val === val) {
      prev.next = curr.next; // bypass curr
      return;
    }
    prev = curr;
    curr = curr.next;
  }
}
```

The key pattern: maintain a `prev` pointer that always lags one step behind `curr`. When you find the target, set `prev.next = curr.next` to skip over it.

## Reverse in-place

Reversing a linked list iteratively uses three pointers: `prev`, `curr`, and `next`. At each step, you flip the arrow — `curr.next` points backward to `prev` instead of forward. You must save `curr.next` before overwriting it, or you lose the rest of the list.

```javascript
function reverseList(head) {
  let prev = null;
  let curr = head;

  while (curr !== null) {
    const next = curr.next; // save forward link before overwriting
    curr.next = prev; // reverse the arrow
    prev = curr; // advance prev
    curr = next; // advance curr using saved reference
  }

  return prev; // prev is now the new head
}
```

Trace through `1 → 2 → 3 → null`:

| Step | prev | curr | curr.next after flip |
| ---- | ---- | ---- | -------------------- |
| 0    | null | 1    | null (1 ← null)      |
| 1    | 1    | 2    | 1 (2 ← 1)            |
| 2    | 2    | 3    | 2 (3 ← 2)            |
| 3    | 3    | null | loop ends            |

Return value is `prev = 3`, which is the new head of `3 → 2 → 1 → null`.

This is O(n) time, O(1) space — no extra memory proportional to input size.

## Find the middle: fast and slow pointers

If you do not know the length and need the middle node, you can find it in a single pass. The **slow pointer** advances one step at a time; the **fast pointer** advances two. When fast reaches the end, slow is at the middle.

```javascript
function findMiddle(head) {
  let slow = head;
  let fast = head;
  while (fast !== null && fast.next !== null) {
    slow = slow.next;
    fast = fast.next.next;
  }
  return slow; // slow is at the midpoint
}
```

For a 5-node list, fast reaches null after 2.5 cycles; slow will be at node 3 — the true middle.

## Detect a cycle: Floyd's algorithm

A **cycle** occurs when a node's `.next` points back to an earlier node, creating an infinite loop. Naive traversal would run forever. Floyd's cycle detection — also called the tortoise and hare algorithm — uses the same fast/slow pointer technique. If there is a cycle, fast will eventually lap slow and they will meet.

```javascript
function detectCycle(head) {
  let slow = head;
  let fast = head;
  while (fast !== null && fast.next !== null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true; // pointers met — cycle exists
  }
  return false; // fast hit null — no cycle
}
```

The insight: in a cycle, fast gains one step on slow per iteration. They are guaranteed to meet within at most `n` iterations where `n` is the cycle length.

## Common traps to avoid

**Null checks before every dereference.** Before accessing `curr.next`, confirm `curr !== null`. Before accessing `curr.next.next`, confirm both `curr !== null` and `curr.next !== null`. A missing null check is the single most common linked list bug.

**The empty list.** Every function that receives a `head` must handle `head === null` as its first check. Many implementations crash on empty input.

**The single-element list.** An edge case where `head.next === null`. Reversal, deletion, and middle-finding all need to handle this without entering their main loops.

**Losing your place.** When reversing or re-linking, always save the forward reference before overwriting `.next`. If you write `curr.next = prev` before saving `curr.next`, you have just dropped the rest of the list.

## What you learned

- Insert and delete both require tracking a previous pointer so you can rewire links around the target node.
- In-place reversal uses three pointers (prev, curr, next) and is O(n) time, O(1) space.
- The fast/slow pointer technique finds the middle of a list and detects cycles in a single pass.
- Always guard against null before dereferencing .next — empty list and single-node list are the most common edge cases that break naive implementations.
