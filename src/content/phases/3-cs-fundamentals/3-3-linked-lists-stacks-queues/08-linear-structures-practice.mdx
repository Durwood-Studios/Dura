## Challenge 1: Reverse a linked list in k-group chunks

Reverse every group of k consecutive nodes. If the remaining nodes at the end are fewer than k, leave them as-is.

For `[1, 2, 3, 4, 5]` with k=2: `[2, 1, 4, 3, 5]`  
For `[1, 2, 3, 4, 5]` with k=3: `[3, 2, 1, 4, 5]`

**Approach**: check whether at least k nodes remain. If yes, reverse exactly those k nodes (the iterative reversal from lesson 02), reconnect to the rest of the list via a recursive call, and return the new head of this group.

```javascript
function reverseKGroup(head, k) {
  // Count whether k nodes remain
  let count = 0;
  let curr = head;
  while (curr && count < k) {
    curr = curr.next;
    count++;
  }
  if (count < k) return head; // fewer than k remaining — leave in place

  // Reverse k nodes starting at head
  let prev = null;
  curr = head;
  for (let i = 0; i < k; i++) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }

  // head is now the tail of the reversed group
  // curr is the start of the remaining list
  head.next = reverseKGroup(curr, k);

  return prev; // prev is the new head of the reversed group
}
```

**Complexity**: O(n) time — every node is visited a constant number of times. O(n/k) space on the call stack from recursion depth.

**Edge cases**: k=1 (no change), k > length (no change), list length not divisible by k (tail group left as-is).

## Challenge 2: Browser history system

Implement a browser history system supporting three operations:

- `visit(url)`: navigate to a new URL, clearing any forward history
- `back(steps)`: go back up to `steps` pages, return the URL you land on
- `forward(steps)`: go forward up to `steps` pages, return the URL you land on

**Approach**: maintain a list of visited URLs and a `current` pointer. Visiting truncates everything after current and appends the new URL. Back and forward clamp at the boundaries.

```javascript
class BrowserHistory {
  constructor(homepage) {
    this._history = [homepage];
    this._current = 0;
  }

  visit(url) {
    // Truncate forward history
    this._history = this._history.slice(0, this._current + 1);
    this._history.push(url);
    this._current++;
  }

  back(steps) {
    // Clamp at index 0 (cannot go before the start)
    this._current = Math.max(0, this._current - steps);
    return this._history[this._current];
  }

  forward(steps) {
    // Clamp at the last page
    this._current = Math.min(this._history.length - 1, this._current + steps);
    return this._history[this._current];
  }
}

const browser = new BrowserHistory("home.com");
browser.visit("a.com");
browser.visit("b.com");
browser.visit("c.com");
console.log(browser.back(1)); // "b.com"
console.log(browser.back(1)); // "a.com"
console.log(browser.forward(1)); // "b.com"
browser.visit("d.com"); // clears "c.com" from forward history
console.log(browser.forward(1)); // "d.com" — no forward history past here
console.log(browser.back(2)); // "home.com" — clamped
```

**Complexity**: `visit` is O(n) due to `slice` (could be O(1) with a doubly linked list). `back` and `forward` are O(1).

## Challenge 3: Reverse Polish Notation evaluation

Evaluate an arithmetic expression given in postfix (Reverse Polish Notation). Tokens are strings: either a number or one of `+`, `-`, `*`, `/`. Division truncates toward zero.

```javascript
function evalRPN(tokens) {
  const stack = [];

  for (const token of tokens) {
    if (token === "+" || token === "-" || token === "*" || token === "/") {
      const b = stack.pop(); // second operand (pushed later)
      const a = stack.pop(); // first operand (pushed earlier)

      if (token === "+") stack.push(a + b);
      else if (token === "-") stack.push(a - b);
      else if (token === "*") stack.push(a * b);
      else if (token === "/") stack.push(Math.trunc(a / b)); // truncate toward zero
    } else {
      stack.push(Number(token));
    }
  }

  return stack[0];
}
```

Trace through `["2","1","+","3","*"]`:

1. Push 2 → `[2]`
2. Push 1 → `[2, 1]`
3. `+`: pop 1 and 2, push 3 → `[3]`
4. Push 3 → `[3, 3]`
5. `*`: pop 3 and 3, push 9 → `[9]`
6. Return 9

**Complexity**: O(n) time, O(n) space for the stack in the worst case (all numbers before any operator).

**Edge cases**: negative numbers (token is a negative number string), integer division with negative operands (`Math.trunc(-7/2) === -3`, which is correct — truncate toward zero, not floor).

## Challenge 4: Sliding window maximum

Given an array and window size `k`, return the maximum value in each window.

```javascript
function maxSlidingWindow(arr, k) {
  const result = [];
  const deque = []; // stores indices; front is always the index of the window maximum

  for (let i = 0; i < arr.length; i++) {
    // Evict indices that have fallen outside the window
    while (deque.length > 0 && deque[0] < i - k + 1) {
      deque.shift();
    }

    // Evict indices from the back whose values are smaller than arr[i]
    // They can never be the window max while arr[i] is in the window
    while (deque.length > 0 && arr[deque[deque.length - 1]] < arr[i]) {
      deque.pop();
    }

    deque.push(i);

    if (i >= k - 1) {
      result.push(arr[deque[0]]);
    }
  }

  return result;
}

maxSlidingWindow([1, 3, -1, -3, 5, 3, 6, 7], 3);
// [3, 3, 5, 5, 6, 7]
```

**Complexity**: O(n) time. Each index is added once and removed at most once. O(k) space for the deque.

**Why the deque maintains the invariant**: when a new element enters, anything smaller that is still in the deque is useless — the new element is more recent and larger, so it will always beat those old smaller values as the window maximum for any future window. Only elements larger than the new arrival should be kept.

## What you learned

- Reversing a linked list in k-groups combines iterative node reversal with recursion for clean structure — O(n) time.
- Browser history uses an array with a current-position pointer; visit truncates forward history to maintain a single linear chain.
- RPN evaluation is a clean O(n) stack application — push numbers, pop two operands for each operator.
- Sliding window maximum uses a deque whose front always holds the current window's maximum index — O(n) with each element entering and leaving at most once.
- All four problems combine linear structures with careful boundary handling — the complexity in edge cases, not the algorithm core.
